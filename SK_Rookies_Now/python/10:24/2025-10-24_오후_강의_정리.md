## 1교시: 과정 소개 및 아이스브레이킹 (임정석 강사)

### 강사 소개 및 강의 목표

- **강사**: 임정석
- **전문 분야**: 자바 기반 풀스택, 데이터 애널리틱스, AI
- **강의 기간**: 총 20일 (4주)
- **주요 내용**:
  - **언어**: 파이썬 (Python)
  - **분석 라이브러리**: 넘파이(Numpy), 판다스(Pandas)
  - **시각화**
  - **머신러닝 & 딥러닝**: 기본 이론 및 개념
  - **생성형 AI (GenAI)**: 활용 방법

> "저는 여러분들의 어떤 그 6개월이라는 시간을 담보하고 강의를 하는 것이기 때문에, 여러분들과 어떤 내적 커뮤니케이션을 통해서 여러분들의 실력 향상을 끌어내는 하나의 어떤 **마중물**이 되려고 생각을 하고 있습니다."

- **마중물이란?**: 지하수를 끌어올리기 위해 펌프에 먼저 붓는 한 바가지의 물. 수강생들의 잠재력을 이끌어내는 역할을 하겠다는 의미.

### 학습 방향 및 당부의 말

1.  **학습 태도 (Attitude)**
    - 4주간 온라인 강의 환경에 적응하고, 자신만의 **바이오리듬**을 만드는 시간이 필요.
    - 하루 7~8시간의 집중력 유지를 위한 명확한 **목표 설정**이 중요.
    - 온라인 교육의 특성상 나태해지기 쉬우므로, 초심을 잃지 않고 과정에 임할 것.

2.  **함께하는 성장**
    - 본 교육은 다양한 역량의 수강생들이 모인 집합 교육.
    - "혼자서 빠르게 질주하는 게 아니고요, **함께 천천히 성장해 나가는** 어떤 교육 과정입니다."
    - 이미 내용을 아는 수강생이라도 다른 사람을 배려하며 함께 나아가는 자세 필요.

3.  **적극적인 질문**
    - 모르는 부분이 생겼을 때 주저하지 말고 질문할 것.
    - Zoom의 1:1 대화 기능을 활용하여 "강사님, 빨라요", "다시 설명해주세요" 등 적극적으로 소통 요청.

### 아이스브레이킹 및 Q&A

- **지역 분포 확인**: 온라인 교육의 장점을 살려 부산, 울진, 광주, 대전 등 전국 각지에서 참여 중임을 확인.
- **사전 경험 조사**: 
  - 타 교육 기관(키트리 등) 수료 경험자 일부 확인.
  - 파이썬을 처음 접하는 수강생은 소수이며, 대부분 다뤄본 경험이 있음.

- **보안과 프로그래밍의 관계**
  - **보안의 종류**: 코드 기반 보안 vs. 시스템 기반 보안.
  - **본 과정의 초점**: 한 달간의 교육은 보안 자체보다는 **프로그래밍(파이썬)과 데이터 분석** 역량 강화에 중점.
    - 보안 데이터를 활용하여 분석 및 시각화, AI 연계 실습 진행.
  - **IT 트랙**: 현재 IT는 **자바 기반의 SI 트랙**과 **파이썬 기반의 데이터/AI 트랙**으로 양분되는 추세.
  - **코드 레벨 보안의 예시**:
    - **입력값 검증 (Validation)**: SQL 인젝션, XSS(Cross-site Scripting) 공격 방지를 위한 필터링.
    - **민감 정보 보호**: API 키 등을 코드에 하드코딩하지 않고, 환경 변수나 별도 설정 파일(.env)로 관리.
    - **의존성 관리**: 서드파티 라이브러리의 취약점 관리.
    - **데이터 암호화**: 사용자 비밀번호 등을 평문이 아닌 해시(Hash)로 암호화하여 저장.
    - **경로 조작 방지**: 파일 입출력 시 경로를 안전하게 처리.
    - **코드 난독화**: 배포 파이프라인에서 코드나 스크립트를 평문으로 두지 않고 난독화.

- **Q&A 세션**
  - **API란?**: "프로그램끼리 대화를 하기 위한 하나의 프로토콜 규칙이다." (Application Programming Interface)
  - **수업 스타일**: 이론과 실습 병행. PPT와 실습 위주로 진행하며, 교재는 참고 자료로 활용.
  - **실습 진행 방식**: 강사와 함께 코드를 따라 치는 시간 + 배운 개념을 응용하는 퀴즈 및 피드백 시간.
  - **비전공자 수준**: 개인 역량 차이가 있어 단정하기 어렵지만, 수업 내용을 잘 복습하고 랩업 시간을 충실히 활용하면 충분히 성장 가능. **선택과 집중**이 중요.
  - **환경 설정**: 로컬에는 **아나콘다(Anaconda)**, 원격 환경으로는 **구글 코랩(Colab)** 사용 예정.
  - **Mac 사용**: 분석 및 인공지능 분야는 OS에 따른 개발 환경 차이가 크지 않아 맥으로 수강하는 데 큰 어려움 없음. 이슈 발생 시 공유하며 해결.

## 2~3교시: 개발 환경 설정 및 마무리

### 실습 환경 구축

- **IDE (통합 개발 환경)**: 코드 작성을 위한 편집 툴.
  - **목적에 따른 툴 선택**: 
    - 웹 개발 등 전문적인 개발: 파이참(PyCharm), VS Code
    - 데이터 분석 및 시각화: **주피터 노트북(Jupyter Notebook)**
  - **아나콘다(Anaconda)**: 데이터 과학에 필요한 주요 파이썬 라이브러리(Numpy, Pandas 등)를 포함한 배포판. 주피터 노트북이 내장되어 있음.

- **아나콘다 설치 과정**
  1.  `anaconda.com` 접속하여 다운로드 페이지로 이동.
  2.  본인 OS(Windows/Mac)에 맞는 설치 파일 다운로드.
  3.  **설치 시 주의사항 (Windows)**:
      - 사용자 계정명이 **한글**일 경우, 설치 경로를 한글이 없는 경로(예: `C:\anaconda3`)로 변경 권장.
      - 권장 옵션(Recommended)을 선택하여 설치 진행.
  4.  설치 완료 후, 시작 메뉴에서 `Anaconda` 폴더의 `Jupyter Notebook` 실행.

- **주피터 노트북 사용법**
  - 실행 시 나타나는 검은색 프롬프트 창은 **종료하면 안 됨**.
  - 브라우저에 주피터 노트북 환경이 열림 (기본 디렉토리는 사용자 폴더).
  - **새 노트 생성**: `New` -> `Python 3` 클릭.
  - **코드 실행 단축키**:
    - `Shift + Enter`: 현재 셀 실행 후, 아래에 새 셀 생성.
    - `Alt + Enter`: 현재 셀 실행 후, 아래에 새 셀 생성 (동일 기능).
    - `Ctrl + Enter`: 현재 셀 실행 후, 커서는 현재 셀에 유지.

- **구글 코랩(Google Colab) 설정**
  - 로컬 환경에 문제가 있을 경우를 대비한 클라우드 기반 개발 환경.
  - 구글에서 `코랩` 검색 후 접속.
  - `새 노트`를 열면 주피터 노트북과 거의 동일한 환경 사용 가능.
  - **장점**: 별도 설치 불필요, GPU 사용이 가능하여 무거운 머신러닝 모델 학습에 유리.
