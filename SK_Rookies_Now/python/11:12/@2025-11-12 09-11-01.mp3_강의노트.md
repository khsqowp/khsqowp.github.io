## 1. 어제의 마무리: 보안 로그 분석 대시보드 완성하기

어제 `LLM`을 통해 로그 데이터를 분석하고 `risk_score`를 부여받는 것까지 성공했습니다. 오늘은 이 분석 결과를 `Streamlit` 대시보드에 다채롭게 시각화하는 작업을 마무리했습니다.

### ### 다중 컬럼 레이아웃으로 여러 차트 표시하기

`st.columns()`를 사용하면 화면을 여러 개의 세로 단으로 나누어 정보를 더 깔끔하게 구성할 수 있습니다. 여기서는 3개의 컬럼을 만들어 각각 '공격 유형 분포', '나라별 공격 비율', '위험도 평균'을 시각화했습니다.

#### #### 💻 코드: `st.columns`를 이용한 3단 차트 레이아웃

```python
# plotly_security_dashboard_app 2.py 파일의 view 함수 일부

# ... (st.dataframe(resultFrm) 아래) ...

# 1. 3개의 컬럼을 생성합니다.
typeCol, countryCol, riskCol = st.columns(3)

# 2. 첫 번째 컬럼에 '공격유형 분포' 차트 그리기
with typeCol:
    st.subheader('공격유형 분포')
    # 'attack_type' 컬럼의 각 값(유형)이 몇 번 나왔는지 세어서 바 차트로 표시
    st.bar_chart(resultFrm['attack_type'].value_counts())

# 3. 두 번째 컬럼에 '나라별 공격비율' 차트 그리기
with countryCol:
    st.subheader('나라별 공격비율')
    st.bar_chart(resultFrm['country'].value_counts())

# 4. 세 번째 컬럼에 '위험도 평균' 차트 그리기
with riskCol:
    st.subheader('위험도 평균')
    # 'attack_type'으로 그룹을 묶고, 각 그룹의 'risk_score' 평균을 계산
    riskMean = resultFrm.groupby('attack_type')['risk_score'].mean()
    st.bar_chart(riskMean)
```

#### #### 💻 코드 실행 상세 분석

1.  **1단계 (컬럼 생성)**: `st.columns(3)`가 세 개의 컬럼 객체를 생성하고, 파이썬의 '언패킹' 기능을 통해 `typeCol`, `countryCol`, `riskCol` 변수에 각각 할당됩니다.
2.  **2단계 (첫 번째 컬럼 작업)**: `with typeCol:` 블록 안의 코드는 첫 번째 컬럼에만 적용됩니다.
    *   `st.subheader()`로 작은 제목을 추가합니다.
    *   `resultFrm['attack_type'].value_counts()`: Pandas의 이 기능은 `attack_type` 시리즈(열)에 있는 고유한 값들의 개수를 셉니다. (예: `{'phishing': 10, 'ddos': 5, ...}`)
    *   `st.bar_chart()`: 위에서 계산된 개수 데이터를 받아 막대그래프로 시각화합니다.
3.  **3단계 (두 번째 컬럼 작업)**: `with countryCol:`도 동일한 방식으로 동작하며, 이번에는 `country` 열의 값들을 세어 국가별 공격 횟수를 보여줍니다.
4.  **4단계 (세 번째 컬럼 작업)**:
    *   `resultFrm.groupby('attack_type')['risk_score'].mean()`: Pandas의 강력한 기능입니다.
        *   `groupby('attack_type')`: 먼저 데이터프레임을 'phishing', 'ddos' 등 공격 유형별로 그룹핑합니다.
        *   `['risk_score']`: 각 그룹 내에서 `risk_score` 열만 선택합니다.
        *   `.mean()`: 선택된 `risk_score` 값들의 평균을 계산합니다.
    *   결과적으로 공격 유형별 평균 위험 점수가 계산되고, `st.bar_chart`로 시각화됩니다.
5.  **최종 결과**: 하나의 웹 페이지 행에 3개의 차트가 나란히 표시되어, 사용자는 여러 관점의 분석 결과를 한눈에 파악할 수 있게 됩니다.

### ### 조건부 필터링: 고위험 IP 목록 추출

`Pandas`의 불리언 인덱싱(boolean indexing)을 사용하면 특정 조건을 만족하는 데이터만 쉽게 필터링할 수 있습니다.

#### #### 💻 코드: `risk_score`를 이용한 데이터 필터링

```python
# plotly_security_dashboard_app 2.py 파일의 view 함수 일부

st.subheader('고위험 IP 목록')
# 1. resultFrm['risk_score'] >= 7 이라는 조건 생성
# 2. 이 조건을 만족하는(True인) 행들만 선택하여 새로운 DataFrame 생성
riskFrm = resultFrm[resultFrm['risk_score'] >= 7]
# 3. 필터링된 결과를 표로 출력
st.dataframe(riskFrm)
```

#### #### 💻 코드 실행 상세 분석
1.  **1단계 (조건 생성)**: `resultFrm['risk_score'] >= 7` 코드는 `risk_score` 열의 모든 값을 7과 비교하여 `True` 또는 `False` 값으로 이루어진 새로운 시리즈(열)를 생성합니다.
2.  **2단계 (필터링)**: `resultFrm[...]` 구문 안에 위에서 생성된 `True`/`False` 시리즈를 넣습니다. `Pandas`는 `True`에 해당하는 행들만 남기고 `False`에 해당하는 행들은 버립니다.
3.  **최종 결과**: `risk_score`가 7 이상인 '고위험' 공격 로그들만 담긴 `riskFrm`이라는 새로운 데이터프레임이 만들어지고, `st.dataframe()`을 통해 화면에 깔끔한 표로 출력됩니다.

## 2. 정규표현식: 자유로운 영혼의 LLM 응답 길들이기

### ### 왜 정규표현식을 배워야 할까?

강사님께서는 `LLM`이 반환하는 결과는 **"자유로운 영혼의 문자열"**이라고 강조하셨습니다. 우리가 `JSON` 형식으로 달라고 아무리 명확하게 지시해도, 모델의 종류나 `temperature` 설정에 따라 원치 않는 텍스트나 `Markdown` 형식이 포함될 수 있습니다.

**실제 문제 상황**:
어제 실습에서 `gpt-4o-mini` 모델은 우리가 요청한 `JSON` 데이터를 `Markdown`의 코드 블록(```json ... ```)으로 감싸서 반환했습니다. 이 문자열을 그대로 `json.loads()` 함수에 넣으면 `JSONDecodeError`가 발생하여 애플리케이션이 멈추게 됩니다.

이처럼 예측 불가능한 `LLM`의 응답에서 우리가 원하는 데이터만 정확히 추출하거나, 불필요한 부분을 제거(클렌징)하기 위해 **정규표현식(Regular Expression, 줄여서 Regex 또는 re)**은 필수적인 도구입니다.

### ### 정규표현식 핵심 개념

정규표현식은 복잡한 **'문자열 패턴'**을 정의하고, 이 패턴을 사용하여 텍스트를 검색, 추출, 변경하는 강력한 방법입니다.

#### 1) 메타문자 (Metacharacters)

특별한 의미를 가지고 패턴을 만드는 데 사용되는 문자들입니다.

| 메타문자 | 의미                                                    | 예시                           |
| :------- |
| `.`      | 모든 문자 하나 (줄바꿈 문자 `\n` 제외)                  | `a.c` -> `abc`, `a3c`, `a_c` |
| `^`      | 문자열의 **시작**                                       | `^Hello` -> `Hello` world      |
| `$`      | 문자열의 **끝**                                         | `world$` -> Hello `world`      |
| `*`      | 바로 앞의 문자가 **0회 이상** 반복                      | `ca*t` -> `ct`, `cat`, `caaat`   |
| `+`      | 바로 앞의 문자가 **1회 이상** 반복                      | `ca+t` -> `cat`, `caaat` (ct는 안됨) |
| `?`      | 바로 앞의 문자가 **0회 또는 1회** 존재                  | `colou?r` -> `color`, `colour`   |
| `{m,n}` | 바로 앞의 문자가 `m`회 이상, `n`회 이하 반복            | `a{2,3}` -> `aa`, `aaa`         |
| `[...]`  | 대괄호 안의 문자들 중 **하나**와 일치 (OR 조건)       | `[abc]` -> `a`, `b`, `c` 중 하나 |
| `(...)`  | 괄호 안의 패턴을 **하나의 그룹**으로 묶음               | `(abc)+` -> `abc`, `abcabc`     |

#### 2) 특수 패턴 (Special Sequences)

자주 사용하는 문자 집합을 간편하게 표현합니다.

| 패턴    | 의미                                     | 동일한 표현           |
| :------ | :--------------------------------------- | :-------------------- |
| `\d`    | 숫자(Digit) 하나                         | `[0-9]`               |
| `\D`    | 숫자가 아닌 문자 하나                    | `[^0-9]`              |
| `\w`    | 단어(Word) 문자 하나 (알파벳, 숫자, `_`) | `[a-zA-Z0-9_]`        |
| `\W`    | 단어 문자가 아닌 문자 하나               | `[^a-zA-Z0-9_]`       |
| `\s`    | 공백(Space) 문자 하나 (스페이스, 탭, `\n`)| `[ \t\n\r\f\v]`      |
| `\S`    | 공백 문자가 아닌 문자 하나               | `[^ \t\n\r\f\v]`      |

### ### Python `re` 모듈 활용

Python에서는 `re` 모듈을 통해 정규표현식 기능을 사용할 수 있습니다.

#### #### 💻 코드: 이메일 및 전화번호 추출

```python
# security_llm_langchain_day03.ipynb 파일의 일부
import re

txt = '문의하신 고객의 전화번호는 010-1234-5678 이고,  이메일은 jslim9413@naver.com 입니다.'

# 이메일 패턴: (단어문자 1회 이상)@(단어문자 1회 이상)\.(단어문자 1회 이상)
patternEmail = r'\w+@\w+\.\w+'
# 전화번호 패턴: (숫자 3자리)-(숫자 3~4자리)-(숫자 4자리)
patternPhone = r'\d{3}-\d{3,4}-\d{4}'

# re.search(): 텍스트 전체에서 패턴과 일치하는 첫 번째 부분을 찾음
email_match = re.search(patternEmail, txt)
if email_match:
    print(email_match.group()) # 결과: jslim9413@naver.com

# re.findall(): 텍스트 전체에서 패턴과 일치하는 모든 부분을 리스트로 반환
phone_matches = re.findall(patternPhone, txt)
print(phone_matches) # 결과: ['010-1234-5678']
```

#### #### 💻 코드 실행 상세 분석

1.  **1단계 (패턴 정의)**:
    *   `patternEmail`: `\w+` (단어문자 1회 이상)가 `@`와 `.`으로 연결된 전형적인 이메일 형식을 정의합니다. `r''`은 'raw string'을 의미하며, `\` 문자를 특별하게 해석하지 말라고 알려주는 역할을 합니다. 정규식 패턴을 정의할 땐 항상 사용하는 것이 좋습니다.
    *   `patternPhone`: `\d{3}` (숫자 3개), `-`, `\d{3,4}` (숫자 3개 또는 4개), `-`, `\d{4}` (숫자 4개) 패턴으로 전화번호 형식을 정의합니다.
2.  **2단계 (이메일 검색)**: `re.search(patternEmail, txt)`가 `txt` 문자열을 스캔하여 `patternEmail`과 일치하는 첫 번째 부분을 찾고, Match 객체로 반환합니다. `email_match.group()`은 실제 매칭된 문자열(`jslim9413@naver.com`)을 추출합니다.
3.  **3단계 (전화번호 찾기)**: `re.findall(patternPhone, txt)`는 `txt` 문자열 전체를 스캔하여 `patternPhone`과 일치하는 모든 부분을 찾아 리스트 형태로 반환합니다.
4.  **최종 결과**: 각 함수는 정의된 패턴에 따라 문자열에서 원하는 정보를 정확하게 찾아내어 출력합니다.

## 3. LangChain 심화: RAG 없는 `LLMChain` 활용법

`LangChain`은 `LLM`과 `RAG`를 연결하는 데만 쓰이는 것이 아닙니다. 프롬프트를 체계적으로 관리하고 `LLM`을 더 쉽게 호출하는 용도로도 매우 유용합니다. 여기서는 RAG 없이 `LLMChain`을 사용하는 방법을 배웠습니다.

### ### `LLMChain`과 `ChatPromptTemplate`

*   **`ChatPromptTemplate`**: `f-string`처럼 변수를 포함할 수 있는 프롬프트 템플릿입니다. 이를 통해 프롬프트를 재사용하고 동적으로 내용을 채울 수 있습니다.
*   **`LLMChain`**: `LLM` 모델과 `ChatPromptTemplate`을 결합한 가장 기본적인 체인입니다.

#### #### 💻 코드: `LLMChain`으로 이메일 추출하기

```python
# security_llm_langchain_day03.ipynb 파일의 일부
from langchain.chat_models import ChatOpenAI
from langchain.prompts import ChatPromptTemplate
from langchain.chains import LLMChain

# 1. LLM 모델 준비 (보수적인 답변을 위해 temperature=0)
chat = ChatOpenAI(model_name="gpt-3.5-turbo", temperature=0)

# 2. {text} 라는 변수를 포함하는 프롬프트 템플릿 생성
question = '섭섭님과 상담이 필요하면 jslim9413@naver.com 또는 jslim9413@gmail.com 으로 문의 부탁드립니다.'
prompt = ChatPromptTemplate.from_template(
    "해당 텍스트에서 이메일을 추출해줘 : {text}"
)

# 3. 모델과 프롬프트 템플릿을 LLMChain으로 연결
chain = LLMChain(llm=chat, prompt=prompt)

# 4. 체인 실행 시, 템플릿의 {text} 변수에 question 값을 전달
answer = chain.run(text=question)
print(answer)
```

#### #### 💻 코드 실행 상세 분석
1.  **1단계 (모델 준비)**: `ChatOpenAI` 모델을 `temperature=0`으로 설정하여 일관된 답변을 생성하도록 준비합니다.
2.  **2단계 (템플릿 생성)**: `ChatPromptTemplate.from_template()`을 사용하여 `{text}`라는 플레이스홀더를 가진 프롬프트 템플릿을 만듭니다.
3.  **3단계 (체인 결합)**: `LLMChain`에 준비된 모델(`llm=chat`)과 템플릿(`prompt=prompt`)을 전달하여 체인을 생성합니다.
4.  **4단계 (체인 실행)**: `chain.run(text=question)`이 실행되면, `LangChain`은 내부적으로 다음 작업을 수행합니다.
    *   `prompt` 템플릿의 `{text}` 부분을 `question` 변수의 내용으로 치환합니다.
    *   완성된 프롬프트 ("해당 텍스트에서 이메일을 추출해줘 : 섭섭님과...")를 `chat` 모델에 전달합니다.
    *   `LLM`이 생성한 답변(`jslim9413@naver.com, jslim9413@gmail.com`)을 `answer` 변수에 반환합니다.
5.  **최종 결과**: `print(answer)`는 `LLM`이 추출한 이메일 주소들만 깔끔하게 출력합니다. 이처럼 `LLMChain`은 프롬프트 관리와 실행을 매우 구조적으로 만들어줍니다.

## 4. Plotly 기반 동적 지도 시각화 추가

이제 대시보드에 `Plotly`를 이용해 상호작용이 가능한 지도를 추가하여, 공격이 어느 국가에서 오는지 시각적으로 보여주는 기능을 구현했습니다.

### ### Plotly 연동 과정

1.  **프롬프트 수정**: `LLM`이 공격 IP의 국가명뿐만 아니라, 해당 국가의 **위도(`latitude`)**와 **경도(`longitude`)**를 함께 반환하도록 `system_content` 프롬프트를 수정해야 합니다.
2.  **`plotly.express` 임포트**: `import plotly.express as px`를 추가합니다. `plotly.express`는 복잡한 차트를 간결한 코드로 만들게 해주는 고수준 API입니다.
3.  **`px.scatter_geo` 호출**: 지도 위에 산점도(scatter plot)를 그리는 `scatter_geo` 함수를 사용합니다.
4.  **`st.plotly_chart`로 렌더링**: `Plotly`로 생성한 `figure` 객체를 `Streamlit`에 표시합니다.

#### #### 💻 코드: `system_content` 프롬프트 수정

```python
# plotly_security_dashboard_app 2.py 파일의 일부

system_content = '''
너는 아주 멋진 사이버보안 전문가야.
사용자가 업로드한 로그 데이터를 기반으로 리스크 위험도 분석해줘.

- ip : 공격IP
- country : 공격 발생 국가명(영문)
- status : 'Blocked' 또는 'Allowed'
- risk_score : 1 ~ 10 숫자
- latitude : 해당 국가의 대략적인 위도(중심값)
- longitude : 해당 국가의 대략적인 경도(중심값)
json 형식이외의 다른 텍스트는 절대 포함하지 마.!!
예시) 
[
    {"ip" : "", "country" : "", "attack_type" : "", "status" : "" , "risk_score" :  8, "latitude" : 37.5, "longitude" : 127.0}
]
'''
```
> 💡 **중요!**: 위도와 경도 정보를 `LLM`이 생성하도록 프롬프트에 명시적으로 요구하는 것이 핵심입니다. 우리는 `LLM`을 '지리 정보 조회 API'처럼 활용하는 셈입니다.

#### #### 💻 코드: `Plotly` 지도 시각화 구현

```python
# plotly_security_dashboard_app 2.py 파일의 일부
import plotly.express as px

# ... (분석 로직 후) ...
st.subheader('공격 위치 지도(위험도 색상표시)')
fig = px.scatter_geo(
    resultFrm,
    lat='latitude',
    lon='longitude',
    color='risk_score',
    hover_name='ip',
    hover_data=['country','attack_type'],
    title='Attack Map',
    projection='natural earth',
    color_continuous_scale='Reds',
    size='risk_score'
)
st.plotly_chart(fig, use_container_width=True)
```

#### #### 💻 코드 실행 상세 분석
1.  **1단계 (Figure 생성)**: `px.scatter_geo` 함수가 `resultFrm` 데이터프레임을 기반으로 지도 시각화 객체(`fig`)를 생성합니다.
    *   `lat`, `lon`: 위도와 경도로 사용할 열을 지정합니다.
    *   `color`: 점의 색상을 결정할 열(`risk_score`)을 지정합니다.
    *   `hover_name`: 마우스를 점 위에 올렸을 때 표시될 주된 정보(`ip`)를 지정합니다.
    *   `projection`: 지도 투영법을 설정합니다. `natural earth`는 자연스러운 지구 형태를 보여줍니다.
    *   `color_continuous_scale`: `risk_score` 값에 따라 적용될 색상 스케일을 지정합니다. `Reds`는 낮은 값은 옅은 빨강, 높은 값은 짙은 빨강으로 표시합니다.
    *   `size`: 점의 크기를 결정할 열(`risk_score`)을 지정합니다. 위험도가 높을수록 점이 커집니다.
2.  **2단계 (렌더링)**: `st.plotly_chart(fig, use_container_width=True)`가 생성된 `fig` 객체를 받아 `Streamlit` 화면에 렌더링합니다. `use_container_width=True`는 지도가 컬럼 폭에 꽉 차게 표시되도록 합니다.
3.  **최종 결과**: 사용자는 전 세계 지도 위에 공격 발생 위치가 점으로 표시되고, 위험도에 따라 색상과 크기가 다른 것을 직관적으로 파악할 수 있는 동적이고 상호작용적인 시각화 결과를 얻게 됩니다.

## 5. 실전 트러블슈팅: 라이브러리 버전 충돌 해결기

오늘 수업의 하이라이트는 예기치 않은 문제에 직면하고 해결하는 과정이었습니다. `Plotly` 코드는 문법적으로 완벽했지만, 지도가 제대로 렌더링되지 않는 문제가 발생했습니다.

> 📌 **문제 상황**: `st.plotly_chart`가 실행되었지만 지도에 아무런 데이터(점)도 표시되지 않았습니다.

#### ### 디버깅 과정

1.  **1차 의심 (코드 오류)**: `lat`, `lon` 열 이름에 오타가 있는지, 데이터가 정상적으로 전달되는지 확인했지만 코드는 정상이었습니다.
2.  **2차 의심 (데이터 타입)**: `LLM`이 반환한 위도, 경도 값이 숫자가 아닌 문자열일 수 있다고 의심하여 `resultFrm.dtypes`를 출력해 확인했지만, `float`과 `int`로 정상적인 숫자 타입이었습니다.
3.  **3차 의심 (라이브러리 버전 충돌)**: 강사님께서 코드나 데이터가 아닌, 라이브러리 간의 호환성 문제일 수 있다고 판단하셨습니다. `Streamlit`과 `Plotly`는 계속 업데이트되므로, 특정 버전 조합에서 호환성 문제가 발생할 수 있습니다.

#### ### 해결 과정

터미널에서 `pip show` 명령어를 통해 설치된 라이브러리 버전을 확인했습니다.

```bash
# Plotly 버전 확인
pip show plotly

# Streamlit 버전 확인
pip show streamlit
```

강사님의 환경과 수강생들의 환경에서 버전이 다른 것을 확인했고, 최신 버전으로 업데이트하여 문제를 해결하기로 했습니다.

#### #### 💻 해결책: 패키지 업그레이드

```bash
# 실행 중인 streamlit 서버를 Ctrl+C로 종료한 후, 아래 명령어를 실행
pip install --upgrade streamlit plotly
```

#### #### 💻 코드 실행 상세 분석

1.  **1단계 (서버 종료)**: `pip`으로 패키지를 수정하기 전에, 해당 패키지를 사용하고 있는 `Streamlit` 프로세스를 먼저 종료해야 합니다.
2.  **2단계 (업그레이드)**: `pip install --upgrade` 명령어는 지정된 패키지들(`streamlit`, `plotly`)을 현재 설치된 버전과 상관없이 `PyPI`(Python Package Index)에 등록된 가장 최신 안정 버전으로 재설치합니다.
3.  **최종 결과**: 최신 버전의 라이브러리들을 설치한 후 `streamlit run ...`으로 앱을 다시 실행하자, 모든 문제가 해결되고 지도가 정상적으로 렌더링되었습니다.

> 💡 **오늘의 가장 큰 교훈**: 개발에서 문제가 발생했을 때, 항상 내 코드만 의심해서는 안 됩니다. 내가 작성한 코드는 완벽하더라도, 코드가 실행되는 **환경(Environment)**, 즉 라이브러리 버전, 운영체제, Python 버전 등의 외부 요인이 문제의 원인일 수 있습니다. '내 코드는 맞는데 왜 안 되지?'라는 생각이 들 때, 실행 환경을 점검해보는 것이 훌륭한 트러블슈팅 전략이라는 것을 몸소 체험했습니다.
