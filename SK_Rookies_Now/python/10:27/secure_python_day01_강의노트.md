### Part 1. 왜 보안을 위해 코딩을 배워야 할까? (중요!)

- 프로그래밍은 이제 개발자만의 영역이 아님. **보안 전문가의 핵심 무기**.

- **공격의 원리를 이해하려면 코드를 알아야 한다.**
    - 대부분의 웹 해킹 (**`SQL 인젝션`**, **`XSS`** 등)은 결국 코드의 논리적 허점을 파고드는 것.
    - 공격 코드를 읽고 이해할 수 있어야, 방어 코드도 제대로 짤 수 있다.

- **반복 업무를 자동화해서 효율을 극대화한다.**
    - **기억할 것**: 보안 업무 = 수많은 반복 작업의 연속.
    - 예시: 수십만 줄 로그 파일 분석, 주기적인 서버 취약점 스캔, 공격 IP 자동 차단.
    - 이런 건 사람이 손으로 할 수 없음. **파이썬**으로 자동화 스크립트를 만들어서 처리해야 함.

- **파이썬의 넓은 활용 범위**
    - 웹 보안, 데이터 분석, **AI 기반 이상행위 탐지**, 클라우드 보안 설정, 시스템 포렌식 등 거의 모든 보안 분야에서 사용됨.

### Part 2. 파이썬의 기본 특징: "왜 이렇게 동작할까?"

#### 1. 컴파일 vs 인터프리터

- **컴파일 언어 (`Java`, `C++`)**: 코드를 한 번에 기계어로 번역해서 실행 파일을 만듦. 실행은 빠르지만, 수정할 때마다 번역(컴파일)을 다시 해야 해서 번거로움.
- **인터프리터 언어 (`Python`, `JavaScript`)**: 코드를 한 줄씩 실시간으로 번역하며 실행. 개발 속도가 빠르고 편리함. 파이썬이 바로 이것!

    > **📌 노트**: 파이썬은 **동적 타입 언어**다. 변수를 만들 때 `int a = 10;` 처럼 타입을 지정하지 않음. `a = 10` 이라고 쓰면 파이썬이 알아서 "아, 이건 정수구나" 하고 타입을 지정함. 이게 편하긴 한데, 내가 의도치 않은 타입의 데이터가 변수에 들어갈 수 있어서 **논리적 오류(Human Error)**가 발생하기 쉬움. **따라서 `type()` 함수 등으로 항상 타입을 확인하는 습관이 중요!**

#### 2. 변수와 이름 짓기 (Naming Convention)

- **변수**: 데이터를 담는 그릇(메모리 공간)에 붙이는 이름.
- **이름 짓기 규칙 (가독성을 위해!)**
    - `camelCase`: `myVariableName` (자바스크립트 등에서 많이 씀)
    - `PascalCase`: `MyClassName` (파이썬에선 **클래스** 이름에만 사용!)
    - `snake_case`: `my_variable_name` (파이썬 커뮤니티 **강력 추천** 방식)

- **주의! 예약어는 변수명으로 쓸 수 없음**
    - `if`, `for`, `class` 등 파이썬이 이미 사용하기로 약속한 단어들.
    - 어떤 것들이 있는지 궁금하면 아래 코드로 확인 가능.

    ```python
    # 실습 코드: 파이썬 예약어 목록 확인하기
    import keyword
    print(keyword.kwlist)
    ```

### Part 3. 파이썬의 핵심 부품: 데이터 타입 (예제 중심)

#### 1. 기본 타입 확인
- `type()` 함수는 모든 것의 타입을 알려주는 만능 도구.
```python
# 실습 코드: 변수 할당 및 타입 확인
year = "2025" # 따옴표가 있으니 문자열
month = 10   # 따옴표가 없으니 숫자

print(year, month)
print('year의 타입은? ', type(year))  # <class 'str'>
print('month의 타입은? ', type(month)) # <class 'int'>
```

#### 2. 문자열 (`str`)
- **핵심 특징**: **불변성(Immutable)**. 한 번 만들면 절대 내용 변경 불가. 순서가 있는 **시퀀스** 데이터.
```python
# 실습 코드: 문자열 인덱싱과 슬라이싱
name = "secure_python"

# 인덱싱 (한 글자 뽑기)
print(f"첫 글자: {name[0]}") # 's'
print(f"마지막 글자: {name[-1]}") # 'n'

# 슬라이싱 (여러 글자 잘라내기)
print(f"0~5번째: {name[0:6]}") # 'secure' (6번째 앞까지)
print(f"뒤에서 6글자: {name[-6:]}") # 'python'

# 실습 코드: 불변성 확인 (에러 발생이 정상!)
try:
    name[0] = 'S'
except TypeError as e:
    print(f"역시 에러! -> {e}")
```

#### 3. 리스트 (`list`)
- **핵심 특징**: **가변형(Mutable)**. 값을 마음대로 추가, 수정, 삭제 가능. 순서가 있는 **시퀀스**.
- 파이썬의 만능 수납상자. 어떤 데이터 타입이든 다 담을 수 있음.
```python
# 실습 코드: 리스트 생성과 조작
my_list = ["apple", 100, True, ["nested", "list"]]

# 수정 가능!
my_list[0] = "banana"
print(f"수정 후: {my_list}")

# 추가/삭제
my_list.append("new_item")
print(f"추가 후: {my_list}")
my_list.pop()
print(f"삭제 후: {my_list}")
```

#### 4. 튜플 (`tuple`)
- **핵심 특징**: **불변성(Immutable)**. 리스트와 거의 똑같은데, 절대 수정 불가!
- **중요!**: 요소가 하나일 땐 `(1,)` 처럼 반드시 쉼표를 찍어야 함. 안 그러면 그냥 숫자로 인식.
- **패킹 & 언패킹**: 소괄호 없이 여러 값을 할당하면 자동으로 튜플로 묶이고(패킹), 튜플을 여러 변수에 나누어 담을 수 있음(언패킹).
```python
# 실습 코드: 튜플의 특성
# 패킹 (소괄호 생략)
tuple_temp = "a", "b", "c"

# 언패킹
a, b, c = tuple_temp
print(f"언패킹 결과: {a}, {b}, {c}")

# 실습 코드: 불변성 확인 (에러 발생이 정상!)
try:
    tuple_temp[0] = 'z'
except TypeError as e:
    print(f"역시 에러! -> {e}")
```
> **📌 보안 노트**: 수정되면 안 되는 중요한 데이터(IP 화이트리스트, 시스템 설정값 등)는 `list` 대신 `tuple`에 보관하는 것이 훨씬 안전하다.

#### 5. 딕셔너리 (`dict`)
- **핵심 특징**: **Key-Value** 쌍으로 데이터를 관리. 순서 개념 대신 고유한 Key로 값에 접근.
- **JSON**과 거의 동일. API 통신, 복잡한 데이터 구조화에 필수.
```python
# 실습 코드: 복잡한 딕셔너리 구조
log = {
    'SQL Injection' : {
        "정의" : "사용자 입력을 검증 없이 DB 쿼리에 포함하는 공격",
        "용어" : ["입력검증", "블라인드"],
        "탐지" : ["웹 취약점 스캐너", "오류 모니터링"]
    }
}

# Key를 통해 값에 접근
print(log['SQL Injection']['용어']) # ['입력검증', '블라인드']
```

#### 6. 셋 (`set`)
- **핵심 특징**: **중복 절대 불가**, **순서 없음**.
- 수학의 집합과 동일. 합집합, 교집합, 차집합 등 연산 가능.
```python
# 실습 코드: 셋의 중복 제거 및 집합 연산
user_roles = {"admin", "user", "guest", "admin"} # "admin" 하나는 자동 삭제
print(f"중복 제거 결과: {user_roles}")

admin_perms = {"read", "write", "delete"}
user_perms = {"read"}

print(f"합집합(|): {admin_perms | user_perms}")
print(f"교집합(&): {admin_perms & user_perms}")
print(f"차집합(-): {admin_perms - user_perms}")
```
> **📌 보안 노트**: 중복을 허용하지 않는 특성 덕분에 **중복 로그인 방지** 로직에 최적. `list`에서 `in`으로 찾는 것보다 `set`에서 찾는 게 훨씬 빠름(해시 기반).

### Part 4. 함수, 메서드, 클래스, 그리고 객체(OOP)

- **함수(Function) vs 메서드(Method)**
    - `print(a)` -> `a`를 인자로 받아 독립적으로 실행 -> **함수**
    - `a.split()` -> `a`라는 객체에 속한 기능 -> **메서드**
- **클래스(Class) vs 객체(Object)**
    - `class str:`은 문자열의 **설계도(클래스)**.
    - `my_name = "kim"`은 그 설계도로 찍어낸 **실체(객체/인스턴스)**.
    - `type(my_name)` 결과가 `<class 'str'>`인 이유!
- **`dir()` 함수**: 객체가 어떤 기능을 가지고 있는지(메서드 목록) 보여주는 탐색 도구.

### Part 5. 출력을 멋지게! 포매팅 예제 모음

- **`print()`의 `sep` 옵션**
```python
print('010', '1234', '5678', sep="-") # 010-1234-5678
```

- **f-string (강력 추천!)**
```python
score = 95.12345
num = 1234567.8912

print(f"점수(소수점2자리): {score:.2f}")
print(f"금액(천단위 쉼표): {num:,}")
print(f"정렬: {'admin':>10}") # 10칸 확보 후 오른쪽 정렬
```

- **`str.format()` 메서드**
```python
print("언어: {lang}, 버전: {ver}".format(lang="Python", ver=3.13))
```

- **C-style 서식 지정 (`%`)** (보안상 비추천, 참고만)
```python
print("언어: %s, 버전: %.2f" % ("Python", 3.13))
```

- **`repr()` 함수**: 디버깅의 친구. 이스케이프 문자(`\n`)를 그대로 보여줌.
```python
data = "Hello \n Python"
print(data)      # 줄바꿈되어 보임
print(repr(data))  # 'Hello \n Python' 문자 그대로 보임
```

- **형 변환 (Type Casting)**
```python
num_str = "100"
num_int = int(num_str) # 문자열을 정수로!
print(type(num_int))
```

### Part 6. 최종 실습: 보안 로그 분석 시나리오

- **상황**: 사용자의 로그인 이벤트를 구조화된 데이터로 저장하고, 정해진 형식의 로그 메시지로 출력한다.
- **핵심 구조**: 여러 개의 로그(`dict`)를 하나의 `list`에 담아 관리 `[{}, {}, ...]`

```python
# 1. 개별 로그 이벤트를 딕셔너리로 정의
loginUser = {
    'type' : 'guest',
    'ip' : '192.168.0.10',
    'event' : 'LOGIN_ACCESS'
}

# 2. 여러 로그를 리스트에 저장
logLst = []
logLst.append(loginUser)
logLst.append(loginUser) # ... 새로운 로그들이 계속 쌓임

print(f"저장된 로그 데이터 구조: {logLst}")

# 3. 정형화된 포맷으로 로그 메시지 출력
for i, log in enumerate(logLst):
    logMsg = f"[ALERT-{i+1}] User={log['type']}, IP={log['ip']}, Event={log['event']}"
    print(logMsg)
```
> **📌 최종 정리**: 이렇게 로그를 `list`와 `dict`로 구조화하면, 반복문을 돌면서 각 로그의 `ip`나 `event` 같은 특정 정보만 정확히 뽑아내서 분석하거나 통계를 내는 등, 훨씬 더 복잡하고 유용한 작업을 수행할 수 있게 된다. 이것이 바로 보안 전문가가 파이썬으로 데이터를 다루는 첫걸음이다.
