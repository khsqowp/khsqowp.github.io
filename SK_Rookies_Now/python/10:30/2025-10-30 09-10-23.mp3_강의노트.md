### 📦 1. 함수 (Function) 와 모듈화

#### 1-1. 왜 함수를 사용해야 하는가?

어제까지 우리는 코드를 위에서 아래로 순차적으로, 절차적으로 작성했다. 이런 방식은 코드가 길어질수록 여러 문제점을 야기한다.

-   **코드의 재사용성 부재**: 동일한 로직이 여러 곳에 필요할 때마다 복사-붙여넣기를 해야 한다.
-   **가독성 저하**: 코드가 길고 복잡해져 전체 흐름을 파악하기 어렵다.
-   **유지보수의 어려움**: 로직 수정이 필요할 때, 복사된 모든 코드를 일일이 찾아 수정해야 하므로 실수가 발생하기 쉽다.

**함수**는 이러한 문제들을 해결하기 위해 특정 기능을 수행하는 코드 뭉치를 **하나의 단위로 묶어 이름을 붙인 것**이다. 함수를 사용함으로써 우리는 다음과 같은 이점을 얻는다.

-   **재사용성**: 필요할 때마다 함수의 이름만 호출하면 되므로 코드가 간결해진다.
-   **모듈화**: 복잡한 문제를 기능 단위로 잘게 쪼개어 관리할 수 있다. 이렇게 만들어진 함수들을 모아 **모듈(Module)**로, 모듈들을 모아 **패키지(Package)**로 관리할 수 있다.
-   **유지보수 용이성**: 기능 수정이 필요할 때 해당 함수 내부만 수정하면 되므로, 변경의 영향 범위를 최소화할 수 있다.

#### 1-2. 함수의 기본 구조

파이썬 함수는 `def` 키워드를 사용하여 정의한다.

```python
def 함수이름(매개변수1, 매개변수2, ...):
    """이 함수는 무엇을 하는지에 대한 설명 (Docstring)"""
    # 함수가 수행할 코드 블록
    실행코드1
    실행코드2
    ...
    return 결과값 # 결과값을 반환 (선택 사항)
```

-   **`def`**: 함수를 정의(Define)하겠다는 키워드.
-   **매개변수 (Parameter)**: 함수가 호출될 때 외부로부터 값을 전달받기 위한 변수.
-   **`return`**: 함수의 실행 결과를 호출한 쪽으로 되돌려주는 키워드. `return`이 없으면 함수는 묵시적으로 `None`을 반환한다.

> **📌 노트:** 함수를 만드는 것을 **함수 선언(Declaration) 또는 정의(Definition)**라고 하며, 만들어진 함수를 사용하는 것을 **함수 호출(Call)**이라고 한다.

#### 1-3. 다양한 인자(Argument) 전달 방식

함수를 호출할 때 매개변수에 값을 전달하는 것을 **인자(Argument)**라고 하며, 여러 방식이 존재한다.

1.  **위치 인자 (Positional Arguments)**: 가장 기본적인 방식으로, 정의된 매개변수의 순서에 맞춰 값을 전달한다.

    ```python
    def power(a, b):
        return a ** b

    # a=2, b=3으로 전달됨
    print(power(2, 3)) # 출력: 8

    # a=3, b=2로 전달됨
    print(power(3, 2)) # 출력: 9
    ```

2.  **키워드 인자 (Keyword Arguments)**: `매개변수이름=값` 형태로 직접 지정하여 전달한다. 순서와 상관없이 정확한 매개변수에 값을 전달할 수 있다.

    ```python
    # 순서를 바꿔도 키워드로 지정했기 때문에 정확히 전달됨
    # a=2, b=3으로 전달됨
    print(power(b=3, a=2)) # 출력: 8
    ```

3.  **기본값 인자 (Default Parameter Value)**: 함수를 정의할 때 매개변수에 기본값을 설정할 수 있다. 함수 호출 시 해당 인자가 생략되면 이 기본값이 사용된다.

    ```python
    def greet(name, message="안녕하세요"):
        return f"{name}님, {message}!"

    # message 인자 생략 시 기본값 "안녕하세요" 사용
    print(greet("김철수")) # 출력: 김철수님, 안녕하세요!

    # message 인자 전달 시 전달된 값 사용
    print(greet("이영희", "반갑습니다")) # 출력: 이영희님, 반갑습니다!
    ```

#### 1-4. 가변 인자 (`*args`, `**kwargs`)

함수를 정의할 때, 몇 개의 인자가 들어올지 미리 알 수 없는 경우가 있다. 이때 가변 인자를 사용한다.

-   **`*args` (arguments)**: 여러 개의 **위치 인자**를 **튜플(tuple)**로 묶어서 받는다.

    ```python
    def sum_all(*args):
        print(f"args 타입: {type(args)}")
        total = sum(args)
        return total

    result = sum_all(1, 2, 3, 4, 5) 
    print(f"합계: {result}") # 출력: 합계: 15
    ```

-   **`**kwargs` (keyword arguments)**: 여러 개의 **키워드 인자**를 **딕셔너리(dict)**로 묶어서 받는다.

    ```python
    def print_user_info(**kwargs):
        print(f"kwargs 타입: {type(kwargs)}")
        for key, value in kwargs.items():
            print(f"{key}: {value}")

    print_user_info(name='jslim', age=30, region='seoul')
    ```

-   **혼합 사용**: 일반 매개변수, `*args`, `**kwargs`는 정해진 순서대로 함께 사용할 수 있다.

    ```python
    def process_data(subject, *args, **kwargs):
        print(f'주제: {subject}')
        print(f'추가 정보 (args): {args}')
        print(f'세부 속성 (kwargs): {kwargs}')

    process_data("사용자 정보", "임섭순", "섭섭해", "쉴더스", id=100, is_active=True)
    ```

##### 🧩 실용 예제: 안전한 API 요청 URL 생성기

`**kwargs`를 활용하면 API 요청 시 필요한 파라미터를 유연하게 받아 처리할 수 있다. 여기에 **화이트리스트(Whitelist)** 기반의 검증 로직을 추가하여 보안을 강화할 수 있다.

```python
from urllib.parse import urlencode

def make_api_request(endpoint, **params):
    """가변 키워드 인자를 받아 API 요청 URL을 생성한다."""
    # ---- 보안 강화 ----
    # 허용된 파라미터 키 목록 (화이트리스트)
    allowed_keys = {'q', 'page', 'lang'}
    
    # 전달된 params 중에서 화이트리스트에 있는 키만 필터링
    safe_params = {k: v for k, v in params.items() if k in allowed_keys}

    if not safe_params:
        return endpoint
        
    query_string = urlencode(safe_params)
    return f"{endpoint}?{query_string}"

# 허용된 파라미터로 요청
result1 = make_api_request('http://api.example.com/search', q='secure', page=2)
print(result1) # http://api.example.com/search?q=secure&page=2

# 허용되지 않은 파라미터는 자동으로 필터링됨
result2 = make_api_request('http://api.example.com/search', q='python', token='hacked')
print(result2) # http://api.example.com/search?q=python
```

> #### 🔐 보안 노트: 파라미터 화이트리스팅
> 위 예제처럼, 서버가 처리할 수 있거나 처리해야만 하는 파라미터의 목록(`allowed_keys`)을 미리 정의하고, 그 외의 모든 파라미터는 무시하거나 거부하는 **화이트리스트 방식**은 매우 효과적인 보안 대책이다. 이를 통해 공격자가 의도치 않은 파라미터(예: `token`, `debug_mode`)를 주입하여 시스템의 동작을 바꾸거나 정보를 유출하려는 시도를 원천적으로 차단할 수 있다.

#### 1-5. 람다(Lambda) 함수: 익명의 한 줄 함수

-   **정의**: `lambda 매개변수: 표현식` 형태로 작성하는 이름 없는 간단한 함수.
-   **특징**:
    -   별도의 `return` 키워드 없이 표현식의 결과가 자동으로 반환된다.
    -   주로 한 번만 사용되고 버려지는 간단한 기능을 정의할 때 사용된다.
    -   다른 함수의 인자로 함수 자체를 전달할 때 매우 유용하다.

##### `map`, `filter`, `sorted`와 함께 사용하기

-   **`map(함수, 열거형)`**: 열거형의 각 요소에 함수를 적용한 결과를 반환한다.
-   **`filter(함수, 열거형)`**: 열거형의 각 요소 중 함수 적용 결과가 `True`인 것들만 걸러서 반환한다.
-   **`sorted(열거형, key=함수)`**: 특정 `key` 함수의 반환값을 기준으로 열거형을 정렬한다.

```python
numbers = [1, 2, 3, 4, 5]
word_lst = ['pineApple', 'apple', 'watermelon', 'banana']

# map과 lambda: 모든 숫자를 제곱하기
squared = list(map(lambda x: x**2, numbers))
print(f"map 결과: {squared}") # 출력: [1, 4, 9, 16, 25]

# filter와 lambda: 짝수만 걸러내기
evens = list(filter(lambda x: x % 2 == 0, numbers))
print(f"filter 결과: {evens}") # 출력: [2, 4]

# sorted와 lambda: 문자열 길이를 기준으로 정렬하기
sorted_by_len = sorted(word_lst, key=lambda x: len(x))
print(f"sorted 결과: {sorted_by_len}") # 출력: ['apple', 'banana', 'pineApple', 'watermelon']
```

> **📌 노트:** `map`과 `filter`는 그 자체로 리스트가 아닌 `map` 객체와 `filter` 객체를 반환한다. 결과를 리스트로 보려면 `list()`로 형변환을 해주어야 한다.

### 🌐 2. 변수의 생존 범위: 스코프(Scope)

#### 2-1. 왜 스코프를 알아야 하는가?

스코프는 변수가 어디까지 유효한지, 즉 **변수의 생존 범위**를 결정하는 규칙이다. 스코프를 이해하지 못하면, 다른 함수에 있는 같은 이름의 변수를 의도치 않게 수정하거나, 접근할 수 없는 변수에 접근하려다 오류를 만나는 등 디버깅하기 어려운 버그를 만들게 된다.

#### 2-2. LEGB 규칙

파이썬은 특정 변수에 접근할 때, 다음과 같은 순서로 해당 변수를 찾아 나선다.

1.  **L (Local)**: 현재 실행 중인 **지역(함수 내부)** 스코프
2.  **E (Enclosing)**: 현재 함수를 감싸고 있는 **외부 함수(들)**의 스코프
3.  **G (Global)**: 모듈 최상단에 정의된 **전역** 스코프
4.  **B (Built-in)**: `print`, `len` 등 파이썬에 내장된 이름들의 스코프

#### 2-3. `global`과 `nonlocal` 키워드

기본적으로 함수 내에서는 상위 스코프의 변수를 **읽을 수만** 있고, **수정할 수는 없다**. 수정을 시도하면, 파이썬은 그 변수를 해당 함수의 지역 변수로 새로 생성하려고 한다.

-   **`global`**: 함수 내부에서 **전역 변수**의 값을 **수정**하고 싶을 때 사용한다.
-   **`nonlocal`**: **중첩 함수**의 내부 함수에서 **외부 함수(Enclosing)의 변수**를 **수정**하고 싶을 때 사용한다.

##### 개념 증명 코드: `global`과 `nonlocal`

```python
count = 0  # G: 전역 변수

def outer():
    outer_var = 10  # E: 외부 함수(Enclosing)의 변수

    def inner():
        nonlocal outer_var  # 외부 함수의 변수를 수정하겠다고 선언
        global count        # 전역 변수를 수정하겠다고 선언

        outer_var += 1
        count += 1
        
        local_var = 100 # L: 지역 변수
        print(f"- inner: outer_var={outer_var}, count={count}, local_var={local_var}")

    inner()
    print(f"- outer: outer_var={outer_var}")

print(f"호출 전: count={count}")
outer()
outer()
print(f"호출 후: count={count}")

# 출력:
# 호출 전: count=0
# - inner: outer_var=11, count=1, local_var=100
# - outer: outer_var=11
# - inner: outer_var=12, count=2, local_var=100
# - outer: outer_var=12
# 호출 후: count=2
```

> #### 🔐 보안 노트: 전역 변수의 위험성
> 전역 변수를 남용하면 코드의 여러 곳에서 해당 변수를 수정할 수 있게 되어, 데이터의 흐름을 추적하기 어렵고 **예상치 못한 값의 변경**이 발생할 수 있다. 특히 멀티스레드 환경에서는 여러 스레드가 동시에 전역 변수에 접근하여 값을 덮어쓰는 **경쟁 상태(Race Condition)**가 발생하여 데이터 무결성을 해칠 수 있다. 따라서 전역 변수 사용은 최소화하는 것이 안전한 코딩의 첫걸음이다.

### 🎁 3. 클로저 (Closure): 상태를 기억하는 함수

#### 3-1. 왜 클로저를 사용하는가?

전역 변수는 위험하지만, 프로그램은 특정 **상태(State)**를 기억하고 유지해야 할 때가 많다 (예: 로그인 상태, 누적 카운트). **클로저**는 전역 변수를 사용하지 않고도 함수가 특정 상태를 안전하게 기억하고 관리할 수 있는 우아한 방법을 제공한다.

-   **정의**: 자신을 감싸고 있는 **외부 함수의 변수를 참조**하는 **내부 함수**를 의미한다. 클로저의 핵심은 **외부 함수가 실행을 마친 뒤에도, 내부 함수가 외부 함수의 변수들을 기억하고 접근할 수 있다**는 점이다.

#### 3-2. 클로저의 조건

1.  함수 안에 함수가 중첩되어 있어야 한다.
2.  내부 함수는 외부 함수의 변수를 참조해야 한다.
3.  외부 함수는 내부 함수를 **반환(`return`)**해야 한다.

#### 3-3. 클로저를 이용한 상태 유지 예제: 카운터 만들기

전역 변수를 사용한 카운터와 클로저를 사용한 카운터를 비교해보자.

##### 👎 나쁜 예: 전역 변수 사용

```python
count = 0

def increment_global():
    global count
    count += 1
    print(count)

increment_global() # 1
increment_global() # 2
# count 변수가 외부에 노출되어 있어 아무 곳에서나 수정될 위험이 있다.
count = 99 
increment_global() # 100
```

##### 👍 좋은 예: 클로저 사용

```python
def counter_closure():
    count = 0  # 외부 함수의 지역 변수 (자유 변수)

    def increment():
        nonlocal count # 외부 함수의 변수를 수정
        count += 1
        return count
    
    return increment # 내부 함수를 반환

# counter_closure를 호출하면, 상태(count)를 기억하는 increment 함수가 반환됨
counter1 = counter_closure()

print(counter1()) # 1
print(counter1()) # 2

# counter2는 counter1과 독립적인 자신만의 상태를 가진다.
counter2 = counter_closure()
print(counter2()) # 1
print(counter1()) # 3

# 외부에서는 count 변수에 직접 접근할 수 없어 안전하다.
# print(count) -> NameError
```

> **💡 중요!:** 클로저는 외부 함수의 실행이 끝나도 그 함수의 **스코프(변수 환경)를 캡처하여 보존**한다. `counter1` 변수는 `count=0`이라는 상태를 기억하는 `increment` 함수 자체를 담고 있는 것이다. 이로 인해 전역 변수 없이도 안전하게 상태를 유지하고 관리할 수 있다.

### ✨ 4. 데코레이터 (Decorator): 함수를 장식하는 함수

#### 4-1. 왜 데코레이터를 사용하는가?

여러 함수에 공통적으로 적용해야 하는 기능(예: 로그인 여부 확인, 실행 시간 측정, 로깅)이 있을 때, 각 함수마다 해당 코드를 반복해서 넣는 것은 비효율적이다. **데코레이터**는 기존 함수의 코드를 전혀 수정하지 않으면서, 함수를 감싸는 형태로 **공통 기능을 덧붙일 수 있게** 해주는 강력한 기능이다.

#### 4-2. 데코레이터의 동작 원리 (클로저 기반)

데코레이터는 본질적으로 **클로저**를 활용한 **고차 함수(Higher-Order Function)**이다.

1.  데코레이터 함수는 꾸며줄 함수를 인자로 받는다.
2.  데코레이터 함수 내부에, 실제 로직을 감싸는 `wrapper` 함수를 정의한다.
3.  `wrapper` 함수는 **(1)공통 기능(전처리)**, **(2)원본 함수 실행**, **(3)공통 기능(후처리)** 의 구조를 가진다.
4.  데코레이터 함수는 이 `wrapper` 함수를 반환한다.

```python
# 공통 기능 함수들
def common_checking():
    print(">>>>>> 권한을 확인합니다.")

def common_logging():
    print(">>>>>> 작업 내용을 로그에 기록합니다.")

# 데코레이터 함수 정의
def my_decorator(checking_func, logging_func):
    def wrapper():
        checking_func()      # (1) 공통 기능 (전처리)
        print('!!! 업무 로직 수행 !!!') # (2) 원본 함수의 역할
        logging_func()       # (3) 공통 기능 (후처리)
    return wrapper

# 데코레이터를 사용하여 새로운 함수를 생성
decorated_logic = my_decorator(common_checking, common_logging)

# 생성된 함수를 실행
decorated_logic()
```

> **📌 노트**: 위 예제는 데코레이터의 기본 원리를 보여주기 위한 것이다. 실제로는 `@` 심볼을 사용하여 훨씬 간결하게 데코레이터를 적용할 수 있다. (예: `@my_decorator`)
