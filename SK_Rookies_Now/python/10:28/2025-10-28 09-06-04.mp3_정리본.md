# 파이썬 기초: 객체, 변수, 복사, 함수 (2일차 강의 정리)

### 1. 강의 핵심 요약

본 강의는 파이썬의 핵심적인 데이터 처리 방식을 설명합니다. 모든 파이썬 변수는 데이터를 직접 담는 **기본 타입**이 아닌, 메모리 주소를 가리키는 **참조 타입**임을 강조합니다. 이로 인해 발생하는 **얕은 복사(Shallow Copy)**와 **깊은 복사(Deep Copy)**의 차이점을 **가변(Mutable)** 및 **불변(Immutable)** 객체의 개념과 함께 설명하며, 이것이 보안상 어떤 의미를 갖는지 다룹니다. 마지막으로, 내장 함수와 사용자 정의 함수, 그리고 반복문의 기초를 학습합니다.

### 2. 주요 개념 정리

#### 가. 파이썬의 객체 지향 특징과 변수

- **객체 지향(OOP)**: 파이썬의 모든 데이터(숫자, 문자열, 리스트 등)는 특정 **클래스(Class)**의 **인스턴스(Instance)**, 즉 **객체(Object)**입니다. 실세계의 모든 것을 모델링하는 개념입니다.
    - **명사적 특징** (이름, 나이 등) → **변수**로 정의
    - **동사적 특징** (걷다, 말하다 등) → **메서드(Method)**로 정의
- **참조 타입**: 변수는 값을 직접 저장하지 않고, 메모리에 생성된 객체의 **주소값**을 저장합니다. 파이썬의 모든 타입은 참조 타입입니다.

    ```python
    # lstTmp01과 lstTmp02는 내용이 같아도 서로 다른 객체입니다.
    lstTmp01 = [1, 2, 3]
    lstTmp02 = [1, 2, 3]

    # id()로 메모리 주소 확인
    print('instance address - ', id(lstTmp01), id(lstTmp02))

    # is 연산자로 주소값 비교 (False가 나옴)
    print('is - 주소번지를 비교하는 연산자 ' , lstTmp01 is lstTmp02)
    ```

#### 나. 함수(Function)와 메서드(Method)

- **함수**: 단독으로 실행 가능합니다. (예: `print()`, `len()`, `dir()`)
- **메서드**: 특정 **객체(인스턴스)**에 속해 있으며, `객체.메서드()` 형태로 호출합니다.

    ```python
    # 문자열(str) 객체의 메서드 활용
    url = 'http://www.naver.com'
    print('find - ', url.find('com')) # 'com'이 시작되는 인덱스 반환

    companyName = '     SK      '
    print(companyName.strip(' ')) # 양쪽 공백 제거

    companyName = 'samsung'
    print('첫번째 문자를 대문자로 ', companyName.capitalize()) # 첫 글자만 대문자로

    fileName = 'report.xls'
    print('xls로 끝나는지 확인 - ', fileName.endswith('.xls')) # 특정 문자열로 끝나는지 확인
    ```
- **사용자 정의 함수**: `def` 키워드를 사용하여 직접 함수를 만들 수 있으며, `return`으로 결과값을 반환할 수 있습니다.
    ```python
    def greet(name):
        return f"Hello~ {name}"

    result = greet('JSLim')
    print(result, type(result))
    ```

#### 다. 가변(Mutable) vs. 불변(Immutable) 객체

- 이 개념은 복사의 동작을 이해하는 데 핵심적입니다.
- **불변 (Immutable) 객체**: 생성 후 내부 값을 변경할 수 없습니다. 변경 시 새로운 객체가 생성됩니다.
  - 예: `int`, `float`, **`str`**, **`tuple`**, `bool`
- **가변 (Mutable) 객체**: 생성 후 내부 값을 변경(추가, 수정, 삭제)할 수 있습니다.
  - 예: **`list`**, **`dict`**, `set`

#### 라. 얕은 복사(Shallow Copy) vs. 깊은 복사(Deep Copy)

- **얕은 복사 (Shallow Copy)**
  - 객체의 **주소값**만 복사하는 방식입니다. 파이썬의 기본 할당(`=`) 및 `copy.copy()`가 이에 해당합니다.
  - 복사된 객체 내부의 요소가 **가변(Mutable)** 객체일 경우, 원본을 변경하면 복사본도 함께 변경됩니다.

- **깊은 복사 (Deep Copy)**
  - 객체 내부의 모든 데이터를 재귀적으로 복사하여 완전히 새로운 객체를 생성합니다.
  - 원본과 복사본이 서로에게 영향을 주지 않습니다.
  - `copy` 모듈의 `deepcopy()` 함수를 사용해야 합니다.

    ```python
    from copy import copy, deepcopy

    original = [[1, 2], [3, 4]] # 리스트는 가변(Mutable) 객체
    shallow_copy = copy(original) # 얕은 복사
    deep_copy = deepcopy(original)    # 깊은 복사

    original[0][0] = 99 # 원본의 내부 리스트(가변)를 수정

    print('original: ', original)          # [[99, 2], [3, 4]]
    print('shallow_copy: ', shallow_copy)  # [[99, 2], [3, 4]] -> 원본의 변경에 영향을 받음
    print('deep_copy: ', deep_copy)         # [[1, 2], [3, 4]] -> 영향을 받지 않음
    ```

- **보안 관점에서의 복사: White List 활용**
  - **얕은 복사**는 원본 데이터의 **무결성**을 해칠 수 있어 민감 정보(사용자 세션, 토큰 등) 처리에 부적합합니다.
  - 함수 내부에서 원본 데이터를 직접 수정하면, 의도치 않은 **정보 변조(Injection)**에 취약할 수 있습니다.
  - **해결책**: **`deepcopy`**로 원본을 보호하고, **`White List`**(허용된 값 목록)를 두어 입력값을 검증합니다.

    ```python
    # 개선 전: 원본 데이터를 직접 수정하며, 검증되지 않은 입력값(newRole)에 취약
    def updateRole_bad(data, newRole):
        data["role"] = newRole # 원본(user)이 직접 수정됨

    # 개선 후: deepcopy와 white list로 보안 강화
    user = {
        "id" : 100,
        "name" : "admin",
        "role" : "superuser"
    }

    allowedRoles = {"user", "guest", "manager"} # White List (Set 형식)

    def updateRole_good(data, newRole):
        copyUser = deepcopy(data) # 깊은 복사로 원본 보호
        if newRole in allowedRoles: # White List를 통한 입력값 검증
            copyUser["role"] = newRole
        return copyUser

    changeUser = updateRole_good(user, "guest")

    print('original - ', user)         # {'id': 100, 'name': 'admin', 'role': 'superuser'} -> 원본 유지
    print('copy     - ', changeUser)   # {'id': 100, 'name': 'admin', 'role': 'guest'}
    ```

#### 마. 반복문과 리스트 컴프리헨션

- **`for-in` 반복문**: **리스트**, **튜플**, **문자열** 등 순서가 있는(iterable) 객체의 요소를 순회합니다.
- **`range()` 함수**: 특정 범위의 숫자 열거형 객체를 생성하여 반복문에 활용됩니다.
- **리스트 컴프리헨션 (List Comprehension)**: `for` 루프와 `if` 조건을 한 줄로 간결하게 작성하여 새로운 리스트를 만드는 방법입니다. 가독성이 좋고 성능도 더 빠릅니다.

    ```python
    # 1부터 100까지의 숫자 중 3의 배수만 담은 리스트 만들기

    # 일반 for문
    hund_lst = []
    for idx in range(1, 101):
        if idx % 3 == 0:
            hund_lst.append(idx)
    print(hund_lst)

    # 리스트 컴프리헨션
    hund_lst_comp = [idx for idx in range(1, 101) if idx % 3 == 0]
    print(hund_lst_comp)
    ```

#### 바. 딕셔너리(Dictionary)와 `zip` 활용

- **딕셔너리(`dict`)**: **Key-Value** 쌍으로 데이터를 저장하며, Key는 중복될 수 없고 **불변(Immutable)** 객체여야 합니다. 데이터 접근 속도가 빠릅니다.
- **순회**: `.keys()`, `.values()`, `.items()` 메서드를 사용하여 다양한 방식으로 순회할 수 있습니다.
- **`zip()` 함수**: 여러 개의 순회 가능한(iterable) 객체들을 동일한 인덱스끼리 묶어주는 역할을 합니다. 두 리스트를 하나의 딕셔너리로 만들 때 유용합니다.

    ```python
    keys = ('key01', 'key02', 'key03')
    datas = ('sk', 'samsung', 'lg')

    dictZip = dict(zip(keys, datas))
    print(dictZip) # {'key01': 'sk', 'key02': 'samsung', 'key03': 'lg'}

    # .items()를 활용한 순회
    for key, data in dictZip.items():
        print(key, ' - ', data)
    ```

### 3. 질의응답 (Q&A)

- **Q: 마크다운(Markdown)에서 제목이나 굵은 글씨는 어떻게 작성하나요?**
  - **A:** 셀 타입을 '마크다운'으로 변경 후, `#` 개수로 제목 수준을 정하고(`<h1>` ~ `<h6>`), `**텍스트**` 형식으로 굵은 글씨를 적용할 수 있습니다. 리스트는 `-` 또는 `*` 기호를 사용합니다.

- **Q: `upper()`와 `capitalize()` 메서드의 차이점은 무엇인가요?**
  - **A:** `upper()`는 문자열의 모든 글자를 대문자로 바꾸지만, `capitalize()`는 첫 글자만 대문자로 바꾸고 나머지는 소문자로 만듭니다.

- **Q: 파일 확장자를 확인할 때 어떤 메서드를 사용하나요?**
  - **A:** 문자열 메서드인 `endswith('확장자명')`을 사용하면 특정 문자열로 끝나는지 여부를 `True`/`False`로 반환해 주어 편리합니다.

- **Q: 함수에서 `return`이 있는 것과 없는 것의 차이는 무엇인가요?**
  - **A:** `return`이 있는 함수는 호출 시 결과값을 반환하므로, 그 값을 변수에 저장하거나 다른 연산에 사용할 수 있습니다. `return`이 없는 함수는 단순히 특정 동작만 수행하고 종료됩니다.
