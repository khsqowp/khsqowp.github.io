# 📘 프로젝트 인수인계 자료 (2026-01-10)

## 1. 프로젝트 개요
- **목표:** 보안 및 개발 학습 로그(SK Rookies)를 위한 포트폴리오 겸 기술 블로그.
- **기반 기술:** Jekyll + Minimal Mistakes 테마.
- **호스팅:** GitHub Pages (`https://khsqowp.github.io`).

## 2. 디렉토리 구조
```text
.
├── SK_Rookies/          # [원본] 마크다운 노트 (이곳에 글을 작성하세요)
├── _posts/              # [생성] Jekyll이 빌드할 포스트 파일들 (스크립트로 자동 변환됨)
├── _config.yml          # [설정] 사이트 설정
├── categories.md        # [페이지] 카테고리 아카이브 (폴더 트리 구조)
├── tags.md              # [페이지] 태그 아카이브
├── index.md             # [페이지] 메인 홈페이지
└── migrate_posts.py     # [도구] 원본 -> 포스트 변환 스크립트
```

## 3. 글 작성 및 배포 가이드

### 3.1 새 글 작성 시
1. `SK_Rookies` 폴더 내 적절한 위치에 마크다운 파일을 만듭니다.
2. 파일 내용은 자유롭게 작성하되, **날짜**를 명시하면 좋습니다.
   ```markdown
   # 내 강의 노트 제목
   > 날짜: 2025-12-25
   
   내용...
   ```

### 3.2 배포 방법
터미널에서 다음 명령어를 실행하면, 원본 파일을 읽어 예쁜 제목, 날짜, 요약문을 자동으로 생성하여 배포합니다.

```bash
# 1. 포스트 변환 및 생성
python3 migrate_posts.py

# 2. GitHub 배포
git add .
git commit -m "Add new post"
git push
```

### 3.3 요약문(Excerpt) 정책
- 스크립트(`migrate_posts.py`)가 본문의 첫 번째 문단이나 인용문을 자동으로 추출하여 `excerpt`로 지정합니다.
- 만약 마음에 들지 않으면 `_posts` 폴더의 파일을 직접 열어 `excerpt: "..."` 부분을 수정해도 되지만, **원본을 수정하고 스크립트를 다시 돌리는 것을 권장**합니다.

## 4. 커스터마이징 내역
- **날짜 정렬:** 파일 내용에서 날짜를 추출하여 `YYYY-MM-DD` 포맷으로 적용, 최신순 정렬.
- **제목 정리:** 파일명이 아닌 본문의 `# 제목`을 사용하여 깔끔하게 표시.
- **카테고리:** 폴더 구조(Python, AWS 등)가 그대로 카테고리로 적용됨.
- **요약문:** 목록 페이지에서 파일명 대신 본문 요약이 보이도록 `excerpt` 필드 활용.
- **Front Matter:** 중복 방지를 위해 모든 파일의 헤더를 정규화함.
- **날짜 표기:** `_config.yml`에서 `date_format: "%Y-%m-%d"` 설정.
