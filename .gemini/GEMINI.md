# 📘 프로젝트 인수인계 자료 (2026-01-17 업데이트)

## 1. 프로젝트 개요
- **목표:** 보안 및 개발 학습 로그(SK Rookies)를 위한 포트폴리오 겸 기술 블로그.
- **기반 기술:** Jekyll + Minimal Mistakes 테마.
- **호스팅:** GitHub Pages (`https://khsqowp.github.io`).

## 2. 디렉토리 구조
```text
.
├── SK_Rookies/          # [원본] 마크다운 노트 (사용자가 이곳에 글을 작성)
├── _posts/              # [배포] Jekyll이 빌드할 최종 포스트 파일들 (AI가 변환/생성)
├── _config.yml          # [설정] 사이트 설정
├── categories.md        # [페이지] 카테고리 아카이브
├── tags.md              # [페이지] 태그 아카이브
└── index.md             # [페이지] 메인 홈페이지
```

## 3. 핵심 작업 원칙 (Critical Mandates)

### 3.1 🔒 기존 파일 절대 보호 (Read-Only)
- **대상:** 2026년 1월 17일 이전에 작성되어 `_posts/`에 저장된 모든 마크다운 파일.
- **원칙:** 해당 파일들은 **최종 완성본**으로 간주하며, 어떠한 경우에도 내용을 수정하거나 삭제하지 않는다. (`chmod 444` 설정됨)
- **예외:** 오직 **새로 추가되는 파일**에 대해서만 작업을 수행한다.

### 3.2 🚫 스크립트 자동화 금지
- `migrate_posts.py` 등의 자동화 스크립트는 사용하지 않는다.
- 모든 변환 작업은 **AI가 직접 원본 파일을 읽고, 내용을 분석하여 최적화된 형태로 `_posts`에 작성**하는 방식으로 진행한다.

## 4. 글 작성 및 배포 워크플로우

### 4.1 사용자 작업 (Step 1)
1. `SK_Rookies` 폴더 내 적절한 카테고리 폴더에 마크다운 파일을 생성한다.
2. 내용은 자유롭게 작성하되, 강의 날짜나 주제가 명확하면 좋다.

### 4.2 AI 작업 (Step 2: 변환 및 배포)
사용자가 "작업 완료" 또는 "동기화"를 요청하면 AI는 다음 절차를 수행한다.

1.  **감지:** `SK_Rookies`에 새로 추가된 파일이 있는지 확인한다.
2.  **분석:** 파일 내용을 정독하여 핵심 주제와 내용을 파악한다.
3.  **생성 (`_posts/`):**
    *   **파일명:** `YYYY-MM-DD-전문적인_영문_제목.md` 형식으로 생성.
    *   **Front Matter:**
        *   `title`: 내용에 걸맞은 **전문적이고 매력적인 한글 제목**으로 작성.
        *   `excerpt`: 본문의 핵심을 요약한 **명확하고 구체적인 설명 (1~2문장)**.
        *   `date`: 강의/작성 날짜.
        *   `categories`: 폴더 구조를 반영.
4.  **보호:** 작업이 완료된 파일은 추후 수정 방지를 위해 보호 대상으로 관리한다.
5.  **배포:** `git add`, `commit`, `push`를 수행하여 GitHub Pages에 배포한다.

## 5. 커스터마이징 내역
- **날짜 정렬:** 파일 내용에서 날짜를 추출하여 최신순 정렬.
- **제목/요약:** 파일명이나 본문 첫 줄을 그대로 쓰지 않고, **AI가 재가공한 고품질 텍스트** 사용.
- **디자인:** VS Code 스타일의 다크 테마 및 기술 블로그에 최적화된 레이아웃 적용.
