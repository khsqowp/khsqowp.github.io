## 🤔 머신러닝을 왜 배워야 할까요? (The "Why")

전통적인 보안 시스템은 '규칙 기반(Rule-based)'으로 작동합니다. 예를 들어, "IP 주소 1.1.1.1에서 5분 안에 로그인 실패가 5회 이상 발생하면 경고"와 같은 규칙을 미리 정의해 놓는 방식입니다. 이러한 방식은 알려진 위협에는 효과적이지만, 다음과 같은 한계가 있습니다.

**[규칙 기반 시스템의 한계]**
*   **새로운 위협 대응 불가:** 매일 새롭게 등장하는 변종 악성코드(Polymorphic Malware)나 제로데이 공격(Zero-day Attack)은 미리 정의된 규칙으로는 탐지하기 어렵습니다.
*   **오탐(False Positive) 및 미탐(False Negative) 증가:** 너무 많은 규칙은 오탐을 유발하고, 너무 적은 규칙은 미탐을 유발합니다. 규칙을 관리하는 데 엄청난 인력과 시간이 소요됩니다.
*   **복잡성 증가:** 시스템이 커질수록 규칙의 수가 기하급수적으로 늘어나 관리 및 유지보수가 불가능해집니다.

**[머신러닝의 해결책]**
머신러닝(Machine Learning)은 컴퓨터가 명시적인 프로그래밍 없이 **데이터로부터 학습하여 패턴을 인식하고 예측을 수행**하는 인공지능의 한 분야입니다. 보안 분야에서는 다음과 같은 방식으로 활용됩니다.

> **보안 전문가에게 머신러닝이란?**
>
> 수많은 보안 이벤트 속에서 **'정상'과 '비정상'을 스스로 학습하여 구분**하고, 인간 분석가가 놓칠 수 있는 미묘한 패턴까지 찾아내어 **새로운 위협을 탐지**하는 **'지능형 보안 분석 엔진'**을 만드는 기술입니다. 이는 보안 전문가가 단순 반복 업무에서 벗어나 더 고차원적인 위협 분석과 대응에 집중할 수 있도록 돕습니다.

## 1. 🤖 머신러닝의 기본 개념

### 1.1. 🧠 학습 유형: 지도 학습 vs 비지도 학습

머신러닝은 크게 두 가지 학습 유형으로 나뉩니다.

| 구분 | 지도 학습 (Supervised Learning) | 비지도 학습 (Unsupervised Learning) |
| :--- | :--- | :--- |
| **목표** | 정답(Label)이 있는 데이터를 학습하여 새로운 데이터의 정답을 예측 | 정답이 없는 데이터에서 숨겨진 패턴이나 구조를 발견 |
| **데이터** | 특징(Feature)과 정답(Label) 쌍으로 구성 | 특징(Feature)만으로 구성 |
| **주요 과제** | **분류(Classification)**: 스팸 메일 분류, 악성코드 탐지<br>**회귀(Regression)**: 미래 트래픽 예측, 취약점 발생 확률 예측 | **군집화(Clustering)**: 유사한 공격 그룹 분류, 사용자 행위 패턴 분석<br>**차원 축소(Dimensionality Reduction)**: 데이터 시각화, 노이즈 제거 |
| **보안 활용** | 악성코드/침입 탐지, 피싱 메일 분류, 이상 행위 예측 | 비정상 사용자/네트워크 행위 탐지, 위협 그룹 분류 |

### 1.2. 📊 모델 학습 및 평가 과정

머신러닝 모델을 개발하는 일반적인 과정은 다음과 같습니다.

```mermaid
graph TD
    A[데이터 수집] --> B{데이터 전처리};
    B --> C[특징 추출/선택];
    C --> D[데이터 분할 (훈련/검증/테스트)];
    D --> E[모델 선택];
    E --> F[모델 훈련 (훈련 데이터)];
    F --> G[모델 평가 (검증/테스트 데이터)];
    G --> H{성능 개선?};
    H -- Yes --> C;
    H -- No --> I[모델 배포];
```

> **용어 해설: 특징(Feature)과 레이블(Label)**
>
> *   **특징(Feature):** 모델이 학습하고 예측하는 데 사용되는 입력 데이터의 속성들. 예를 들어, 네트워크 트래픽 분석에서 '패킷 크기', '목적지 포트', '통신 시간' 등이 특징이 될 수 있습니다.
> *   **레이블(Label):** 모델이 예측해야 할 정답. 지도 학습에서만 존재합니다. 예를 들어, '정상/악성', '스팸/정상', '로그인 성공/실패' 등이 레이블이 될 수 있습니다.

## 2. 🧪 Scikit-learn을 이용한 머신러닝 실습

`Scikit-learn`은 파이썬의 대표적인 머신러닝 라이브러리로, 다양한 알고리즘과 유틸리티를 제공합니다.

### 2.1. 🛡️ 분류(Classification) 모델 구축: 악성 네트워크 트래픽 탐지

네트워크 트래픽의 특징(패킷 크기, 포트, 프로토콜)을 기반으로 해당 트래픽이 '정상(Benign)'인지 '악성(Malicious)'인지 분류하는 모델을 만들어 보겠습니다.

#### **PoC: `DecisionTreeClassifier`를 이용한 악성 트래픽 분류**

```python
import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score
import io

# --- 1. 데이터 준비 (가상 데이터) ---
# 실제 환경에서는 Pandas로 로그를 분석하여 이런 특징들을 추출합니다.
data = {
    'packet_size': [64, 128, 1500, 60, 1024, 70, 1400, 50, 200, 1600],
    'dest_port': [80, 443, 22, 53, 8080, 21, 443, 23, 80, 25],
    'protocol_type': [0, 0, 1, 0, 2, 1, 0, 1, 0, 1], # 0:TCP, 1:UDP, 2:ICMP (Label Encoding)
    'is_malicious': [0, 0, 1, 0, 1, 0, 0, 1, 0, 1] # 0:정상, 1:악성 (Label)
}
df = pd.DataFrame(data)

# 특징(X)과 레이블(y) 분리
X = df[['packet_size', 'dest_port', 'protocol_type']]
y = df['is_malicious']

print("--- 📊 원본 데이터 (특징과 레이블) ---")
print(df)

# --- 2. 데이터 분할: 훈련 세트와 테스트 세트 ---
# train_test_split: 데이터를 훈련용과 테스트용으로 나눕니다.
# test_size=0.3: 전체 데이터의 30%를 테스트 세트로 사용
# random_state=42: 재현 가능한 결과를 위해 난수 시드 고정
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

print(f"\n훈련 세트 크기: {len(X_train)}개, 테스트 세트 크기: {len(X_test)}개")

# --- 3. 모델 훈련 ---
# DecisionTreeClassifier: 의사결정 트리 분류기 모델 생성
model = DecisionTreeClassifier(random_state=42)
model.fit(X_train, y_train) # 훈련 데이터로 모델 학습

print("\n✅ 모델 훈련 완료!")

# --- 4. 모델 예측 및 평가 ---
y_pred = model.predict(X_test) # 테스트 데이터로 예측 수행

print("\n--- 📊 모델 평가 결과 ---")
print(f"  - 실제 레이블 (y_test): {list(y_test)}")
print(f"  - 예측 레이블 (y_pred): {list(y_pred)}")

# 💡 보안 관점: 정확도(Accuracy)만으로는 부족합니다!
#    특히 보안 데이터는 '악성' 데이터가 '정상' 데이터보다 훨씬 적은 '불균형 데이터'인 경우가 많습니다.
#    이런 경우, 악성을 제대로 탐지하는지(Recall)와 오탐이 적은지(Precision)가 더 중요합니다.
print(f"  - 정확도 (Accuracy): {accuracy_score(y_test, y_pred):.4f}")
print(f"  - 정밀도 (Precision): {precision_score(y_test, y_pred):.4f}") # 악성으로 예측한 것 중 실제 악성 비율
print(f"  - 재현율 (Recall): {recall_score(y_test, y_pred):.4f}")     # 실제 악성 중 악성으로 예측한 비율
print(f"  - F1-Score: {f1_score(y_test, y_pred):.4f}")             # Precision과 Recall의 조화 평균
```
**[분석]**
`DecisionTreeClassifier`를 훈련시켜 악성 트래픽을 분류하는 모델을 만들고, `accuracy_score`, `precision_score`, `recall_score`, `f1_score` 등의 지표로 모델의 성능을 평가했습니다. 특히 보안 분야에서는 **재현율(Recall)**이 매우 중요합니다. 실제 악성 트래픽을 놓치지 않고 얼마나 잘 탐지하는지를 나타내기 때문입니다.

### 2.2. 💥 모델의 취약점: 과적합(Overfitting)과 과소적합(Underfitting)

모델이 훈련 데이터를 너무 완벽하게 학습하여 새로운 데이터에 대한 예측 성능이 떨어지는 것을 **과적합(Overfitting)**이라고 합니다. 반대로 모델이 훈련 데이터조차 제대로 학습하지 못하는 것을 **과소적합(Underfitting)**이라고 합니다.

> **보안 관점:**
>
> *   **과적합된 보안 모델:** 훈련 데이터에만 특화되어, 조금만 변형된 새로운 공격 패턴은 탐지하지 못하는 '규칙 기반' 시스템과 다를 바 없게 됩니다.
> *   **과소적합된 보안 모델:** 기본적인 공격조차 탐지하지 못하여 무용지물이 됩니다.

## 3. 🛡️ 머신러닝 모델의 보안 위협

머신러닝 모델 자체도 공격의 대상이 될 수 있습니다. AI 기반 보안 시스템을 구축할 때는 이러한 위협을 반드시 고려해야 합니다.

### 3.1. 💥 데이터 오염 공격 (Data Poisoning Attack)

공격자가 모델의 훈련 데이터에 악의적인 데이터를 주입하여, 모델이 특정 공격을 '정상'으로 오분류하거나 특정 사용자를 '악성'으로 분류하도록 유도하는 공격입니다.

#### **PoC: 훈련 데이터 조작을 통한 모델 오염 (개념)**

```python
# ⚠️ 실제 공격은 훨씬 정교하게 이루어집니다.
#    여기서는 개념 이해를 위한 단순화된 예시입니다.

# --- 정상적인 훈련 데이터 (가정) ---
# (패킷 크기, 목적지 포트, 프로토콜 타입) -> 정상(0) / 악성(1)
X_normal = np.array([[64, 80, 0], [128, 443, 0], [1500, 22, 1]])
y_normal = np.array([0, 0, 1]) # 1500, 22, 1은 악성

# --- 공격자가 주입한 오염된 데이터 ---
# 공격자는 자신의 악성 트래픽 패턴을 '정상'으로 학습시키고 싶어 합니다.
# (공격자의 악성 패킷 크기, 공격자의 악성 목적지 포트, 공격자의 악성 프로토콜 타입) -> 정상(0)
X_poison = np.array([[1024, 8080, 2]]) # 공격자의 악성 트래픽 특징
y_poison = np.array([0]) # ⚠️ 공격자는 이 악성 트래픽을 '정상'으로 레이블링하여 주입

# --- 오염된 훈련 데이터셋 ---
X_tainted_train = np.vstack((X_normal, X_poison))
y_tainted_train = np.hstack((y_normal, y_poison))

print("--- 🧪 오염된 훈련 데이터셋 ---")
print(f"X_tainted_train:\n{X_tainted_train}")
print(f"y_tainted_train:\n{y_tainted_train}")

# 이 데이터로 모델을 훈련시키면, 모델은 공격자의 악성 트래픽을 정상으로 오분류할 가능성이 높아집니다.
# 🛡️ 방어: 데이터 수집 단계에서 엄격한 검증, 이상치 탐지, 데이터 출처 확인 등이 중요합니다.
```

### 3.2. 💥 적대적 공격 (Adversarial Attack)

모델이 예측을 수행할 때, 공격자가 입력 데이터에 인간의 눈으로는 거의 인지할 수 없는 미세한 노이즈를 추가하여 모델을 오분류하도록 유도하는 공격입니다.

> **보안 관점:**
>
> *   악성코드 탐지 모델에 미세한 노이즈를 추가하여 악성코드를 '정상'으로 우회시키거나,
> *   침입 탐지 시스템에 정상 트래픽처럼 보이는 미세한 변형을 가하여 공격을 숨길 수 있습니다.

## 👨‍💻 현직자 통합 시나리오: AI 기반 침입 탐지 시스템 (IDS) 구축 및 평가

**[상황]**
AI 보안 엔지니어 '제미니'는 네트워크 트래픽 데이터를 기반으로 침입을 탐지하는 머신러닝 모델을 개발하고 있습니다. 특히, 실제 공격을 놓치지 않는 것(높은 재현율)이 중요합니다.

**[데이터셋]**
가상의 네트워크 트래픽 데이터셋. 각 행은 하나의 네트워크 세션을 나타내며, 특징(Feature)으로는 `duration`, `src_bytes`, `dst_bytes`, `protocol_type` 등이 있고, 레이블(Label)로는 `attack` (1: 공격, 0: 정상)이 있습니다.

```python
import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier # 앙상블 모델 사용
from sklearn.metrics import classification_report, confusion_matrix
import io

# --- 1. 데이터 준비 ---
# 가상 네트워크 트래픽 데이터 (실제 데이터는 훨씬 복잡하고 큽니다)
data = {
    'duration': [10, 5, 120, 2, 30, 1, 60, 3, 90, 7],
    'src_bytes': [100, 50, 10000, 20, 500, 10, 2000, 30, 3000, 80],
    'dst_bytes': [200, 100, 50000, 40, 1000, 20, 4000, 60, 6000, 160],
    'protocol_type': [0, 0, 1, 0, 0, 0, 1, 0, 1, 0], # 0:TCP, 1:UDP
    'attack': [0, 0, 1, 0, 0, 0, 1, 0, 1, 0] # 0:정상, 1:공격
}
df = pd.DataFrame(data)

X = df[['duration', 'src_bytes', 'dst_bytes', 'protocol_type']]
y = df['attack']

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

print("--- 📊 데이터셋 요약 ---")
print(f"훈련 데이터 공격 비율: {y_train.sum() / len(y_train):.2f}")
print(f"테스트 데이터 공격 비율: {y_test.sum() / len(y_test):.2f}")

# --- 2. 모델 훈련 ---
# RandomForestClassifier: 여러 Decision Tree를 결합한 앙상블 모델로, 높은 성능과 안정성을 가집니다.
model = RandomForestClassifier(n_estimators=100, random_state=42)
model.fit(X_train, y_train)

print("\n✅ RandomForest 모델 훈련 완료!")

# --- 3. 모델 예측 및 평가 ---
y_pred = model.predict(X_test)

print("\n--- 📊 모델 성능 보고서 ---")
# classification_report: Precision, Recall, F1-score를 한눈에 보여주는 유용한 함수
print(classification_report(y_test, y_pred, target_names=['정상', '공격']))

print("\n--- 📈 혼동 행렬 (Confusion Matrix) ---")
# 혼동 행렬: 실제 값과 예측 값의 관계를 표로 보여줍니다.
# [[TN, FP], [FN, TP]]
# TN: True Negative (실제 정상 -> 예측 정상)
# FP: False Positive (실제 정상 -> 예측 공격) - 오탐
# FN: False Negative (실제 공격 -> 예측 정상) - 미탐 (가장 위험!)
# TP: True Positive (실제 공격 -> 예측 공격)
print(confusion_matrix(y_test, y_pred))

# 💡 보안 관점: 미탐(False Negative)을 최소화하는 것이 중요합니다.
#    실제 공격을 정상으로 판단하는 것은 치명적인 결과를 초래할 수 있기 때문입니다.
#    따라서 Recall (재현율) 지표를 특히 중요하게 봐야 합니다.
```

## 📌 요약 정리 (Executive Summary)

1.  **머신러닝의 필요성**: 규칙 기반 시스템의 한계를 극복하고, 새로운 위협 패턴을 데이터로부터 스스로 학습하여 탐지하는 지능형 보안 시스템 구축에 필수적이다.
2.  **학습 유형**:
    *   **지도 학습 (Supervised Learning)**: 정답(Label)이 있는 데이터로 학습하여 분류(Classification)나 회귀(Regression) 문제를 해결한다. (예: 악성코드 분류)
    *   **비지도 학습 (Unsupervised Learning)**: 정답이 없는 데이터에서 숨겨진 패턴을 찾아 군집화(Clustering)나 차원 축소(Dimensionality Reduction)를 수행한다. (예: 이상 행위 탐지)
3.  **모델 평가 지표**:
    *   **정확도 (Accuracy)**: 전체 예측 중 올바르게 예측한 비율. (불균형 데이터에서는 한계)
    *   **정밀도 (Precision)**: 모델이 '악성'이라고 예측한 것 중 실제 '악성'의 비율. (오탐 감소)
    *   **재현율 (Recall)**: 실제 '악성'인 것 중 모델이 '악성'이라고 예측한 비율. (미탐 감소, 보안에서 중요!)
    *   **F1-Score**: 정밀도와 재현율의 조화 평균.
4.  **모델의 보안 위협**:
    *   **데이터 오염 공격 (Data Poisoning)**: 훈련 데이터에 악의적인 데이터를 주입하여 모델의 판단을 왜곡시킨다.
    *   **적대적 공격 (Adversarial Attack)**: 입력 데이터에 미세한 노이즈를 추가하여 모델을 오분류하도록 유도한다.
5.  **Scikit-learn**: 파이썬의 대표적인 머신러닝 라이브러리로, 다양한 알고리즘과 모델 평가 도구를 제공한다.
6.  **보안 관점**: 머신러닝 모델은 강력하지만, 훈련 데이터의 품질, 모델의 견고성, 그리고 평가 지표의 선택에 따라 보안 시스템의 신뢰성이 크게 좌우된다. 특히 **미탐(False Negative)**을 최소화하는 것이 보안 모델의 핵심 목표이다.
