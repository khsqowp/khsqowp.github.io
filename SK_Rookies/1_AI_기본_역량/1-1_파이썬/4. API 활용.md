## 🤔 API를 왜 배워야 할까요? (The "Why")

보안 분석은 혼자 하는 것이 아닙니다. 내가 분석한 결과를 다른 팀원에게 공유해야 하고, 내가 발견한 의심스러운 IP나 파일 해시가 정말로 악성인지 외부의 전문 시스템에 물어봐야 합니다. 또한, 매일 반복되는 분석 작업을 자동화하여 더 중요한 위협에 집중할 시간을 확보해야 합니다.

**[API가 없던 시절의 문제점]**

- **수작업 정보 조회:** 분석 중 발견한 의심스러운 IP 주소를 VirusTotal 같은 웹사이트에 접속하여 직접 복사/붙여넣기 하고 결과를 확인해야 했습니다. IP가 100개라면 이 작업을 100번 반복해야 합니다.
    
- **단절된 워크플로우:** 로그 분석 결과(e.g., 공격 IP 목록)를 엑셀 파일로 저장한 뒤, 메신저나 이메일로 다른 팀원에게 전달해야 했습니다. 실시간 공유가 어렵고 과정이 번거롭습니다.
    
- **분석의 고립:** 나의 분석 시스템과 회사의 다른 보안 시스템(e.g., 방화벽, SIEM)이 연동되지 않아, 분석 결과를 바탕으로 방화벽에 차단 정책을 추가하는 등의 대응 조치가 즉각적으로 이루어지기 어려웠습니다.
    

**[API의 해결책]** API(Application Programming Interface)는 **'애플리케이션을 조작하기 위한 인터페이스(설명서)'** 입니다. 웹사이트에 직접 접속하지 않고도, 프로그램 코드를 통해 VirusTotal의 데이터에 접근하거나, Slack으로 메시지를 보내거나, AWS 클라우드 인프라를 제어할 수 있게 해주는 '연결 통로'입니다.

> **보안 전문가에게 API란?**
> 
> 나의 분석 스크립트를 다른 강력한 시스템들과 연결하여 **'자동화된 군단'**을 만드는 설계도입니다. API를 통해 우리는 24시간 쉬지 않고 위협 정보를 수집하고, 분석 결과를 실시간으로 공유하며, 탐지된 위협을 자동으로 차단하는 **'능동적인 보안 분석 및 대응 시스템(Active Security Analysis & Response System)'**을 구축할 수 있습니다.

## 1. 🎯 API 통신의 기초: `requests` 라이브러리

파이썬에서 HTTP 기반의 API를 호출할 때 가장 널리 사용되는 라이브러리는 `requests` 입니다. 사용법이 매우 직관적이고 강력합니다.

### 1.1. 📡 HTTP 메서드와 `requests`

|HTTP 메서드|목적|`requests` 함수|보안 관점에서의 주 사용처|
|---|---|---|---|
|**`GET`**|**정보 조회 (Read)**|`requests.get()`|위협 인텔리전스 조회, 설정값 확인|
|**`POST`**|**정보 생성 (Create)**|`requests.post()`|데이터 전송, 시스템에 작업 명령|
|**`PUT` / `PATCH`**|**정보 수정 (Update)**|`requests.put()` / `requests.patch()`|보안 정책 업데이트, 설정 변경|
|**`DELETE`**|**정보 삭제 (Delete)**|`requests.delete()`|임시 파일 삭제, 방화벽 룰 제거|

### 1.2. 🔬 API 응답(Response) 분석

API를 호출하면 서버는 '응답 객체'를 반환합니다. 이 객체 안에는 우리가 필요로 하는 모든 정보가 담겨 있습니다.

- `response.status_code`: HTTP 상태 코드. `200`이면 성공, `404`는 Not Found, `401`/`403`은 인증/권한 오류.
    
- `response.text`: 응답 본문(body)을 텍스트(str) 형태로 반환.
    
- `response.json()`: 응답 본문이 JSON 형식일 경우, 파이썬 `dict`나 `list`로 변환. **(가장 많이 사용!)**
    
- `response.headers`: 응답 헤더 정보를 `dict` 형태로 반환.
    

#### **PoC: 공개 API를 이용한 IP 주소 위치 정보 조회**

```
import requests
import json # JSON 데이터를 예쁘게 출력하기 위해 import

# ⚠️ 보안 관점: API 키는 절대 코드에 하드코딩하지 마세요!
#    실제 환경에서는 환경 변수(os.environ)나 별도의 설정 파일에서 불러와야 합니다.
#    여기서는 교육 목적으로 공개된 무료 API를 사용합니다.
ip_to_lookup = "8.8.8.8" # 구글의 Public DNS 서버 IP
api_url = f"[http://ip-api.com/json/](http://ip-api.com/json/){ip_to_lookup}"

try:
    print(f"📡 API 요청: {api_url}")
    response = requests.get(api_url)

    # ✅ 1. 상태 코드 확인: 항상 응답이 성공했는지 먼저 확인하는 습관!
    response.raise_for_status() # 2xx 코드가 아니면 예외(HTTPError) 발생
    
    print(f"\n✅ 요청 성공! (상태 코드: {response.status_code})")

    # ✅ 2. JSON 데이터 파싱
    ip_info = response.json()
    
    print("\n--- 🔬 수신된 JSON 데이터 ---")
    # json.dumps를 사용하여 딕셔너리를 보기 좋게 출력
    print(json.dumps(ip_info, indent=2, ensure_ascii=False))

    # ✅ 3. 필요한 정보 추출
    country = ip_info.get('country', 'N/A')
    city = ip_info.get('city', 'N/A')
    isp = ip_info.get('isp', 'N/A')

    print("\n--- 📊 분석 결과 ---")
    print(f"  - IP 주소: {ip_to_lookup}")
    print(f"  - 국가: {country}")
    print(f"  - 도시: {city}")
    print(f"  - ISP: {isp}")

except requests.exceptions.HTTPError as http_err:
    print(f"❌ HTTP 에러 발생: {http_err}")
except requests.exceptions.RequestException as req_err:
    print(f"❌ 요청 에러 발생: {req_err}")
```

**[분석]** `requests.get()` 함수로 간단하게 API를 호출하고, `.json()` 메서드로 응답을 파이썬 딕셔너리로 변환하여 손쉽게 원하는 정보를 추출했습니다. `try...except` 구문을 사용하여 네트워크 오류나 서버 오류에 대응하는 **안정적인 코드**를 작성하는 것이 매우 중요합니다.

## 2. 🎯 보안 분야의 API 활용 사례

### 2.1. 💥 VirusTotal API: 파일 해시 및 IP 평판 조회 자동화

VirusTotal은 파일, URL, IP 주소 등을 수십 개의 안티바이러스 엔진과 비교하여 악성 여부를 알려주는 매우 강력한 서비스입니다. API를 사용하면 이 과정을 완벽하게 자동화할 수 있습니다.

> **용어 해설: 파일 해시(File Hash)**
> 
> 파일의 '디지털 지문'. MD5, SHA-1, SHA-256 등의 해시 함수를 사용하여 파일의 고유한 식별값을 생성합니다. 파일 내용이 1비트만 바뀌어도 해시값은 완전히 달라집니다. 악성코드 분석 및 식별에 필수적으로 사용됩니다.

#### **PoC: 의심스러운 파일 해시(SHA-256) 목록 자동 조회**

```
import requests
import time
import os

# ⚠️ 실제 API 키는 환경 변수 등으로 안전하게 관리해야 합니다.
#    (예: 터미널에서 export VT_API_KEY="YOUR_VIRUSTOTAL_API_KEY")
API_KEY = os.environ.get("VT_API_KEY")

# API 키가 설정되지 않았다면 스크립트 중단 또는 경고
if not API_KEY:
    print("❌ 에러: VT_API_KEY 환경 변수가 설정되지 않았습니다.")
    # 이 예제에서는 계속 진행하지 않고 중단하는 것이 좋습니다.
    # raise ValueError("VT_API_KEY
else:
    HEADERS = {"x-apikey": API_KEY}
    BASE_URL = "[https://www.virustotal.com/api/v3](https://www.virustotal.com/api/v3)"

    # 분석할 파일 해시 목록 (SHA-256)
    # 첫 번째는 WannaCry 랜섬웨어, 두 번째는 정상 파일(예시)의 해시
    hashes_to_check = [
        "ed01ebfbc9eb5bbea545af4d01bf5f1071661840480439c6e5babe8e080e41aa",
        "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855" 
    ]

    for file_hash in hashes_to_check:
        print(f"\n" + "="*50)
        print(f"🔬 파일 해시 조회 중: {file_hash[:30]}...")
        
        analysis_url = f"{BASE_URL}/files/{file_hash}"
        
        try:
            response = requests.get(analysis_url, headers=HEADERS)
            response.raise_for_status()
            
            result = response.json()
            attributes = result.get('data', {}).get('attributes', {})
            
            # 최종 분석 결과 (harmless, suspicious, malicious 등)
            last_analysis_stats = attributes.get('last_analysis_stats', {})
            malicious_votes = last_analysis_stats.get('malicious', 0)
            total_votes = sum(last_analysis_stats.values())
            
            print(f"  - 📊 분석 결과: {malicious_votes} / {total_votes} 개의 엔진이 악성으로 탐지")
            
            if malicious_votes > 5: # 5개 이상 엔진이 악성으로 판단하면 '악성'으로 결론
                print(f"  - 🔴 결론: 악성 파일 (Malicious)")
                # 추가 정보 추출
                file_type = attributes.get('type_description', 'N/A')
                file_size = attributes.get('size', 0)
                print(f"  - 파일 종류: {file_type}, 크기: {file_size} bytes")
            else:
                print(f"  - ✅ 결론: 정상 파일 (Benign)")

        except requests.exceptions.HTTPError as e:
            if e.response.status_code == 404:
                print("  - ❔ 정보 없음: VirusTotal 데이터베이스에 존재하지 않는 해시입니다.")
            elif e.response.status_code == 401:
                print("  - ❌ 인증 에러: VT_API_KEY가 유효한지 확인하세요.")
            else:
                print(f"  - ❌ HTTP 에러: {e}")
        except Exception as e:
            print(f"  - ❌ 알 수 없는 에러: {e}")
            
        # ⚠️ 중요: 대부분의 Public API는 분당 요청 횟수 제한(Rate Limit)이 있습니다.
        #    이를 준수하지 않으면 일시적/영구적으로 IP나 API 키가 차단될 수 있습니다.
        #    VirusTotal 무료 API는 분당 4회 요청 제한이 있으므로, 요청 사이에 지연 시간(sleep)을 줍니다.
        time.sleep(16) # 60초 / 4회 = 15초. 여유있게 16초 대기.
```

## 👨‍💻 현직자 통합 시나리오: 보안 관제 자동화 시스템 구축

**[상황]** 보안 관제팀(SOC)의 '쉴더' 분석가는 심야 시간에 발생하는 대량의 로그인 실패 이벤트를 자동으로 탐지하고, 팀의 Slack 채널에 경고 메시지를 보내는 자동화 시스템을 구축하고자 합니다.

**[워크플로우]**

1. **(Pandas & Numpy)**: 인증 로그를 분석하여 특정 시간 내에 임계값 이상의 로그인 실패를 시도한 IP를 탐지한다.
    
2. **(API #1 - ip-api.com)**: 탐지된 IP의 위치 정보를 조회한다.
    
3. **(API #2 - Slack)**: 분석된 모든 정보를 종합하여 Slack 채널에 실시간 경고 메시지를 전송한다.
    

```
import pandas as pd
import numpy as np
import requests
import json
import io
import os # 

# --- 1. 데이터 분석 (이전 단원 복습) ---
log_content = """timestamp,source_ip,user,status
2023-11-12 23:50:10,1.2.3.4,admin,SUCCESS
2023-11-12 23:55:05,185.191.205.10,root,FAILURE
2023-11-12 23:55:06,185.191.205.10,admin,FAILURE
2023-11-12 23:55:08,185.191.205.10,guest,FAILURE
2023-11-12 23:56:20,1.2.3.4,guest,SUCCESS
2023-11-12 23:57:01,185.191.205.10,test,FAILURE
2023-11-12 23:57:02,185.191.205.10,oracle,FAILURE
"""
auth_df = pd.read_csv(io.StringIO(log_content))
failed_logs = auth_df[auth_df['status'] == 'FAILURE']
ip_failure_counts = failed_logs['source_ip'].value_counts()
suspicious_ips = ip_failure_counts[ip_failure_counts >= 5]

# --- 2. API 연동 및 자동화 ---
def get_ip_info(ip):
    """IP 주소 정보를 조회하는 함수"""
    try:
        response = requests.get(f"[http://ip-api.com/json/](http://ip-api.com/json/){ip}")
        response.raise_for_status()
        return response.json()
    except requests.RequestException:
        return None

def send_slack_alert(message):
    """Slack에 메시지를 보내는 함수"""
    # ⚠️ 실제 Slack Webhook URL은 절대 코드에 노출하면 안 됩니다.
    #    환경 변수(Environment Variable)에서 불러와야 합니다.
    #    (예: export SLACK_WEBHOOK_URL="your_real_url")
    webhook_url = os.environ.get("SLACK_WEBHOOK_URL")

    if not webhook_url:
        print("❌ Slack 알림 전송 실패: SLACK_WEBHOOK_URL 환경 변수가 설정되지 않았습니다.")
        return

    headers = {'Content-type': 'application/json'}
    payload = {'text': message}
    try:
        response = requests.post(webhook_url, headers=headers, data=json.dumps(payload))
        response.raise_for_status()
        print("✅ Slack 알림 전송 성공!")
    except requests.RequestException as e:
        print(f"❌ Slack 알림 전송 실패: {e}")

# --- 3. 메인 워크플로우 실행 ---
if not suspicious_ips.empty:
    print("🚨 대량 로그인 실패 탐지! 자동 분석 및 알림 시작...")
    for ip, count in suspicious_ips.items():
        
        # IP 정보 조회
        ip_info = get_ip_info(ip)
        country = ip_info.get('country', 'N/A') if ip_info else 'N/A'
        
        # Slack 메시지 생성
        alert_message = f"""
        🚨 *보안 경고: 대량 로그인 실패 탐지* 🚨
        --------------------------------------
        - *공격 의심 IP*: `{ip}`
        - *국가*: {country}
        - *실패 횟수*: *{count}회*
        - *탐지 시간*: {pd.Timestamp.now()}
        --------------------------------------
        분석팀의 즉각적인 확인이 필요합니다.
        """
        
        # Slack 알림 전송
        send_slack_alert(alert_message)
else:
    print("✅ 분석 결과: 특이사항 없음.")

```

**[시나리오 분석]** 이 스크립트가 서버에서 10분마다 실행되도록 설정(e.g., Cron job)해 놓으면, '쉴더' 분석가는 잠을 자는 동안에도 시스템이 자동으로 위협을 탐지하고 팀에 공유하는 **24/7 보안 관제 시스템**의 첫 단계를 완성하게 됩니다. 이것이 바로 API를 활용한 자동화의 힘입니다.

## 📌 요약 정리 (Executive Summary)

1. **API의 역할**: 나의 분석 스크립트와 외부 시스템(e.g., VirusTotal, Slack)을 연결하는 '통로' 역할을 하며, **분석-공유-대응**의 워크플로우를 자동화하는 핵심 기술이다.
    
2. **`requests` 라이브러리**: 파이썬에서 HTTP API를 호출하기 위한 표준 라이브러리이다. `requests.get()`, `requests.post()` 등의 함수를 사용한다.
    
3. **안전한 API 호출 3원칙**:
    
    - **성공 여부 확인**: `response.raise_for_status()`로 항상 HTTP 상태 코드를 먼저 확인한다.
        
    - **예외 처리**: `try...except` 구문으로 네트워크, 서버 등에서 발생할 수 있는 모든 예외에 대비한다.
        
    - **비밀 관리**: API 키, Webhook URL 등 민감 정보는 절대 코드에 하드코딩하지 않고 환경 변수 등을 사용한다.
        
4. **보안 분야 활용**:
    
    - **정보 보강(Enrichment)**: VirusTotal, Shodan 등의 API를 호출하여 분석 중인 IP, 도메인, 파일 해시에 대한 외부 평판 정보를 자동으로 가져와 분석의 정확도를 높인다.
        
    - **자동화(Automation)**: 탐지된 위협 정보를 Slack, Jira 등의 협업 도구 API와 연동하여 팀에 실시간으로 공유하고 대응 프로세스를 자동화한다.
        
5. **API Rate Limit**: 대부분의 API는 과도한 사용을 막기 위해 시간당/분당 요청 횟수 제한이 있다. `time.sleep()` 등을 사용하여 이를 반드시 준수해야 한다.
