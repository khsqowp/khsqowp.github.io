# 🐼 2. 데이터 핸들링 (Pandas): 보안 데이터 분석의 스위스 아미 나이프

## 🔄 이전 단원 복습: 보안 관점에서의 파이썬 기초

이전 단원 `1. 파이썬 기초 문법.md`에서 우리는 파이썬의 기본 요소들이 단순한 문법이 아니라, 각 라인이 잠재적인 공격 벡터(Attack Vector) 또는 방어 지점이 될 수 있음을 배웠습니다.

> **핵심 복습:**
>
> *   **모든 입력은 오염되었다 (Tainted Input):** `input()`으로 받은 데이터는 물론, 파일이나 네트워크로부터 읽어 들인 모든 데이터는 검증 없이는 신뢰할 수 없습니다.
> *   **자료형의 중요성:** 가변(Mutable) 자료형의 잘못된 사용은 '권한 상승'과 같은 심각한 논리적 결함을 야기할 수 있습니다.
> *   **민감정보 로깅:** `print()` 함수의 무분별한 사용은 시스템 로그에 API 키나 비밀번호와 같은 PII(개인 식별 정보)를 노출시켜 2차, 3차 공격의 빌미를 제공합니다.
>
> 이 원칙들은 오늘 배울 Pandas에서 더욱 중요해집니다. 수백만 줄의 데이터를 다루는 Pandas에서 하나의 잘못된 데이터 타입이나 검증되지 않은 데이터는 전체 분석 결과를 왜곡하고, 잘못된 보안 의사결정을 내리게 할 수 있습니다.

## 🤔 Pandas를 왜 배워야 할까요? (The "Why")

보안 분석가는 매일같이 엄청난 양의 데이터와 싸웁니다. 방화벽, 웹서버, 안티바이러스 등 각종 보안 장비는 하루에도 수십 기가바이트(GB)의 로그를 쏟아냅니다. 이 로그들은 대부분 비정형 텍스트 파일 형태로, 이 속에서 공격의 흔적(IOC, Indicator of Compromise)을 찾는 것은 모래사장에서 바늘을 찾는 것과 같습니다.

**[Pandas 이전의 문제점]**
*   **수작업 분석:** `grep`, `awk` 같은 리눅스 명령어나 메모장으로 로그를 분석하는 것은 수백만 줄의 데이터 앞에서는 불가능에 가깝습니다.
*   **느린 처리 속도:** 순수 파이썬의 `for` 루프로 대용량 파일을 한 줄씩 읽어 처리하는 것은 매우 비효율적이며 많은 시간이 소요됩니다.
*   **복잡한 데이터 연산:** 특정 IP의 시간대별 접속 횟수 집계, 여러 로그 파일의 데이터를 IP 기준으로 병합하는 등의 복잡한 연산은 구현하기가 매우 까다롭습니다.

**[Pandas의 해결책]**
Pandas는 이러한 문제들을 해결하기 위해 등장한, 파이썬을 위한 **고성능 데이터 분석 라이브러리**입니다. Pandas는 `DataFrame`이라는, 엑셀 시트와 유사한 2차원 테이블 형태의 강력한 데이터 구조를 제공합니다.

> **보안 전문가에게 Pandas란?**
>
> 흩어져 있는 수많은 증거(로그)들을 하나의 작업 테이블(`DataFrame`) 위에 올려놓고, 강력한 도구들(함수)을 이용해 데이터를 정제(Cleaning), 변형(Transforming), 분석(Analyzing)하여 최종적으로 범인(공격자)의 동선을 밝혀내는 **디지털 포렌식(Digital Forensics) 수사관의 만능 도구**와 같습니다.

## 1. 🎯 Pandas의 핵심: DataFrame과 Series

### 1.1. 🐼 DataFrame과 Series의 구조

*   **Series:** 1차원 배열과 같은 데이터 구조입니다. 하나의 열(Column)에 해당합니다.
*   **DataFrame:** 2차원 테이블 형태의 데이터 구조입니다. 여러 개의 Series가 모여 하나의 DataFrame을 이룹니다. 각 열은 서로 다른 데이터 타입을 가질 수 있습니다.

```python
import pandas as pd

# 딕셔너리를 사용하여 DataFrame 생성
# 현업에서는 보통 CSV, JSON, DB 등에서 데이터를 불러옵니다.
data = {
    'timestamp': ['2023-11-12 10:00:00', '2023-11-12 10:01:00', '2023-11-12 10:01:15'],
    'source_ip': ['192.168.1.10', '10.0.0.5', '192.168.1.10'],
    'destination_port': [443, 80, 22],
    'action': ['ALLOW', 'DENY', 'ALLOW']
}
firewall_logs_df = pd.DataFrame(data)

print("--- 🛡️ 방화벽 로그 DataFrame ---")
print(firewall_logs_df)
print("\n--- 🔬 'source_ip' Series ---")
print(firewall_logs_df['source_ip'])
```

### 1.2. 🧹 데이터 로딩과 정제 (Data Loading & Hygiene)

보안 데이터는 대부분 지저분합니다(Messy). 값이 누락되거나, 형식이 제멋대로인 경우가 많습니다. 분석에 앞서 데이터를 깨끗하게 만드는 **데이터 위생(Data Hygiene)** 작업이 필수적입니다.

> **용어 해설: ETL (Extract, Transform, Load)**
>
> 데이터를 다양한 소스에서 추출(Extract)하고, 분석하기 좋은 형태로 변환(Transform)하며, 최종 목적지(예: 데이터베이스, 데이터 웨어하우스)에 적재(Load)하는 일련의 과정을 의미합니다. 데이터 정제는 'Transform' 단계의 핵심입니다.

#### **PoC: 오염된 로그 파일 정제하기**

다음과 같이 값이 누락되고(`N/A`), 타임스탬프 형식이 섞여있는 로그 파일을 다뤄보겠습니다.

**`dirty_log.csv` 파일 내용:**
```csv
timestamp,source_ip,bytes_sent
2023/11/12 10:00:00,1.1.1.1,1024
2023-11-12T10:01:02,2.2.2.2,N/A
2023/11/12 10:02:10,1.1.1.1,2048
```

```python
import pandas as pd
import io # 가상 파일을 만들기 위해 import

# 가상 CSV 파일 생성
csv_content = """timestamp,source_ip,bytes_sent
2023/11/12 10:00:00,1.1.1.1,1024
2023-11-12T10:01:02,2.2.2.2,N/A
2023/11/12 10:02:10,1.1.1.1,2048
"""

# 🛡️ 데이터 로딩: 이 순간부터 모든 데이터는 '오염된 데이터'로 간주합니다.
# na_values='N/A' : 'N/A' 문자열을 결측치(NaN)로 인식하도록 지정
df = pd.read_csv(io.StringIO(csv_content), na_values='N/A')
print("--- 🧹 1. 원본 데이터 ---")
print(df)
print("\n--- 데이터 정보 (dtypes) ---")
print(df.info())

# --- 데이터 정제 (Data Hygiene) ---

# 1. 결측치(NaN) 처리: bytes_sent 열의 결측치를 0으로 채움
#    보안 관점: 결측치를 무작정 삭제하면 중요한 증거가 사라질 수 있으므로, 의미있는 값(e.g., 0)으로 채우는 것이 나을 수 있음.
df['bytes_sent'] = df['bytes_sent'].fillna(0)
print("\n--- 🧹 2. 결측치 처리 후 ---")
print(df)

# 2. 데이터 타입 변환: 
#    - bytes_sent를 정수형(int)으로 변환
#    - timestamp를 datetime 객체로 변환하여 시간 기반 분석이 가능하도록 함
df['bytes_sent'] = df['bytes_sent'].astype(int)
#    to_datetime은 다양한 형식의 날짜/시간 문자열을 자동으로 파싱하는 강력한 기능
df['timestamp'] = pd.to_datetime(df['timestamp'])
print("\n--- 🧹 3. 데이터 타입 변환 후 ---")
print(df)
print("\n--- 최종 데이터 정보 (dtypes) ---")
print(df.info())
```
**[분석]**
`info()` 결과를 보면, `bytes_sent`가 `float64`에서 `int64`로, `timestamp`가 `object`(문자열)에서 `datetime64[ns]`로 변경된 것을 확인할 수 있습니다. 이제 이 `DataFrame`은 시간대별 분석이나 수치 연산이 가능한, 깨끗한 상태가 되었습니다.

## 2. 🎯 데이터 분석 및 필터링: 공격의 흔적 찾기

깨끗해진 데이터를 바탕으로, 이제 본격적으로 공격의 흔적을 찾아보겠습니다. Pandas의 필터링(Filtering)과 그룹화(Grouping) 기능은 이 과정의 핵심입니다.

### 2.1. 🔬 필터링(Filtering): 원하는 데이터만 골라내기

`DataFrame`에서 특정 조건을 만족하는 행(row)들만 추출하는 기능입니다.

| 연산자 | 의미 | 예시 |
| :--- | :--- | :--- |
| `==` | 같음 | `df['status_code'] == 404` |
| `!=` | 다름 | `df['source_ip'] != '127.0.0.1'` |
| `&` | 그리고 (AND) | `(df['port'] == 80) & (df['action'] == 'DENY')` |
| `|` | 또는 (OR) | `(df['user'] == 'admin') | (df['user'] == 'root')` |
| `.isin()` | 목록에 포함 | `df['attack_type'].isin(['SQLi', 'XSS'])` |

#### **PoC: 대량 로그인 실패(Brute-force Attack) 시도 IP 탐지**

```python
import pandas as pd
import io

# 가상 인증 로그 생성
log_content = """timestamp,source_ip,user,status
2023-11-12 11:00:01,1.2.3.4,admin,SUCCESS
2023-11-12 11:00:05,5.6.7.8,rookies,FAILURE
2023-11-12 11:00:06,5.6.7.8,rookies,FAILURE
2023-11-12 11:00:07,5.6.7.8,rookies,FAILURE
2023-11-12 11:00:08,5.6.7.8,admin,FAILURE
2023-11-12 11:00:09,5.6.7.8,root,FAILURE
2023-11-12 11:01:20,1.2.3.4,guest,SUCCESS
"""
auth_df = pd.read_csv(io.StringIO(log_content))
auth_df['timestamp'] = pd.to_datetime(auth_df['timestamp'])

# 1. 로그인 실패(FAILURE) 로그만 필터링
failed_logs = auth_df[auth_df['status'] == 'FAILURE']
print("--- 💥 로그인 실패 로그 ---")
print(failed_logs)

# 2. 실패 로그들을 IP 주소 기준으로 그룹화하고, 각 IP의 실패 횟수 계산
#    value_counts()는 Series에 포함된 각 값의 개수를 세어주는 매우 유용한 함수
ip_failure_counts = failed_logs['source_ip'].value_counts()
print("\n---  IP별 로그인 실패 횟수 ---")
print(ip_failure_counts)

# 3. 특정 횟수(예: 3회) 이상 실패한 IP를 '공격 의심 IP'로 식별
suspicious_ips = ip_failure_counts[ip_failure_counts >= 3]
print("\n--- 🚨 공격 의심 IP (3회 이상 실패) ---")
print(suspicious_ips)
```
**[분석]**
단 몇 줄의 코드로, 수많은 로그 속에서 IP `5.6.7.8`이 5번이나 로그인을 실패하여 '무차별 대입 공격(Brute-force Attack)'을 시도했을 가능성이 높다는 **의미있는 정보(Intelligence)**를 찾아냈습니다.

## 👨‍💻 현직자 통합 시나리오: 침해사고 분석가의 악성 C2 서버 통신 분석

**[상황]**
침해사고 대응팀(CERT) 분석가 '쉴더'는 내부망의 한 PC가 악성코드에 감염되어 외부의 C2(Command & Control) 서버와 통신하고 있다는 경보를 받았습니다. '쉴더'는 해당 PC의 네트워크 트래픽 로그와 외부 위협 인텔리전스(Threat Intelligence) 정보를 결합하여 악성 통신을 식별해야 합니다.

**[데이터]**
1.  `traffic.csv`: 감염된 PC의 대량 네트워크 로그
2.  `threat_intel.csv`: 알려진 악성 IP 주소 목록 (외부 제공)

```python
import pandas as pd
import io

# --- 1. 데이터 준비 ---
# 가상 PC 트래픽 로그 (1GB짜리 파일이라고 상상)
pc_traffic_log = """timestamp,destination_ip,destination_port,bytes_sent
2023-11-12 12:00:01,8.8.8.8,53,64
2023-11-12 12:00:05,104.21.5.194,443,10240
2023-11-12 12:00:06,45.137.21.87,6667,128
2023-11-12 12:01:00,8.8.8.8,53,64
2023-11-12 12:01:10,45.137.21.87,6667,128
2023-11-12 12:01:15,172.217.25.4,443,20480
2023-11-12 12:02:30,45.137.21.87,6667,128
"""
# 가상 위협 인텔리전스 (알려진 악성 IP 목록)
threat_intel_feed = """ip,threat_type,country
45.137.21.87,C2_SERVER,Russia
203.0.113.10,SPAM_BOT,China
"""

traffic_df = pd.read_csv(io.StringIO(pc_traffic_log))
threat_df = pd.read_csv(io.StringIO(threat_intel_feed))

print("--- 🖥️ PC 트래픽 로그 ---")
print(traffic_df.head())
print("\n--- 😈 위협 인텔리전스 ---")
print(threat_df.head())

# --- 2. 데이터 병합 (Merge) 및 분석 ---
# 'destination_ip'와 'ip'를 기준으로 두 DataFrame을 병합(merge)
# how='inner': 두 DataFrame에 모두 존재하는 IP만 결과에 포함
# 💡 이것이 바로 데이터를 '보강(Enrichment)'하는 과정입니다.
merged_df = pd.merge(traffic_df, threat_df, left_on='destination_ip', right_on='ip', how='inner')

print("\n" + "="*40)
print("🚨🚨🚨 악성 C2 서버와의 통신 탐지! 🚨🚨🚨")
print("="*40)
print(merged_df)

# --- 3. 보고서 작성 ---
if not merged_df.empty:
    # 탐지된 악성 IP
    malicious_ip = merged_df['destination_ip'].iloc[0]
    # 관련 정보
    c2_info = merged_df.iloc[0]

    print("\n--- 📄 침해사고 분석 보고서 (요약) ---")
    print(f"  - 감염 PC: (내부 IP 주소)")
    print(f"  - 🔴 악성 C2 서버 IP: {c2_info['destination_ip']}")
    print(f"  - 🌍 국가: {c2_info['country']}")
    print(f"  - 💥 위협 유형: {c2_info['threat_type']}")
    print(f"  - 📡 통신 포트: {c2_info['destination_port']}")
    print(f"  - 횟수: {len(merged_df)} 회")
    print("\n  - 조치 권고: 방화벽에서 해당 IP 즉시 차단, 감염 PC 네트워크 격리 필요.")
    print("--- 보고서 종료 ---")
```
**[시나리오 분석]**
'쉴더' 분석가는 Pandas를 사용하여 두 개의 서로 다른 데이터 소스를 결합하고, 단 몇 초 만에 PC가 러시아에 위치한 C2 서버 IP(`45.137.21.87`)와 6667번 포트로 3차례 통신한 사실을 **데이터에 기반하여** 명확히 밝혀냈습니다. 이는 수작업으로는 몇 시간이 걸릴 수도 있는 분석 작업입니다.

## ➡️ 다음 단원에서는?

이번 단원에서는 Pandas를 이용해 흩어진 텍스트 로그를 의미있는 정보로 바꾸는 방법을 배웠습니다. 데이터 정제, 필터링, 그룹화, 병합은 모든 보안 데이터 분석의 기초 체력입니다.

**다음 단원인 `3. 수치 연산 (Numpy).md`** 에서는, 오늘 우리가 정제한 이 데이터를 AI 모델이 이해할 수 있는 언어, 즉 **숫자의 배열(Array)**로 바꾸는 방법을 배웁니다. Numpy는 파이썬에서 과학적 계산과 수치 연산을 위한 핵심 라이브러리로, Pandas의 DataFrame을 머신러닝 모델의 입력값인 행렬(Matrix)로 변환하는 데 필수적인 역할을 합니다. Pandas가 데이터의 '의미'를 다룬다면, Numpy는 데이터의 '수치'를 다룹니다.

## 📌 요약 정리 (Executive Summary)

1.  **Pandas의 역할**: 대용량의 비정형 보안 데이터를 `DataFrame`이라는 구조화된 형태로 변환하여, 빠르고 효율적인 분석을 가능하게 하는 데이터 분석의 핵심 도구이다.
2.  **데이터 위생 (Data Hygiene)**: 분석의 정확성을 위해, 로딩된 데이터의 결측치(`NaN`)를 처리하고 각 열의 데이터 타입(`dtype`)을 올바르게 변환하는 과정이 선행되어야 한다.
3.  **핵심 분석 기술**:
    *   **필터링**: `[]`와 `&`, `|`, `.isin()` 등의 연산자를 사용하여 원하는 조건의 데이터(e.g., 로그인 실패 로그)를 신속하게 추출할 수 있다.
    *   **그룹화 (`.groupby()`, `.value_counts()`)**: 특정 기준(e.g., IP 주소)으로 데이터를 묶고 집계하여, 공격 시도 횟수 등을 파악하고 이상 징후를 탐지할 수 있다.
    *   **병합 (`.merge()`)**: 서로 다른 데이터 소스(e.g., 내부 로그와 외부 위협 인텔리전스)를 결합하여 데이터를 보강하고, 분석의 깊이를 더할 수 있다.
4.  **보안 분석 워크플로우**: Pandas를 활용한 보안 분석은 **[데이터 로딩 -> 정제 -> 분석(필터링/그룹화) -> 보강(병합) -> 보고]** 의 흐름으로 이루어진다.
5.  **보안 관점**: Pandas로 다루는 모든 데이터는 잠재적으로 오염되었거나 민감정보를 포함할 수 있음을 인지하고, 분석 결과물을 외부에 공유할 때는 항상 익명화/마스킹을 고려해야 한다.
