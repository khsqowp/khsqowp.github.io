# 💥 3. 네트워크 공격과 방어: AI 기반 탐지 및 대응 전략

## 🔄 이전 단원 복습: 주요 프로토콜 분석과 취약점

이전 단원 `2. 주요 프로토콜 분석.md`에서 우리는 TCP, UDP, ICMP, ARP, HTTP, DNS 등 주요 네트워크 프로토콜의 상세한 작동 원리와 각 프로토콜의 취약점을 이용한 공격 기법을 학습했습니다. 또한 AI 모델이 이러한 공격을 탐지하기 위해 어떤 특징을 학습해야 하는지에 대해서도 구체적으로 살펴보았습니다.

> **핵심 복습:**
> 
> *   **TCP/UDP:** SYN Flooding, UDP Flooding 등 서비스 거부(DoS) 공격의 대상이 될 수 있으며, AI는 패킷 빈도, 플래그 비율 등으로 탐지 가능.
> *   **IP/ICMP/ARP:** IP 스푸핑, ICMP Flooding, ARP 스푸핑 등 네트워크 계층 공격의 핵심이며, AI는 비대칭 라우팅, MAC 주소 변경 등으로 탐지 가능.
> *   **HTTP/DNS:** 웹 공격(SQLi, XSS), DNS 증폭 공격 등 응용 계층 공격의 주요 통로이며, AI는 요청 패턴, 도메인 정보 등으로 탐지 가능.
> *   **AI 탐지 특징:** 각 프로토콜의 공격 유형별로 AI 모델이 학습해야 할 특징(패킷 빈도, 플래그 비율, 페이로드 패턴, IP/MAC 주소 변경 등)을 구체적으로 이해하는 것이 중요.
> 
> 이제 우리는 네트워크의 각 계층과 프로토콜에서 어떤 공격이 발생할 수 있는지에 대한 깊이 있는 이해를 갖추었습니다. 오늘 배울 '네트워크 공격과 방어' 단원에서는 이러한 공격들을 더욱 심층적으로 다루고, 실제 공격 시나리오를 바탕으로 **구체적인 방어 전략**과 **AI 기반 탐지 및 대응 기법**을 학습하게 될 것입니다.

## 🤔 네트워크 공격과 방어를 왜 배워야 할까요? (The "Why")

네트워크는 모든 사이버 공격의 통로이자 전장입니다. 공격자는 끊임없이 새로운 기법을 개발하거나 기존 기법을 변형하여 네트워크 방어 체계를 우회하려 합니다. 보안 전문가는 이러한 공격의 메커니즘을 정확히 이해하고, 이에 대응하는 방어 기술을 숙지해야 합니다.

**[네트워크 공격/방어 지식의 중요성]**
*   **능동적인 방어:** 공격자의 전술을 알면, 단순히 규칙에 의존하는 것이 아니라 선제적으로 방어 체계를 강화하고 취약점을 보완할 수 있습니다.
*   **효과적인 탐지:** 공격의 특징을 정확히 이해해야만, AI 모델이 학습할 유의미한 특징을 추출하고, 오탐/미탐을 줄이는 고성능 탐지 모델을 구축할 수 있습니다.
*   **신속한 대응:** 침해 사고 발생 시, 어떤 공격이 어떤 프로토콜을 통해 이루어졌는지 빠르게 파악하여 피해를 최소화하고 복구 시간을 단축할 수 있습니다.
*   **AI 모델의 한계 보완:** AI 모델은 패턴을 학습하지만, 공격자의 의도나 새로운 공격 기법의 원리를 직접 이해하지 못합니다. 보안 전문가는 AI의 탐지 결과를 해석하고, AI가 놓칠 수 있는 부분을 보완하여 최종적인 판단을 내려야 합니다.

> **보안 전문가에게 네트워크 공격과 방어란?**
> 
> 공격자의 **'전술 교본'**을 읽고, 이에 대응하는 **'방어 교본'**을 작성하는 것입니다. AI는 강력한 도구이지만, 그 도구를 어떻게 활용하여 실제 위협에 대응할지는 결국 보안 전문가의 깊이 있는 지식과 경험에 달려 있습니다. 이 단원은 AI 기반 보안 시스템을 **'지휘'**하는 데 필요한 핵심 역량을 제공합니다.

## 1. 💥 서비스 거부 공격 (DoS/DDoS): 네트워크 마비와 AI 기반 방어

서비스 거부(Denial of Service, DoS) 공격은 시스템의 자원을 고갈시키거나 네트워크 대역폭을 마비시켜 정상적인 서비스 제공을 방해하는 공격입니다. 분산 서비스 거부(DDoS, Distributed DoS) 공격은 여러 대의 좀비 PC(Botnet)를 이용하여 공격 규모를 증폭시킵니다.

### 1.1. 💥 SYN Flooding 공격 (TCP 기반)

이전 단원에서 다룬 SYN Flooding은 TCP 3-Way Handshake의 취약점을 이용합니다. 공격자는 대량의 SYN 패킷을 보내고 ACK를 보내지 않아 서버의 연결 대기 큐를 고갈시킵니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |
| **탐지** | 📈 특정 IP/포트의 SYN 패킷 비율 급증 <br> 📉 SYN-ACK 대비 ACK 패킷 비율 현저히 낮음 <br> 🌐 출발지 IP의 다양성 (스푸핑 여부) <br> ⏱️ 연결 설정 시간 초과율 증가 <br> 📊 비정상적인 TCP 플래그 조합 | **방화벽/IPS:** 비정상적인 SYN 패킷 비율 탐지 및 차단 <br> **SYN Cookie:** 서버가 SYN-ACK에 쿠키를 포함하여 ACK 수신 시에만 연결 설정 <br> **연결 타임아웃 조정:** Half-open 연결의 타임아웃 시간 단축 <br> **DDoS 방어 서비스:** 클라우드 기반 DDoS 방어 서비스 활용 <br> **Rate Limiting:** 특정 IP/포트에서 들어오는 SYN 패킷 속도 제한 |
| **AI 모델** | **지도 학습:** SYN Flooding 패턴을 레이블링하여 분류 모델 훈련 <br> **비지도 학습:** 정상 트래픽 패턴 학습 후 SYN Flooding을 이상치로 탐지 <br> **시계열 분석 (LSTM):** SYN 패킷 유입량의 비정상적인 시계열 패턴 변화 탐지 |

#### PoC: `hping3`를 이용한 SYN Flooding 공격 (개념)

```bash
# ⚠️ 경고: 이 명령어는 실제 공격을 시뮬레이션하며, 대상 시스템에 DoS를 유발할 수 있습니다.
#          실제 네트워크에 실행하면 법적 문제가 발생할 수 있습니다.
#          오직 교육 목적으로만 사용하며, 허가된 환경에서만 테스트해야 합니다.

# hping3 -S [대상 IP] -p [대상 포트] --flood
# -S: SYN 플래그 설정
# -p: 대상 포트
# --flood: 가능한 한 빠르게 패킷 전송 (Flooding)

# 예시: 대상 IP 192.168.1.100의 80번 포트에 SYN Flooding
# hping3 -S 192.168.1.100 -p 80 --flood
```

### 1.2. 💥 UDP Flooding 공격 (UDP 기반)

공격자가 대상 서버에 대량의 UDP 패킷을 보내 서버의 대역폭을 고갈시키거나, 서버가 응답 패킷을 처리하느라 바빠지게 만들어 서비스 거부(DoS)를 유발합니다. 특히 DNS, NTP, SNMP와 같은 UDP 기반 서비스를 악용하는 증폭 공격(Amplification Attack)에 사용됩니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |
| **탐지** | 📈 특정 IP/포트의 UDP 패킷 비율 급증 <br> 📦 UDP 페이로드 크기의 비정상성 (증폭 공격 징후) <br> 🌐 출발지 IP의 다양성 (스푸핑 여부) <br> 📊 비정상적인 UDP 포트 사용 | **방화벽/IPS:** 비정상적인 UDP 트래픽 비율 탐지 및 차단 <br> **Rate Limiting:** 특정 IP/포트에서 발생하는 UDP 트래픽 속도 제한 <br> **DDoS 방어 서비스:** 클라우드 기반 DDoS 방어 서비스 활용 <br> **서비스 설정 강화:** DNS 서버의 재귀 쿼리 제한, NTP/SNMP 서비스의 접근 제어 강화 |
| **AI 모델** | **지도 학습:** UDP Flooding 패턴을 레이블링하여 분류 모델 훈련 <br> **비지도 학습:** 정상 트래픽 패턴 학습 후 UDP Flooding을 이상치로 탐지 |

### 1.3. 💥 DNS 증폭 공격 (DNS 기반)

공격자가 위조된 출발지 IP(공격 대상의 IP)로 DNS 서버에 짧은 쿼리(예: `ANY` 타입 쿼리)를 보내면, DNS 서버는 이 쿼리에 대한 매우 큰 응답을 공격 대상에게 보내게 됩니다. 이를 통해 공격 대상의 네트워크 대역폭을 고갈시키는 DDoS 공격입니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |
| **탐지** | 📈 특정 IP로 유입되는 DNS 응답 패킷의 비정상적인 증가 <br> 📦 DNS 응답 패킷의 크기가 비정상적으로 큰 경우 <br> 🌐 출발지 IP의 다양성 (위조된 출발지 IP) <br> 📊 `ANY` 쿼리 등 비정상적인 DNS 쿼리 타입 증가 | **DNS 서버 설정:** 재귀 쿼리(Recursive Query)를 허용하는 IP 제한 <br> **Rate Limiting:** DNS 쿼리/응답 속도 제한 <br> **DDoS 방어 서비스:** 클라우드 기반 DDoS 방어 서비스 활용 <br> **DNSSEC:** DNS 응답의 무결성 검증 |
| **AI 모델** | **지도 학습:** DNS 증폭 공격 패턴을 레이블링하여 분류 모델 훈련 <br> **비지도 학습:** 정상 DNS 트래픽 패턴 학습 후 증폭 공격을 이상치로 탐지 |

### 1.4. 💥 HTTP Flooding / Slowloris 공격 (응용 계층 DoS)

*   **HTTP Flooding:** 웹 서버에 대량의 HTTP 요청을 보내 서버의 자원(CPU, 메모리, 대역폭)을 고갈시키는 공격입니다. 정상적인 요청처럼 보이기 때문에 탐지가 어렵습니다.
*   **Slowloris:** 웹 서버에 불완전한 HTTP 요청 헤더를 보내 연결을 오랫동안 유지시켜 웹 서버의 최대 동시 연결 수를 소진시키는 공격입니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |
| **탐지** | 📈 특정 IP로부터의 HTTP 요청 빈도 급증 <br> ⏱️ 비정상적으로 긴 HTTP 연결 유지 시간 (Slowloris) <br> 📊 HTTP 요청 헤더의 비정상성 (불완전한 헤더, 비정상적인 User-Agent) <br> 📦 요청 본문 크기의 비정상성 | **웹 방화벽 (WAF):** HTTP 요청 패턴 분석 및 비정상 요청 차단 <br> **로드 밸런서:** 트래픽 분산 및 비정상 트래픽 필터링 <br> **Rate Limiting:** 특정 IP 또는 세션으로부터의 HTTP 요청 속도 제한 <br> **연결 타임아웃 설정:** 웹 서버의 연결 타임아웃을 짧게 설정 |
| **AI 모델** | **지도 학습:** HTTP Flooding/Slowloris 패턴을 레이블링하여 분류 모델 훈련 <br> **비지도 학습:** 정상 HTTP 트래픽 패턴 학습 후 이상치로 탐지 <br> **시계열 분석 (LSTM):** HTTP 요청량의 비정상적인 시계열 패턴 변화 탐지 |

## 2. 💥 스푸핑 공격 (Spoofing Attack): 신분 위장과 AI 기반 방어

스푸핑(Spoofing) 공격은 공격자가 자신의 신분을 위장하여 다른 시스템이나 사용자인 것처럼 속이는 공격입니다.

### 2.1. 💥 ARP 스푸핑 (ARP 기반)

공격자가 자신의 MAC 주소를 다른 장치(예: 게이트웨이)의 MAC 주소인 것처럼 속여, 네트워크 트래픽을 가로채거나 변조하는 중간자 공격(Man-in-the-Middle Attack)입니다. 공격자는 위조된 ARP 응답을 네트워크에 지속적으로 전송하여 ARP 테이블을 오염시킵니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |
| **탐지** | 📈 특정 MAC 주소에서 비정상적으로 많은 ARP 응답 발생 <br> 🔄 특정 IP 주소에 대한 MAC 주소가 짧은 시간 내에 자주 변경 <br> ⚠️ 게이트웨이의 MAC 주소가 실제와 다른 ARP 응답 <br> 📊 ARP 응답 패킷의 SHA와 SPA 불일치 | **정적 ARP 테이블:** 중요한 서버나 게이트웨이의 ARP 테이블을 수동으로 설정하여 변경 방지 <br> **ARP 감시 도구:** ARP 트래픽을 모니터링하여 비정상적인 ARP 응답 탐지 (예: `arpwatch`) <br> **네트워크 접근 제어 (NAC):** 비인가 장치의 네트워크 접속 차단 <br> **스위치 보안:** 포트 보안(Port Security) 기능 활성화 |
| **AI 모델** | **지도 학습:** ARP 스푸핑 패턴을 레이블링하여 분류 모델 훈련 <br> **비지도 학습:** 정상 ARP 트래픽 패턴 학습 후 스푸핑을 이상치로 탐지 |

#### PoC: `arpspoof`를 이용한 ARP 스푸핑 공격 (개념)

```bash
# ⚠️ 경고: 이 명령어는 실제 공격을 시뮬레이션하며, 네트워크에 혼란을 줄 수 있습니다.
#          실제 네트워크에 실행하면 법적 문제가 발생할 수 있습니다.
#          오직 교육 목적으로만 사용하며, 허가된 환경에서만 테스트해야 합니다.

# arpspoof -i [인터페이스] -t [대상 IP] [게이트웨이 IP]
# -i: 공격에 사용할 네트워크 인터페이스 (예: eth0)
# -t: 스푸핑할 대상 IP (예: 피해자 PC)
# [게이트웨이 IP]: 게이트웨이의 IP 주소

# 예시: 피해자 PC(192.168.1.100)에게 게이트웨이(192.168.1.1)인 것처럼 속임
# arpspoof -i eth0 -t 192.168.1.100 192.168.1.1
```

### 2.2. 💥 IP 스푸핑 (IP 기반)

공격자가 자신의 IP 주소를 위조하여 다른 시스템인 것처럼 속이는 공격입니다. 방화벽이나 침입 탐지 시스템을 우회하거나, 신뢰할 수 있는 내부 IP인 것처럼 위장하여 공격을 수행할 수 있습니다. 주로 TCP 세션 하이재킹, DoS 공격, 인증 우회 등에 사용됩니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |
| **탐지** | ↔️ 출발지 IP와 목적지 IP 간의 통신 경로가 비대칭 <br> 🌐 내부망 IP 주소가 외부 네트워크에서 출발지 IP로 사용 <br> ⚠️ TCP 시퀀스 번호의 비정상성 <br> 📊 TTL 값의 비정상성 | **Ingress Filtering:** 외부에서 내부 IP를 출발지로 하는 패킷 차단 <br> **Egress Filtering:** 내부에서 외부 IP를 출발지로 하는 패킷 차단 <br> **TCP 시퀀스 번호 예측:** 비정상적인 시퀀스 번호 탐지 <br> **인증 강화:** IP 주소만으로 인증하지 않고, 강력한 인증 메커니즘 사용 |
| **AI 모델** | **지도 학습:** IP 스푸핑 패턴을 레이블링하여 분류 모델 훈련 <br> **비지도 학습:** 정상 IP 트래픽 패턴 학습 후 스푸핑을 이상치로 탐지 |

### 2.3. 💥 DNS 스푸핑 (DNS 기반)

공격자가 DNS 응답을 위조하여 사용자가 악성 웹사이트로 접속하도록 유도하는 공격입니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |
| **탐지** | 🔄 DNS 응답의 비정상적인 변경 (IP 주소 변경) <br> ⏱️ DNS 응답 시간의 비정상적인 지연 또는 급격한 단축 <br> 📊 DNS 응답 패킷의 ID 불일치 <br> 🌐 악성 도메인으로의 리다이렉션 | **DNSSEC:** DNS 응답의 무결성을 검증하여 위조된 응답 방지 <br> **DNS 캐시 서버 보안:** 캐시 포이즈닝 방지 <br> **클라이언트 측 검증:** 웹 브라우저의 HTTPS 인증서 검증 |
| **AI 모델** | **지도 학습:** DNS 스푸핑 패턴을 레이블링하여 분류 모델 훈련 <br> **비지도 학습:** 정상 DNS 응답 패턴 학습 후 스푸핑을 이상치로 탐지 |

### 2.4. 💥 이메일 스푸핑 (Email 기반)

공격자가 발신자 이메일 주소를 위조하여 신뢰할 수 있는 사람이나 기관인 것처럼 속여 피싱 메일을 보내는 공격입니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |

## 3. 💥 스니핑 공격 (Sniffing Attack): 도청과 AI 기반 방어

스니핑(Sniffing) 공격은 네트워크를 통해 전송되는 데이터를 몰래 가로채어 엿듣는 도청 공격입니다. 암호화되지 않은 평문 통신에서 주로 발생합니다.

#### **AI 기반 탐지 및 방어 전략**

| 구분 | AI 탐지 특징 | 방어 전략 |
| :--- | :--- | :--- |
| **탐지** | 📈 특정 네트워크 인터페이스의 비정상적인 트래픽 증가 <br> ⚠️ 암호화되지 않은 민감 정보(비밀번호, API 키)의 평문 전송 <br> 🔄 ARP 스푸핑과 동반 발생 (중간자 공격) <br> 📊 네트워크 인터페이스의 무차별 모드(Promiscuous Mode) 활성화 탐지 | **암호화 통신:** 모든 민감 정보는 SSL/TLS를 사용하여 암호화하여 전송 <br> **스위치 보안:** 포트 보안(Port Security) 설정, DHCP 스누핑 <br> **VPN 사용:** 공용 네트워크에서 안전한 통신 채널 확보 <br> **네트워크 세그멘테이션:** 네트워크를 분리하여 스니핑 범위 제한 |
| **AI 모델** | **지도 학습:** 스니핑 공격 패턴(예: ARP 스푸핑 후 평문 통신)을 레이블링하여 분류 모델 훈련 <br> **비지도 학습:** 정상 트래픽 대비 비정상적인 평문 통신 증가를 이상치로 탐지 |

## 4. 🛡️ 네트워크 방어의 핵심: 다층 방어 (Defense-in-Depth)

네트워크 공격은 단일 방어 메커니즘으로는 막기 어렵습니다. 여러 보안 솔루션과 기술을 계층적으로 배치하여 공격자가 하나의 방어선을 뚫더라도 다음 방어선에서 저지될 수 있도록 하는 **다층 방어(Defense-in-Depth)** 전략이 필수적입니다.

### 4.1. 💡 방화벽 (Firewall)

네트워크의 최전선에서 트래픽을 허용/차단하는 문지기 역할을 합니다. IP 주소, 포트, 프로토콜 등을 기반으로 규칙을 적용합니다.

### 4.2. 💡 침입 탐지/방지 시스템 (IDS/IPS)

네트워크 트래픽이나 시스템 로그를 분석하여 침입 행위를 탐지하고(IDS), 나아가 차단(IPS)까지 수행합니다. 시그니처 기반과 이상 행위 기반 탐지를 모두 사용합니다.

### 4.3. 💡 웹 방화벽 (WAF)

웹 애플리케이션에 특화된 방화벽으로, SQL Injection, XSS 등 웹 공격을 탐지하고 차단합니다.

### 4.4. 💡 VPN (Virtual Private Network)

공중망을 통해 사설망에 안전하게 접속할 수 있도록 암호화된 터널을 제공합니다.

### 4.5. 💡 네트워크 접근 제어 (NAC)

네트워크에 접속하는 장치들의 보안 정책 준수 여부를 검사하고, 정책에 따라 네트워크 접근을 제어합니다.

## 👨‍💻 현직자 통합 시나리오: AI 기반 네트워크 침입 방지 시스템 (NIPS)

**[상황]**
AI 보안 엔지니어 '제미니'는 내부 네트워크를 보호하기 위해 AI 기반의 네트워크 침입 방지 시스템(NIPS)을 구축하고자 합니다. 이 시스템은 SYN Flooding, ARP Spoofing과 같은 주요 네트워크 공격을 실시간으로 탐지하고, 자동으로 차단 조치를 취해야 합니다.

**[데이터]**
네트워크 패킷 데이터 (출발지/목적지 IP, 포트, 프로토콜, TCP 플래그, MAC 주소 등)

**[AI 모델]**
*   **SYN Flooding 탐지:** TCP SYN 패킷 비율, SYN-ACK 대비 ACK 비율, 출발지 IP 다양성 등을 특징으로 하는 지도 학습 분류 모델.
*   **ARP Spoofing 탐지:** ARP 응답 빈도, MAC 주소 변경 빈도, 게이트웨이 MAC 주소 불일치 등을 특징으로 하는 지도 학습 분류 모델.

```python
import pandas as pd
import numpy as np
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import classification_report, confusion_matrix
from sklearn.preprocessing import LabelEncoder
import io

# --- 1. 가상 네트워크 공격 데이터 생성 ---
# 실제 환경에서는 Wireshark 등으로 캡처한 트래픽에서 특징을 추출합니다.

# 정상 트래픽 데이터
normal_traffic = {
    'src_ip': ['192.168.1.10', '192.168.1.11'] * 50,
    'dst_ip': ['192.168.1.1', '8.8.8.8'] * 50,
    'src_port': np.random.randint(1024, 65535, 100),
    'dst_port': np.random.choice([80, 443, 53], 100),
    'protocol': ['TCP'] * 80 + ['UDP'] * 20,
    'syn_ratio': np.random.uniform(0.1, 0.3, 100), # SYN 패킷 비율
    'ack_ratio': np.random.uniform(0.7, 0.9, 100), # ACK 패킷 비율
    'arp_reply_count': np.random.randint(1, 5, 100), # ARP 응답 횟수
    'mac_change_freq': np.random.uniform(0, 0.01, 100), # MAC 변경 빈도
    'is_attack': [0] * 100
}
normal_df = pd.DataFrame(normal_traffic)

# SYN Flooding 공격 데이터
syn_flood_traffic = {
    'src_ip': ['203.0.113.10', '203.0.113.11'] * 5, # 외부 공격 IP
    'dst_ip': ['192.168.1.100'] * 10, # 내부 서버
    'src_port': np.random.randint(1024, 65535, 10),
    'dst_port': [80] * 10,
    'protocol': ['TCP'] * 10,
    'syn_ratio': np.random.uniform(0.9, 1.0, 10), # SYN 패킷 비율 매우 높음
    'ack_ratio': np.random.uniform(0.0, 0.1, 10), # ACK 패킷 비율 매우 낮음
    'arp_reply_count': np.random.randint(1, 3, 10),
    'mac_change_freq': np.random.uniform(0, 0.01, 10),
    'is_attack': [1] * 10
}
syn_flood_df = pd.DataFrame(syn_flood_traffic)

# ARP Spoofing 공격 데이터
arp_spoof_traffic = {
    'src_ip': ['192.168.1.10', '192.168.1.11'] * 5,
    'dst_ip': ['192.168.1.1', '192.168.1.100'] * 5,
    'src_port': np.random.randint(1024, 65535, 10),
    'dst_port': np.random.choice([80, 443, 53], 10),
    'protocol': ['TCP'] * 10,
    'syn_ratio': np.random.uniform(0.1, 0.3, 10),
    'ack_ratio': np.random.uniform(0.7, 0.9, 10),
    'arp_reply_count': np.random.randint(10, 50, 10), # ARP 응답 횟수 매우 높음
    'mac_change_freq': np.random.uniform(0.5, 1.0, 10), # MAC 변경 빈도 매우 높음
    'is_attack': [1] * 10
}
arp_spoof_df = pd.DataFrame(arp_spoof_traffic)

# 데이터 결합
df = pd.concat([normal_df, syn_flood_df, arp_spoof_df], ignore_index=True)

# 문자열 데이터를 숫자로 변환
for col in ['src_ip', 'dst_ip', 'protocol']:
    le = LabelEncoder()
    df[col] = le.fit_transform(df[col])

X = df.drop('is_attack', axis=1)
y = df['is_attack']

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

# --- 2. AI 모델 훈련 ---
model = RandomForestClassifier(n_estimators=100, random_state=42)
model.fit(X_train, y_train)

print("✅ AI 모델 훈련 완료!")

# --- 3. AI 모델 기반 탐지 및 자동 차단 (개념) ---
y_pred = model.predict(X_test)

print("\n--- 🚨 AI 기반 네트워크 공격 탐지 결과 ---")
for i, (actual, pred) in enumerate(zip(y_test, y_pred)):
    if pred == 1 and actual == 1:
        print(f"  - 🔴 공격 탐지! (실제 공격) - 테스트 데이터 {i}번")
        # 💡 실제 NIPS에서는 탐지된 공격에 대해 자동으로 차단 조치를 취합니다.
        #    예: 방화벽 API 호출, IPS 정책 업데이트 등
        # print(f"    -> 자동 차단 조치: {X_test.iloc[i]['src_ip']} 차단 (개념)")
    elif pred == 1 and actual == 0:
        print(f"  - ⚠️ 오탐! (실제 정상) - 테스트 데이터 {i}번")
    elif pred == 0 and actual == 1:
        print(f"  - ❌ 미탐! (실제 공격) - 테스트 데이터 {i}번")

print("\n--- 📊 모델 성능 보고서 ---")
print(classification_report(y_test, y_pred, target_names=['정상', '공격']))
print("\n--- 📈 혼동 행렬 ---")
print(confusion_matrix(y_test, y_pred))
```
**[시나리오 분석]**
'제미니' 엔지니어는 `RandomForestClassifier`를 사용하여 SYN Flooding과 ARP Spoofing과 같은 네트워크 공격을 탐지하는 AI 모델을 구축했습니다. 이 모델은 각 공격의 특징(SYN/ACK 비율, ARP 응답 빈도 등)을 학습하여 실시간으로 네트워크 트래픽을 분석하고, 공격이 탐지되면 자동으로 차단 조치(개념적)를 취하는 NIPS의 핵심 기능을 시연했습니다. `classification_report`와 `confusion_matrix`를 통해 모델의 성능을 평가하며, 특히 미탐(False Negative)을 최소화하는 것이 중요함을 다시 한번 강조했습니다.

## 📌 요약 정리 (Executive Summary)

1.  **네트워크 공격과 방어의 중요성**: 공격자의 전술을 이해하고 이에 대응하는 방어 기술을 숙지하는 것은 능동적인 방어, 효과적인 탐지, 신속한 대응, 그리고 AI 모델의 한계 보완을 위해 필수적이다.
2.  **서비스 거부 공격 (DoS/DDoS)**:
    *   **SYN Flooding**: TCP 3-Way Handshake 취약점 이용. AI는 SYN/ACK 비율, 패킷 빈도, 출발지 IP 다양성으로 탐지. `hping3` 등으로 공격 시뮬레이션.
    *   **UDP Flooding**: 대량의 UDP 패킷 전송. AI는 UDP 패킷 빈도, 페이로드 크기로 탐지.
    *   **DNS 증폭 공격**: 위조된 IP로 DNS 쿼리 증폭. AI는 DNS 응답 패킷 증가, 크기, 출발지 IP 다양성으로 탐지.
    *   **HTTP Flooding / Slowloris**: 웹 서버 자원 고갈. AI는 HTTP 요청 빈도, 연결 유지 시간, 헤더 비정상성으로 탐지.
3.  **스푸핑 공격 (Spoofing Attack)**:
    *   **ARP 스푸핑**: MAC 주소 위장. AI는 ARP 응답 빈도, MAC 주소 변경 빈도로 탐지. `arpspoof` 등으로 공격 시뮬레이션.
    *   **IP 스푸핑**: IP 주소 위장. AI는 비대칭 라우팅, 내부 IP의 외부 출발지 사용으로 탐지.
    *   **DNS 스푸핑**: DNS 응답 위조. AI는 DNS 응답 변경, 응답 시간 지연 등으로 탐지.
    *   **이메일 스푸핑**: 발신자 이메일 주소 위조. AI는 SPF/DKIM/DMARC 실패, 악성 URL 포함 등으로 탐지.
4.  **스니핑 공격 (Sniffing Attack)**: 네트워크 데이터 도청. AI는 비정상 트래픽 증가, 암호화되지 않은 민감 정보 전송, 무차별 모드 활성화 탐지로 탐지. `tcpdump` 등으로 방어 확인.
5.  **네트워크 방어의 핵심**: 다층 방어(Defense-in-Depth) 전략을 통해 여러 보안 솔루션(방화벽, IDS/IPS, WAF, VPN, NAC)을 계층적으로 배치하여 공격을 저지한다. `iptables` 등으로 방화벽 규칙 설정.
6.  **AI 기반 탐지 및 대응 전략**:
    *   **지도 학습**: 알려진 공격 패턴을 레이블링하여 분류 모델 훈련.
    *   **비지도 학습**: 정상 트래픽 패턴 학습 후 이상치를 공격으로 탐지.
    *   **자동 차단**: AI 탐지 결과를 바탕으로 방화벽/IPS API를 통해 자동으로 공격 IP 차단 등의 조치 수행.
7.  **보안 관점**: AI는 네트워크 공격 탐지에 강력한 도구이지만, AI의 탐지 결과를 해석하고, AI가 놓칠 수 있는 부분을 보완하여 최종적인 판단을 내리는 것은 보안 전문가의 역할이다. AI 기반 NIPS(Network Intrusion Prevention System)는 실시간 탐지 및 자동 차단을 통해 보안 운영의 효율성을 극대화한다.
