# 🤖 3. AI 기반 악성코드 분류: 지능형 위협 식별과 패밀리 분석

## 🔄 이전 단원 복습: 리버스 엔지니어링으로 악성코드 해부

이전 단원 `2. 리버스 엔지니어링.md`에서 우리는 악성코드 분석의 심화 과정인 리버스 엔지니어링의 기본 개념(어셈블리, 디스어셈블러, 디컴파일러), 난독화 및 패킹 해제 기법, 그리고 AI 기반으로 리버스 엔지니어링 과정을 지원하고 자동화하는 방법을 학습했습니다.

> **핵심 복습:**
>
> *   **리버스 엔지니어링의 목적:** 난독화된 악성코드의 실제 작동 원리, 숨겨진 기능, C2 서버 통신 방식 등을 파악하여 제로데이 취약점 및 익스플로잇 분석, 위협 인텔리전스 고도화.
> *   **주요 도구:** 디스어셈블러(IDA Pro, Ghidra)로 코드 흐름 분석, 디컴파일러로 고수준 로직 파악.
> *   **난독화/패킹 해제:** AI는 난독화 패턴 인식, 문자열 복원, 패커 종류 분류, 언패킹 힌트 제공 등으로 분석 지원.
> *   **AI 기반 지원:** AI는 바이너리 코드 분석, 난독화 해제/안티-분석 우회, 익스플로잇 생성 지원 등을 통해 리버스 엔지니어링 효율성 극대화.
>
> 리버스 엔지니어링을 통해 악성코드의 상세한 작동 원리를 파악할 수 있지만, 매일 쏟아지는 수많은 악성코드 샘플을 모두 수동으로 분석하는 것은 불가능합니다. 오늘 배울 'AI 기반 악성코드 분류' 단원에서는 AI를 활용하여 악성코드를 더욱 정교하게 분류하는 방법에 대해 학습합니다. 정적/동적 특징을 넘어, 딥러닝 기반의 이미지 분석(바이너리 시각화), 시퀀스 분석(API 호출 시퀀스), 그래프 분석(함수 호출 그래프) 등 고급 AI 기법을 사용하여 악성코드 패밀리를 분류하고, 변종 악성코드를 탐지하는 방법을 배우게 될 것입니다.

---

## 🤔 AI 기반 악성코드 분류를 왜 배워야 할까요? (The "Why")

매일 수십만 개의 새로운 악성코드 샘플이 발견되며, 기존 악성코드의 변종도 끊임없이 생성됩니다. 이러한 방대한 양의 악성코드를 수동으로 분석하고 분류하는 것은 불가능하며, 기존의 시그니처 기반 백신으로는 변종 악성코드를 탐지하는 데 한계가 있습니다. AI 기반 악성코드 분류는 이러한 문제를 해결하고, 악성코드 분석 및 대응의 효율성과 정확도를 혁신적으로 높입니다.

**[AI 기반 악성코드 분류 지식의 중요성]**
*   **신속한 위협 식별:** 새로운 악성코드 샘플이 발견되었을 때, AI가 자동으로 악성 여부를 판단하고 어떤 패밀리에 속하는지 분류하여 신속한 대응을 가능하게 합니다.
*   **변종 악성코드 탐지:** 기존 시그니처 기반 방식으로는 탐지하기 어려운 변종 악성코드를 AI가 학습된 패턴을 기반으로 탐지합니다.
*   **위협 인텔리전스 고도화:** 악성코드 패밀리별 특징과 행위를 파악하여 위협 인텔리전스를 더욱 풍부하게 구축하고, 공격자의 TTPs(전술, 기술, 절차)를 심층적으로 이해합니다.
*   **자동화된 대응:** AI가 악성코드를 분류하면, 해당 악성코드의 특성에 맞는 자동화된 대응(격리, 차단, 시그니처 업데이트)을 수행할 수 있습니다.

> **보안 전문가에게 AI 기반 악성코드 분류란?**
>
> 수많은 악성코드라는 **'미지의 생명체'**들을 **'DNA 분석(정적/동적 특징)'**을 통해 **'종(種)과 과(科)로 분류'**하고, 그들의 **'진화 과정(변종)'**까지 추적하는 **'지능형 생물학자'**를 고용하는 것과 같습니다. AI는 악성코드의 본질을 파악하고, 그들의 계보를 정리하여 **'악성코드 생태계 지도'**를 구축함으로써, 인간 분석가가 더욱 효과적으로 위협에 대응할 수 있도록 돕습니다.

## 1. 🤖 AI 기반 악성코드 분류의 특징: 정적 vs 동적 vs 하이브리드

AI 기반 악성코드 분류는 악성코드의 어떤 특징을 사용하는지에 따라 크게 세 가지로 나눌 수 있습니다.

### 1.1. 🔬 정적 특징 기반 분류

*   **설명:** 악성코드를 실행하지 않고 파일 자체에서 추출한 특징(파일 헤더, 섹션 정보, 임포트/익스포트 함수, 문자열, 바이트 시퀀스, 엔트로피 등)을 기반으로 분류합니다.
*   **장점:** 분석 속도가 빠르고, 악성코드가 분석 환경을 회피하려는 시도에 영향을 받지 않음.
*   **단점:** 난독화, 패킹, 암호화된 악성코드의 경우 실제 행위를 파악하기 어려워 분류 정확도가 떨어질 수 있음.
*   **AI 연동 전략:**
    *   **바이트 시퀀스 분석 (CNN):** 악성코드의 바이트 시퀀스를 이미지처럼 간주하여 CNN 모델로 특징을 추출하고 분류.
    *   **API 임포트/문자열 분석 (NLP):** 임포트 함수 목록이나 추출된 문자열을 텍스트 데이터로 간주하여 NLP 모델로 분류.
    *   **파일 메타데이터 분석 (머신러닝):** 파일 크기, 섹션 수, 엔트로피 등 수치형 메타데이터를 기반으로 분류.

### 1.2. 🔬 동적 특징 기반 분류

*   **설명:** 샌드박스 등 격리된 환경에서 악성코드를 실행하여 관찰된 행위(API 호출 시퀀스, 파일/레지스트리 변경, 네트워크 통신, 프로세스 생성 등)를 기반으로 분류합니다.
*   **장점:** 난독화, 패킹된 악성코드의 실제 행위를 파악할 수 있어 분류 정확도가 높음.
*   **단점:** 분석 시간이 오래 걸리고, 악성코드가 샌드박스 환경을 탐지하고 회피할 경우 분석이 어려움.
*   **AI 연동 전략:**
    *   **API 호출 시퀀스 분석 (RNN, LSTM):** 악성코드가 호출하는 API 함수의 순서를 시퀀스 데이터로 간주하여 RNN/LSTM 모델로 특징을 추출하고 분류.
    *   **행위 그래프 분석 (GNN):** 악성코드의 행위(파일 생성, 레지스트리 변경, 네트워크 통신) 간의 관계를 그래프로 표현하여 GNN 모델로 분류.
    *   **시스템 호출 통계 (머신러닝):** 특정 시스템 호출의 빈도, 종류 등을 통계적으로 분석하여 분류.

### 1.3. 🔬 하이브리드 특징 기반 분류

*   **설명:** 정적 특징과 동적 특징을 모두 활용하여 악성코드를 분류합니다. 각 방법의 장점을 결합하여 분류 정확도를 극대화합니다.
*   **장점:** 가장 높은 분류 정확도를 기대할 수 있음.
*   **단점:** 데이터 수집 및 전처리 과정이 복잡하고, 분석 시간이 길어질 수 있음.
*   **AI 연동 전략:**
    *   **앙상블 모델:** 정적 특징 기반 모델과 동적 특징 기반 모델의 예측 결과를 결합하여 최종 분류.
    *   **멀티모달 딥러닝:** 정적 특징(이미지, 텍스트)과 동적 특징(시퀀스, 그래프)을 동시에 입력받아 학습하는 딥러닝 모델.

## 2. 🤖 AI 기반 악성코드 분류의 고급 기법

### 2.1. 🖼️ 바이너리 시각화 (Binary Visualization) 및 CNN

악성코드의 바이너리 데이터를 이미지처럼 시각화하여 CNN(Convolutional Neural Network) 모델로 분류하는 기법입니다.

*   **원리:** 악성코드의 바이트 스트림을 픽셀 값에 매핑하여 흑백 또는 컬러 이미지로 변환합니다. 악성코드 패밀리마다 고유한 시각적 패턴을 가지므로, CNN이 이를 학습하여 분류합니다.
*   **AI 연동 전략:**
    *   **이미지 전처리:** 바이너리 데이터를 고정된 크기의 이미지로 변환.
    *   **CNN 모델 훈련:** 이미지 분류에 특화된 CNN 모델(ResNet, VGG 등)을 훈련하여 악성코드 패밀리 분류.

#### **PoC: 바이너리 시각화 기반 악성코드 분류 (개념)**

```python
import numpy as np
import tensorflow as tf
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Conv2D, MaxPooling2D, Flatten, Dense
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report, confusion_matrix
import os

# --- 0. 가상 바이너리 이미지 데이터 생성 ---
# 실제 악성코드 바이너리 이미지는 복잡하고 다양하므로,
# 여기서는 개념 이해를 돕기 위한 가상의 흑백 이미지(32x32 픽셀) 데이터셋을 생성합니다.
# 각 이미지는 악성코드 패밀리(0: 정상, 1: 랜섬웨어, 2: 스파이웨어)를 나타냅니다.
num_samples = 300
image_size = 32
num_classes = 3 # 정상, 랜섬웨어, 스파이웨어

# (샘플 수, 높이, 너비, 채널 수)
X_images = np.random.rand(num_samples, image_size, image_size, 1).astype(np.float32) 
y_labels = np.random.randint(num_classes, size=num_samples) 

# 특정 패밀리 패턴 주입 (단순 예시)
for i in range(num_samples // num_classes):
    # 랜섬웨어 패턴 (가상)
    X_images[i, 5:10, 5:10, 0] = 1.0 
    y_labels[i] = 1
    # 스파이웨어 패턴 (가상)
    X_images[i + num_samples // num_classes, 15:20, 15:20, 0] = 0.8
    y_labels[i + num_samples // num_classes] = 2

X_train, X_test, y_train, y_test = train_test_split(X_images, y_labels, test_size=0.2, random_state=42)

print(f"훈련 이미지 데이터 형태: {X_train.shape}")
print(f"훈련 라벨 데이터 형태: {y_train.shape}")

# --- 1. CNN 모델 구축 ---
model = Sequential([
    Conv2D(32, (3, 3), activation='relu', input_shape=(image_size, image_size, 1)),
    MaxPooling2D((2, 2)),
    Conv2D(64, (3, 3), activation='relu'),
    MaxPooling2D((2, 2)),
    Flatten(),
    Dense(128, activation='relu'),
    Dense(num_classes, activation='softmax') # 다중 클래스 분류이므로 softmax
])

model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
model.summary()

# --- 2. 모델 훈련 ---
history = model.fit(X_train, y_train, epochs=5, batch_size=32, validation_split=0.2, verbose=0)
print("\n✅ CNN 모델 훈련 완료!")

# --- 3. 모델 예측 및 평가 ---
y_pred_proba = model.predict(X_test)
y_pred = np.argmax(y_pred_proba, axis=1)

print("\n--- 📊 CNN 기반 악성코드 분류 모델 평가 결과 ---")
print(classification_report(y_test, y_pred, target_names=['정상', '랜섬웨어', '스파이웨어']))
print("\n--- 📈 혼동 행렬 ---")
print(confusion_matrix(y_test, y_pred))
```
**[분석]**
바이너리 데이터를 이미지로 변환하여 CNN 모델로 학습시킴으로써, 악성코드 패밀리별 고유한 시각적 패턴을 기반으로 악성코드를 분류할 수 있습니다. 이는 난독화나 패킹으로 인해 정적 분석이 어려운 경우에도 효과적인 분류를 가능하게 합니다.

### 2.2. 📊 그래프 신경망 (Graph Neural Network, GNN)

악성코드의 함수 호출 그래프, API 호출 그래프, 시스템 행위 그래프 등을 GNN 모델로 분석하여 분류하는 기법입니다.

*   **원리:** 악성코드의 내부 구조나 행위 간의 관계를 그래프 형태로 표현하고, GNN이 그래프의 노드(함수, API)와 엣지(호출 관계) 간의 복잡한 상호작용을 학습하여 분류합니다.
*   **AI 연동 전략:**
    *   **그래프 생성:** 디스어셈블된 코드나 동적 분석 로그를 기반으로 함수 호출 그래프, API 호출 그래프 생성.
    *   **GNN 모델 훈련:** 그래프 분류에 특화된 GNN 모델을 훈련하여 악성코드 패밀리 분류.

## 3. 🛡️ AI 기반 악성코드 분류의 보안 고려사항

AI 기반 악성코드 분류는 강력하지만, 다음과 같은 보안 고려사항이 있습니다.

### 3.1. 💥 적대적 공격 (Adversarial Attacks)

공격자가 AI 모델을 속이기 위해 악성코드에 미세한 변경을 가하여 AI가 이를 '정상'으로 오분류하도록 유도하는 공격입니다.

#### **방어 전략:**
*   **적대적 훈련 (Adversarial Training):** 적대적 예제를 훈련 데이터에 포함시켜 모델의 견고성을 높입니다.
*   **앙상블 모델:** 여러 AI 모델의 예측 결과를 결합하여 단일 모델의 취약점을 보완합니다.
*   **특징 다양화:** 정적/동적/하이브리드 특징을 다양하게 활용하여 공격자가 모든 특징을 동시에 조작하기 어렵게 만듭니다.

### 3.2. 💥 데이터 편향 (Data Bias)

훈련 데이터셋에 특정 악성코드 패밀리나 정상 파일이 과도하게 많거나 적을 경우, AI 모델이 편향된 분류 결과를 내놓을 수 있습니다.

#### **방어 전략:**
*   **균형 잡힌 데이터셋 구축:** 다양한 악성코드 패밀리와 정상 파일을 포함하는 균형 잡힌 훈련 데이터셋을 구축합니다.
*   **데이터 증강 (Data Augmentation):** 기존 데이터를 변형하여 새로운 훈련 데이터를 생성하여 데이터 부족 문제를 해결합니다.

## 👨‍💻 현직자 통합 시나리오: AI 기반 멀티모달 악성코드 분류 시스템

**[상황]**
AI 보안 엔지니어 '제미니'는 매일 수집되는 의심 파일에 대해 AI 기반의 멀티모달 악성코드 분류 시스템을 구축하고자 합니다. 이 시스템은 파일의 정적 특징(바이너리 이미지), 동적 특징(API 호출 시퀀스), 그리고 메타데이터(파일 크기, 엔트로피)를 종합적으로 분석하여 악성코드 여부 및 패밀리를 정확하게 분류해야 합니다.

**[워크플로우]**
1.  **의심 파일 수집:** 이메일 첨부 파일, 웹 다운로드 파일 등 의심 파일을 자동으로 수집합니다.
2.  **멀티모달 특징 추출:**
    *   **바이너리 이미지 특징:** 파일의 바이트 스트림을 흑백 이미지로 변환하여 CNN 모델의 입력으로 사용합니다.
    *   **API 호출 시퀀스 특징:** 샌드박스에서 추출된 API 호출 시퀀스를 LSTM 모델의 입력으로 사용합니다.
    *   **메타데이터 특징:** 파일 크기, 섹션 수, 엔트로피 등 수치형 메타데이터를 추출하여 MLP 모델의 입력으로 사용합니다.
3.  **AI 기반 멀티모달 분류:**
    *   각 특징(이미지, 시퀀스, 메타데이터)별로 딥러닝 모델(CNN, LSTM, MLP)을 훈련합니다.
    *   각 모델의 출력(특징 벡터)을 결합(Concatenate)하여 최종 분류를 수행하는 앙상블 또는 멀티모달 딥러닝 모델을 구축합니다.
4.  **악성코드 패밀리 분류 및 변종 탐지:**
    *   최종 분류 모델이 악성코드 여부뿐만 아니라, 어떤 악성코드 패밀리(예: WannaCry, Emotet, Zeus)에 속하는지 분류합니다.
    *   기존 패밀리에 속하지 않지만 악성으로 분류되는 경우, 새로운 변종 악성코드로 탐지합니다.
5.  **자동 대응 및 알림:**
    *   분류된 악성코드의 특성에 따라 EDR, 방화벽 등 보안 솔루션에 연동하여 감염된 시스템을 자동으로 격리하거나 악성코드 실행을 차단합니다.
    *   새로운 악성코드 패밀리나 변종이 탐지되면, 상세 분석 보고서와 함께 Slack으로 알림을 전송합니다.

```python
import numpy as np
import tensorflow as tf
from tensorflow.keras.models import Model
from tensorflow.keras.layers import Input, Conv2D, MaxPooling2D, Flatten, Embedding, LSTM, Dense, Concatenate
from tensorflow.keras.preprocessing.sequence import pad_sequences
from sklearn.model_selection import train_test_split
from sklearn.metrics import classification_report, confusion_matrix
import os
import pefile
import hashlib
import pandas as pd

# --- 0. 가상 악성코드 파일 생성 및 특징 추출 함수 ---
def create_dummy_malware_file(file_path, family_id):
    """가상 악성코드 파일 생성 (개념)"""
    with open(file_path, 'wb') as f:
        f.write(b'MZ\x90\x00\x03\x00\x00\x00\x04\x00\x00\x00\xff\xff\x00\x00\xb8\x00\x00\x00\x00\x00\x00\x00@\x00\x00\x00')
        f.write(b'\x00' * 100)
        
        # 패밀리 ID에 따른 API 문자열 (가상)
        if family_id == 1: # 랜섬웨어
            f.write(b'EncryptFile\x00CryptGenKey\x00DeleteShadowCopies\x00')
        elif family_id == 2: # 스파이웨어
            f.write(b'GetKeyboardState\x00InternetOpenA\x00SendMail\x00')
        else: # 정상
            f.write(b'MessageBoxA\x00GetModuleHandleA\x00')
        
        f.write(b'\x00' * 5000) # 파일 크기 조절

def get_static_metadata_features(file_path):
    """파일의 정적 메타데이터 특징 추출"""
    features = {
        'file_size': os.path.getsize(file_path),
        'num_sections': 0,
        'avg_entropy': 0.0
    }
    try:
        pe = pefile.PE(file_path)
        features['num_sections'] = pe.FILE_HEADER.NumberOfSections
        entropy = 0
        for section in pe.sections:
            entropy += section.get_entropy()
        features['avg_entropy'] = entropy / pe.FILE_HEADER.NumberOfSections if pe.FILE_HEADER.NumberOfSections > 0 else 0
    except pefile.PEFormatError:
        pass # PE 파일이 아니면 기본값 유지
    return features

def get_binary_image(file_path, image_size=32):
    """바이너리 데이터를 이미지로 변환 (개념)"""
    with open(file_path, 'rb') as f:
        byte_data = f.read()
    
    # 바이트 데이터를 0-255 스케일로 변환
    byte_array = np.frombuffer(byte_data, dtype=np.uint8)
    
    # 이미지 크기에 맞게 패딩 또는 자르기
    target_len = image_size * image_size
    if len(byte_array) < target_len:
        padded_array = np.pad(byte_array, (0, target_len - len(byte_array)), 'constant')
    else:
        padded_array = byte_array[:target_len]
    
    return padded_array.reshape(image_size, image_size, 1).astype(np.float32) / 255.0 # 0-1 스케일링

def get_api_sequence(file_path, family_id):
    """가상 API 호출 시퀀스 생성 (동적 분석 결과 가정)"""
    if family_id == 1: # 랜섬웨어
        return [0, 1, 2, 3, 4] # CreateProcess, WriteFile, EncryptFile, CryptGenKey, DeleteShadowCopies
    elif family_id == 2: # 스파이웨어
        return [0, 5, 6, 7] # CreateProcess, GetKeyboardState, InternetOpenA, SendMail
    else: # 정상
        return [0, 8, 9] # CreateProcess, MessageBoxA, GetModuleHandleA

# --- 1. 멀티모달 특징 추출을 위한 데이터셋 생성 ---
num_samples_per_class = 50
image_size = 32
api_vocab_size = 10 # 가상 API 개수
max_api_seq_len = 5 # 최대 API 시퀀스 길이

all_image_features = []
all_api_sequences = []
all_metadata_features = []
all_labels = []

for family_id in range(3): # 0:정상, 1:랜섬웨어, 2:스파이웨어
    for i in range(num_samples_per_class):
        file_name = f"temp_malware_{family_id}_{i}.exe"
        create_dummy_malware_file(file_name, family_id)
        
        all_image_features.append(get_binary_image(file_name, image_size))
        all_api_sequences.append(get_api_sequence(file_name, family_id))
        all_metadata_features.append(get_static_metadata_features(file_name))
        all_labels.append(family_id)
        
        os.remove(file_name) # 임시 파일 삭제

image_features = np.array(all_image_features)
api_sequences = pad_sequences(all_api_sequences, maxlen=max_api_seq_len, padding='post')
metadata_features = pd.DataFrame(all_metadata_features)
labels = np.array(all_labels)

# 메타데이터 특징 스케일링
scaler = StandardScaler()
metadata_features_scaled = scaler.fit_transform(metadata_features)

# 데이터 분할
img_train, img_test, api_train, api_test, meta_train, meta_test, y_train, y_test = train_test_split(
    image_features, api_sequences, metadata_features_scaled, labels, test_size=0.2, random_state=42
)

# --- 2. AI 기반 멀티모달 분류 모델 구축 ---
print("\n--- 🤖 AI 기반 멀티모달 악성코드 분류 모델 구축 ---")

# --- 이미지 입력 파이프라인 (CNN) ---
img_input = Input(shape=(image_size, image_size, 1), name='image_input')
x = Conv2D(32, (3, 3), activation='relu')(img_input)
x = MaxPooling2D((2, 2))(x)
x = Flatten()(x)
img_output = Dense(64, activation='relu')(x)

# --- API 시퀀스 입력 파이프라인 (LSTM) ---
api_input = Input(shape=(max_api_seq_len,), name='api_sequence_input')
y = Embedding(input_dim=api_vocab_size, output_dim=16, input_length=max_api_seq_len)(api_input)
y = LSTM(32)(y)
api_output = Dense(32, activation='relu')(y)

# --- 메타데이터 입력 파이프라인 (MLP) ---
meta_input = Input(shape=(metadata_features_scaled.shape[1],), name='metadata_input')
z = Dense(16, activation='relu')(meta_input)
meta_output = Dense(16, activation='relu')(z)

# --- 모든 출력을 결합 (Concatenate) ---
merged = Concatenate()([img_output, api_output, meta_output])

# --- 최종 분류층 ---
final_output = Dense(num_classes, activation='softmax')(merged)

# --- 모델 생성 ---
multimodal_model = Model(inputs=[img_input, api_input, meta_input], outputs=final_output)

multimodal_model.compile(optimizer='adam', loss='sparse_categorical_crossentropy', metrics=['accuracy'])
multimodal_model.summary()

# --- 3. 모델 훈련 ---
history = multimodal_model.fit(
    {'image_input': img_train, 'api_sequence_input': api_train, 'metadata_input': meta_train},
    y_train,
    epochs=5,
    batch_size=32,
    validation_split=0.2,
    verbose=0
)
print("\n✅ 멀티모달 악성코드 분류 모델 훈련 완료!")

# --- 4. 모델 예측 및 평가 ---
y_pred_proba = multimodal_model.predict({'image_input': img_test, 'api_sequence_input': api_test, 'metadata_input': meta_test})
y_pred = np.argmax(y_pred_proba, axis=1)

print("\n--- 📊 멀티모달 악성코드 분류 모델 평가 결과 ---")
print(classification_report(y_test, y_pred, target_names=['정상', '랜섬웨어', '스파이웨어']))
print("\n--- 📈 혼동 행렬 ---")
print(confusion_matrix(y_test, y_pred))

# --- 5. 악성코드 패밀리 분류 및 변종 탐지 (개념) ---
print("\n--- 🤖 악성코드 패밀리 분류 및 변종 탐지 ---")
# 실제 환경에서는 새로운 샘플에 대해 예측을 수행하고,
# 예측된 패밀리 외에 악성으로 분류되면 변종으로 판단.
# 여기서는 테스트 데이터에 대한 예측 결과를 활용.
for i in range(len(y_test)):
    actual_label = y_test[i]
    predicted_label = y_pred[i]
    
    if predicted_label != 0 and predicted_label != actual_label: # 악성으로 예측했지만 실제 패밀리와 다름
        print(f"  - 🚨 새로운 변종 악성코드 탐지 가능성! (예측: {predicted_label}, 실제: {actual_label})")
    elif predicted_label != 0:
        print(f"  - ✅ 악성코드 패밀리 분류: {['정상', '랜섬웨어', '스파이웨어'][predicted_label]}")

print("\n--- 🚀 AI 기반 멀티모달 악성코드 분류 시스템 (개념) ---")
print("AI가 파일의 정적/동적/메타데이터 특징을 종합 분석하여 악성코드 여부 및 패밀리를 정확하게 분류하는 시스템.")
```

'제미니' 엔지니어는 AI 기반 멀티모달 악성코드 분류 시스템의 핵심 기능을 시연했습니다. 파일의 바이너리 데이터를 이미지로 변환하여 CNN 모델에, 가상 API 호출 시퀀스를 LSTM 모델에, 그리고 메타데이터를 MLP 모델에 입력하여 각 특징을 학습시켰습니다. 이 세 모델의 출력을 `Concatenate` 층으로 결합하여 최종적으로 악성코드 여부 및 패밀리(정상, 랜섬웨어, 스파이웨어)를 분류하는 멀티모달 딥러닝 모델을 구축했습니다. 이는 AI가 다양한 형태의 특징을 종합적으로 분석하여 변종 악성코드까지 탐지할 수 있는 고도화된 분류 시스템을 구축하는 방법을 보여주었습니다.

## 📌 요약 정리 (Executive Summary)

1.  **AI 기반 악성코드 분류의 중요성**: 매일 쏟아지는 수많은 악성코드 샘플과 변종 악성코드를 신속하고 정확하게 분류하여 위협 인텔리전스를 고도화하고, 자동화된 대응을 가능하게 한다.
2.  **AI 기반 악성코드 분류의 특징**:
    *   **정적 특징 기반**: 파일 자체에서 추출한 특징(바이트 시퀀스, API 임포트, 메타데이터)으로 분류. AI는 CNN, NLP, 머신러닝 활용.
    *   **동적 특징 기반**: 샌드박스에서 관찰된 행위(API 호출 시퀀스, 행위 그래프)로 분류. AI는 RNN, LSTM, GNN 활용.
    *   **하이브리드 특징 기반**: 정적/동적 특징을 모두 활용하여 분류 정확도 극대화. 앙상블 또는 멀티모달 딥러닝 활용.
3.  **AI 기반 악성코드 분류의 고급 기법**:
    *   **바이너리 시각화 (Binary Visualization) 및 CNN**: 악성코드의 바이트 스트림을 이미지처럼 변환하여 CNN으로 시각적 패턴 학습 및 분류.
    *   **그래프 신경망 (Graph Neural Network, GNN)**: 함수 호출 그래프, API 호출 그래프 등 악성코드의 관계형 데이터를 GNN으로 분석 및 분류.
4.  **AI 기반 악성코드 분류의 보안 고려사항**:
    *   **적대적 공격 (Adversarial Attacks)**: AI 모델을 속이기 위한 악성코드 변형. 적대적 훈련, 앙상블 모델, 특징 다양화로 방어.
    *   **데이터 편향 (Data Bias)**: 훈련 데이터셋의 불균형. 균형 잡힌 데이터셋 구축, 데이터 증강으로 대응.
5.  **현직자 통합 시나리오**: AI 기반 멀티모달 악성코드 분류 시스템을 통해 파일의 정적 특징(바이너리 이미지), 동적 특징(API 호출 시퀀스), 메타데이터를 종합적으로 분석하여 악성코드 여부 및 패밀리를 정확하게 분류하고, 변종 악성코드까지 탐지하는 고도화된 시스템을 구축한다.
6.  **보안 관점**: AI 기반 악성코드 분류는 악성코드 분석가의 업무 효율성을 극대화하고, 알려지지 않은 악성코드 및 변종까지 탐지하여 신속하고 효과적인 위협 대응을 가능하게 하는 핵심 도구이다. 다양한 AI 기법과 특징을 조합하여 분류 정확도를 높이고, AI 모델 자체의 보안 취약점(적대적 공격)에도 대비해야 한다.
