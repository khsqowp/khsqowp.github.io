## 🤔 네트워크 격리 (VPC)를 왜 배워야 할까요? (The "Why")

클라우드 환경은 기본적으로 공유 인프라 위에 구축됩니다. 즉, 나의 가상 서버가 다른 고객의 가상 서버와 물리적으로 동일한 하드웨어에 존재할 수 있다는 의미입니다. 이러한 공유 환경에서 나의 자원을 안전하게 보호하고, 외부의 위협으로부터 격리하며, 내부 자원 간의 통신을 제어하는 것은 클라우드 보안의 필수 요소입니다.

**[VPC 지식의 중요성]**
*   **논리적 격리:** 퍼블릭 클라우드 환경에서 나만의 독립적인 가상 네트워크 공간을 생성하여, 다른 고객의 자원으로부터 나의 자원을 논리적으로 완벽하게 분리합니다.
*   **네트워크 제어:** IP 주소 범위, 서브넷, 라우팅 테이블, 네트워크 게이트웨이 등을 직접 정의하여 네트워크 트래픽의 흐름을 세밀하게 제어할 수 있습니다.
*   **다층 방어 구축:** VPC 내부에 여러 서브넷을 생성하고, 각 서브넷에 보안 그룹(Security Group)과 네트워크 ACL(Access Control List)을 적용하여 다층적인 네트워크 보안을 구축할 수 있습니다.
*   **AI 기반 VPC 보안:** AI는 VPC의 네트워크 설정(보안 그룹, NACL, 라우팅 테이블)을 분석하여 취약점을 탐지하고, VPC Flow Logs를 분석하여 비정상적인 네트워크 트래픽 패턴을 탐지하여 자동 대응할 수 있습니다.

> **보안 전문가에게 VPC란?**
>
> 클라우드 환경에서 **'나만의 안전한 성(城)을 쌓고, 성벽과 해자를 구축하며, 성문과 통로를 통제하는 설계도'**입니다. VPC를 통해 우리는 클라우드 자원을 외부 위협으로부터 보호하고, 내부 자원 간의 통신을 안전하게 제어하며, AI 기반 보안 시스템이 네트워크 트래픽을 분석하고 대응할 수 있는 기반 환경을 마련합니다.

## 1. 🌐 AWS VPC의 핵심 구성 요소

AWS VPC(Virtual Private Cloud)는 AWS 클라우드에서 정의하는 논리적으로 격리된 네트워크입니다.

### 1.1. 🗺️ VPC (Virtual Private Cloud)

*   **설명:** AWS 클라우드 내에서 사용자가 정의하는 가상 네트워크입니다. IP 주소 범위(CIDR 블록)를 지정하여 생성합니다.
*   **보안 관점:** VPC는 클라우드 자원을 위한 격리된 공간을 제공합니다. VPC 외부의 트래픽은 명시적으로 허용되지 않는 한 접근할 수 없습니다.

### 1.2. 🗺️ 서브넷 (Subnet)

*   **설명:** VPC를 더 작은 네트워크 세그먼트로 나눈 것입니다. 서브넷은 가용 영역(Availability Zone) 내에 존재합니다.
*   **보안 관점:**
    *   **Public Subnet:** 인터넷 게이트웨이를 통해 인터넷과 직접 통신할 수 있는 서브넷. 웹 서버 등 외부에 노출되어야 하는 자원 배치.
    *   **Private Subnet:** 인터넷 게이트웨이를 통해 인터넷과 직접 통신할 수 없는 서브넷. 데이터베이스, 애플리케이션 서버 등 민감한 자원 배치.
    *   **네트워크 분리:** 서브넷을 통해 네트워크를 분리하여, 한 서브넷에서 발생한 침해가 다른 서브넷으로 확산되는 것을 방지합니다.

### 1.3. 🛣️ 라우팅 테이블 (Route Table)

*   **설명:** 네트워크 트래픽이 어디로 가야 하는지(라우팅)를 결정하는 규칙들의 집합입니다. 서브넷과 연결됩니다.
*   **보안 관점:** 잘못된 라우팅 설정은 트래픽이 의도치 않은 경로로 흐르거나, 민감한 자원이 외부에 노출될 수 있습니다.

### 1.4. 🚪 인터넷 게이트웨이 (Internet Gateway, IGW)

*   **설명:** VPC와 인터넷 간의 통신을 가능하게 하는 게이트웨이입니다. Public Subnet이 인터넷과 통신하려면 IGW가 필요합니다.
*   **보안 관점:** IGW는 VPC 외부로 나가는 트래픽의 유일한 통로이므로, IGW를 통해 들어오고 나가는 트래픽에 대한 보안 정책(NACL, 보안 그룹)을 철저히 적용해야 합니다.

### 1.5. 🛡️ 네트워크 ACL (Network Access Control List, NACL)

*   **설명:** 서브넷 수준에서 트래픽을 제어하는 방화벽입니다. 인바운드(Inbound) 및 아웃바운드(Outbound) 규칙을 모두 정의할 수 있습니다.
*   **보안 관점:**
    *   **Stateless:** 연결 상태를 추적하지 않으므로, 인바운드 규칙을 허용하면 해당 응답 트래픽에 대한 아웃바운드 규칙도 명시적으로 허용해야 합니다.
    *   **Deny 규칙:** 명시적인 Deny 규칙을 설정할 수 있어, 특정 IP 대역이나 포트의 트래픽을 서브넷 전체에서 차단하는 데 유용합니다.

### 1.6. 🛡️ 보안 그룹 (Security Group)

*   **설명:** EC2 인스턴스 등 인스턴스 수준에서 트래픽을 제어하는 가상 방화벽입니다.
*   **보안 관점:**
    *   **Stateful:** 연결 상태를 추적하므로, 인바운드 규칙을 허용하면 해당 응답 트래픽에 대한 아웃바운드 규칙은 자동으로 허용됩니다.
    *   **최소 권한 원칙:** 필요한 포트와 IP 대역만 허용하여 공격 표면을 최소화해야 합니다. `0.0.0.0/0` (모든 IP)에 대한 SSH(22) 또는 RDP(3389) 포트 개방은 매우 위험합니다.

## 2. 🛡️ VPC 보안 모범 사례와 AI 기반 분석

VPC는 클라우드 네트워크의 근간이므로, 안전하게 설계하고 관리하는 것이 중요합니다.

### 2.1. 🌐 VPC 설계: Public/Private Subnet 분리

웹 서버는 Public Subnet에, 데이터베이스는 Private Subnet에 배치하여 민감한 자원을 외부로부터 격리합니다.

#### **AI 연동 전략**

| 구분 | AI 연동 방안 |
| :--- | :--- |
| **설계 검토** | AI가 VPC 네트워크 구성을 분석하여 Public Subnet에 민감한 자원(예: RDS 인스턴스)이 배치되어 있는지 탐지하고 경고. |
| **트래픽 흐름 분석** | AI가 VPC Flow Logs를 분석하여 Private Subnet에서 외부 인터넷으로의 비정상적인 직접 통신 시도를 탐지. |

### 2.2. 🛡️ 보안 그룹 및 NACL 규칙 최적화

최소 권한 원칙에 따라 필요한 포트와 IP 대역만 허용하고, 불필요한 규칙은 제거합니다.

#### **AI 연동 전략**

| 구분 | AI 연동 방안 |
| :--- | :--- |
| **규칙 취약점 탐지** | AI가 보안 그룹 및 NACL 규칙을 분석하여 `0.0.0.0/0`에 대한 SSH/RDP 포트 개방, 불필요한 포트 개방 등 취약한 규칙을 탐지. |
| **규칙 최적화 제안** | AI가 실제 트래픽 흐름을 분석하여 사용되지 않는 규칙을 식별하고 제거를 제안하거나, 너무 광범위한 규칙을 더 세분화하도록 제안. |
| **비정상 규칙 변경 탐지** | AI가 CloudTrail 로그를 분석하여 보안 그룹/NACL 규칙의 비정상적인 변경 시도를 탐지. |

### 2.3. 🔬 VPC Flow Logs 분석

VPC Flow Logs는 VPC 내의 네트워크 트래픽 흐름에 대한 정보를 기록합니다. 이를 분석하여 비정상적인 통신 패턴을 탐지할 수 있습니다.

#### **AI 연동 전략**

| 구분 | AI 연동 방안 |
| :--- | :--- |
| **이상 트래픽 탐지** | AI가 VPC Flow Logs를 분석하여 평소와 다른 포트/프로토콜 사용, 비정상적인 데이터 전송량, 특정 IP로부터의 대량 연결 시도 등 이상 트래픽을 탐지. <br> (예: 비지도 학습 모델(IsolationForest)을 사용하여 정상 트래픽 패턴에서 벗어나는 이상치 탐지) |
| **위협 인텔리전스 연동** | AI가 Flow Logs의 출발지/목적지 IP를 외부 위협 인텔리전스(VirusTotal 등)와 연동하여 악성 IP와의 통신을 탐지. |

## 📌 요약 정리 (Executive Summary)

1.  **VPC의 중요성**: 클라우드 환경에서 나만의 독립적인 가상 네트워크 공간을 생성하고, 네트워크 트래픽의 흐름을 세밀하게 제어하여 클라우드 자원을 외부 위협으로부터 격리하는 핵심 서비스이다.
2.  **AWS VPC 핵심 구성 요소**:
    *   **VPC**: 사용자가 정의하는 논리적으로 격리된 네트워크.
    *   **서브넷**: VPC를 더 작은 네트워크 세그먼트로 분리 (Public/Private).
    *   **라우팅 테이블**: 트래픽의 경로를 결정하는 규칙 집합.
    *   **인터넷 게이트웨이 (IGW)**: VPC와 인터넷 간의 통신을 가능하게 하는 통로.
    *   **네트워크 ACL (NACL)**: 서브넷 수준의 Stateless 방화벽. 명시적 Deny 규칙 설정 가능.
    *   **보안 그룹 (Security Group)**: 인스턴스 수준의 Stateful 방화벽. 최소 권한 원칙 적용.
3.  **VPC 보안 모범 사례**:
    *   **Public/Private Subnet 분리**: 민감한 자원을 Private Subnet에 배치하여 외부로부터 격리.
    *   **보안 그룹 및 NACL 규칙 최적화**: 최소 권한 원칙에 따라 필요한 포트와 IP 대역만 허용. `0.0.0.0/0`에 대한 SSH/RDP 개방은 매우 위험.
    *   **VPC Flow Logs 분석**: 네트워크 트래픽 흐름을 기록하여 비정상적인 통신 패턴 탐지.
4.  **AI 기반 VPC 보안**:
    *   **설정 취약점 분석**: AI가 보안 그룹, NACL, 라우팅 테이블 규칙을 분석하여 취약한 설정(예: 과도한 포트 개방)을 탐지하고 개선 제안.
    *   **비정상 트래픽 탐지**: AI가 VPC Flow Logs를 분석하여 평소와 다른 포트/프로토콜 사용, 비정상적인 데이터 전송량 등 이상 트래픽을 탐지 (비지도 학습 활용).
    *   **자동 대응**: 탐지된 위협에 대해 보안 그룹 규칙 변경, IP 차단 등의 자동화된 조치 수행.
5.  **보안 관점**: VPC는 클라우드 네트워크 보안의 근간이며, AI를 활용하여 VPC 설정을 지속적으로 모니터링하고, 비정상적인 트래픽 흐름을 탐지하며, 자동화된 대응을 통해 클라우드 인프라를 더욱 안전하게 구축하고 운영해야 한다.
