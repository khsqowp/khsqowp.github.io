## 🤔 계정 및 접근 관리 (IAM)를 왜 배워야 할까요? (The "Why")

클라우드 환경에서는 모든 자원(가상 서버, 스토리지, 데이터베이스 등)이 API를 통해 접근 가능합니다. 따라서 '누가' 어떤 자원에 접근할 수 있는지를 통제하는 것은 클라우드 보안의 핵심 중의 핵심입니다. IAM은 이러한 접근을 중앙에서 관리하고 통제하는 서비스입니다.

**[IAM 지식의 중요성]**
*   **최소 권한 원칙 (Principle of Least Privilege):** 사용자나 서비스에게 필요한 최소한의 권한만 부여하여, 권한 오남용이나 계정 탈취 시 피해 범위를 최소화합니다. IAM은 이를 구현하는 핵심 도구입니다.
*   **계정 탈취 방지:** 강력한 인증(MFA)과 세분화된 권한 관리를 통해 계정 탈취 시 공격자가 얻을 수 있는 이득을 제한하고, 비정상적인 접근을 조기에 탐지할 수 있습니다.
*   **컴플라이언스 준수:** GDPR, HIPAA 등 규제 준수를 위해 사용자 접근 기록을 감사하고, 적절한 권한 정책이 적용되었음을 증명해야 합니다.
*   **AI 기반 IAM 보안:** AI는 방대한 IAM 활동 로그를 분석하여 비정상적인 권한 변경, 역할 위임, 접근 패턴 등을 탐지하고, IAM 정책의 취약점을 자동으로 분석하여 개선을 제안할 수 있습니다.

> **보안 전문가에게 IAM이란?**
>
> 클라우드 환경의 **'신분증 발급 및 출입 통제 시스템'**입니다. 누가 어떤 문을 열 수 있고, 어떤 자원을 사용할 수 있는지에 대한 모든 규칙을 정의하고 강제하는 핵심 보안 메커니즘입니다. IAM을 제대로 이해하고 활용하는 것은 클라우드 환경에서 **'보안의 시작이자 끝'**이라고 할 수 있습니다.

## 1. 🔑 AWS IAM의 핵심 구성 요소

AWS IAM(Identity and Access Management)은 AWS 리소스에 대한 접근을 안전하게 관리할 수 있도록 돕는 웹 서비스입니다.

### 1.1. 👤 사용자 (Users)

AWS 리소스와 상호 작용하는 사람 또는 애플리케이션을 나타냅니다.

*   **IAM 사용자:** AWS 계정 내에서 생성되는 개별 사용자. 고유한 자격 증명(액세스 키, 비밀번호)을 가집니다.
*   **루트 사용자:** AWS 계정을 생성할 때 만들어지는 최상위 사용자. 모든 권한을 가지므로 일상적인 작업에는 사용하지 않는 것이 강력히 권고됩니다.

### 1.2. 👥 그룹 (Groups)

IAM 사용자들의 집합입니다. 그룹에 권한을 부여하면, 해당 그룹에 속한 모든 사용자에게 동일한 권한이 적용됩니다.

### 1.3. 👮 역할 (Roles)

특정 엔티티(IAM 사용자, AWS 서비스, 외부 계정 등)가 임시적으로 권한을 위임받아 AWS 리소스에 접근할 수 있도록 하는 보안 자격 증명입니다. 비밀번호나 액세스 키를 가지지 않습니다.

> **보안 관점: 역할(Role)의 중요성**
>
> *   **최소 권한 원칙 구현:** 특정 작업 수행 시에만 필요한 권한을 가진 역할을 부여하여, 평소에는 최소한의 권한만 가지도록 합니다.
> *   **자격 증명 관리 간소화:** EC2 인스턴스나 Lambda 함수와 같은 AWS 서비스가 다른 AWS 서비스에 접근할 때, 액세스 키를 하드코딩하는 대신 역할을 부여하여 자격 증명 유출 위험을 줄입니다.

### 1.4. 📜 정책 (Policies)

AWS 리소스에 대한 권한을 정의하는 문서입니다. JSON 형식으로 작성되며, 어떤 작업(Action)을 어떤 리소스(Resource)에 대해 허용(Allow)하거나 거부(Deny)할지 명시합니다.

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "s3:GetObject",
                "s3:ListBucket"
            ],
            "Resource": [
                "arn:aws:s3:::my-secure-bucket",
                "arn:aws:s3:::my-secure-bucket/*"
            ]
        },
        {
            "Effect": "Deny",
            "Action": "s3:DeleteObject",
            "Resource": "arn:aws:s3:::my-secure-bucket/*"
        }
    ]
}
```

> **용어 해설: ARN (Amazon Resource Name)**
>
> AWS 리소스를 고유하게 식별하는 표준화된 이름 형식입니다. `arn:aws:s3:::my-secure-bucket`과 같이 사용됩니다.

## 2. 🛡️ IAM 보안 모범 사례와 AI 기반 분석

IAM은 강력하지만, 잘못 설정하면 심각한 보안 취약점이 됩니다. 다음은 IAM 보안 모범 사례와 이를 AI로 어떻게 강화할 수 있는지 보여줍니다.

### 2.1. 🔑 강력한 인증 (Multi-Factor Authentication, MFA)

비밀번호 외에 추가적인 인증 수단(OTP, 지문 등)을 사용하여 계정 탈취 위험을 크게 줄입니다.

#### **AI 연동 전략**

| 구분 | AI 연동 방안 |
| :--- | :--- |
| **MFA 미사용 탐지** | AI가 IAM 사용자 목록을 스캔하여 MFA가 활성화되지 않은 루트 사용자나 관리자 계정을 탐지하고 경고. |
| **MFA 우회 탐지** | AI가 로그인 로그를 분석하여 MFA가 활성화된 계정에서 비정상적으로 MFA가 우회된 시도를 탐지. |

### 2.2. 👮 최소 권한 원칙 (Principle of Least Privilege)

사용자나 서비스에게 필요한 최소한의 권한만 부여합니다. `AdministratorAccess`와 같은 광범위한 권한은 피해야 합니다.

#### **AI 연동 전략**

| 구분 | AI 연동 방안 |
| :--- | :--- |
| **권한 과부여 탐지** | AI가 IAM 정책을 분석하여 사용되지 않는 권한(Unused Permissions)이나 과도하게 부여된 권한(Over-privileged)을 식별하고 개선 제안. <br> (예: S3 Read 권한만 필요한 사용자에게 S3 Full Access 정책이 부여된 경우) |
| **비정상 권한 변경 탐지** | AI가 CloudTrail 로그를 분석하여 IAM 정책 변경, 역할 위임 등 비정상적인 권한 변경 시도를 탐지. |

### 2.3. 🔄 임시 자격 증명 사용 (Temporary Credentials)

장기적인 액세스 키 대신 IAM 역할을 통해 임시 자격 증명을 사용하여 자격 증명 유출 위험을 줄입니다.

#### **AI 연동 전략**

| 구분 | AI 연동 방안 |
| :--- | :--- |
| **장기 액세스 키 탐지** | AI가 IAM 사용자 목록을 스캔하여 장기 액세스 키가 활성화된 사용자나 루트 사용자의 액세스 키를 탐지하고 경고. |
| **액세스 키 유출 탐지** | AI가 GitHub 등 공개 저장소를 모니터링하여 유출된 AWS 액세스 키를 탐지. |

**[상황]**
AI 보안 엔지니어 '제미니'는 회사의 AWS 클라우드 환경에서 IAM 정책의 취약점을 분석하고, 비정상적인 접근 시도를 실시간으로 탐지하는 시스템을 구축하고자 합니다.

**[워크플로우]**
1.  **IAM 데이터 수집:** AWS IAM API를 통해 사용자, 그룹, 역할, 정책 정보를 수집하고, CloudTrail 로그를 통해 IAM 활동 로그를 수집합니다. (boto3)
2.  **AI 기반 IAM 정책 분석:**
    *   AI가 IAM 정책을 파싱하여 `*` (모든 리소스/액션)와 같은 와일드카드 사용 여부, `Deny` 규칙의 부재 등을 분석하여 권한 과부여 가능성을 탐지합니다.
    *   AI가 IAM Access Analyzer와 유사하게 정책의 잠재적 접근 경로를 분석하여 외부 공개 여부를 판단합니다.
3.  **AI 기반 비정상 접근 탐지:**
    *   AI가 CloudTrail 로그인 로그를 분석하여 평소와 다른 지역/IP에서의 로그인, 짧은 시간 내 다수 로그인 실패 등 비정상적인 로그인 패턴을 탐지합니다.
    *   AI가 IAM 활동 로그를 분석하여 비정상적인 권한 변경(예: `iam:AttachUserPolicy` 호출 빈도 급증)이나 역할 위임 시도를 탐지합니다.
4.  **자동 대응 및 알림:**
    *   탐지된 권한 과부여 정책에 대해 개선 권고를 Slack으로 알립니다.
    *   탐지된 비정상 로그인 시도에 대해 해당 계정을 일시적으로 잠그거나, MFA를 강제하는 조치를 취합니다.

```python
import boto3
import pandas as pd
import numpy as np
import json
from sklearn.ensemble import IsolationForest
from sklearn.preprocessing import LabelEncoder
import re # 정책 분석을 위한 정규표현식

# --- 0. AWS 자격 증명 설정 (실제 환경에서는 환경 변수, IAM Role 사용) ---
# os.environ['AWS_ACCESS_KEY_ID'] = 'YOUR_ACCESS_KEY'
# os.environ['AWS_SECRET_ACCESS_KEY'] = 'YOUR_SECRET_KEY'
# os.environ['AWS_DEFAULT_REGION'] = 'ap-northeast-2'

# --- 1. IAM 데이터 수집 (개념) ---
# iam_client = boto3.client('iam')
# cloudtrail_client = boto3.client('cloudtrail')

def get_iam_policies():
    """가상 IAM 정책 데이터 반환"""
    return [
        {'name': 'AdminPolicy', 'document': '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Action":"*","Resource":"*"}]}'}, # ⚠️ 과도한 권한
        {'name': 'S3ReadOnlyPolicy', 'document': '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Action":["s3:GetObject","s3:ListBucket"],"Resource":["arn:aws:s3:::my-bucket","arn:aws:s3:::my-bucket/*"]}]}'},
        {'name': 'DevPolicy', 'document': '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Action":["ec2:*","s3:PutObject"],"Resource":"*"}]}'} # ⚠️ 과도한 리소스
    ]

def get_cloudtrail_events():
    """가상 CloudTrail 이벤트 로그 반환"""
    return pd.DataFrame({
        'timestamp': pd.to_datetime(['2023-11-12 10:00', '2023-11-12 10:05', '2023-11-12 10:10', '2023-11-12 10:11', '2023-11-12 10:12']),
        'event_name': ['ConsoleLogin', 'ConsoleLogin', 'ConsoleLogin', 'iam:AttachUserPolicy', 'ConsoleLogin'],
        'user_identity': ['user1', 'user1', 'user2', 'user1', 'user1'],
        'source_ip_address': ['1.1.1.1', '1.1.1.1', '2.2.2.2', '5.5.5.5', '1.1.1.1'], # 5.5.5.5는 비정상 IP 가정
        'aws_region': ['ap-northeast-2', 'ap-northeast-2', 'us-east-1', 'eu-west-1', 'ap-northeast-2'], # eu-west-1은 비정상 지역 가정
        'error_code': [np.nan, np.nan, np.nan, np.nan, np.nan]
    })

# --- 2. AI 기반 IAM 정책 분석 ---
print("--- 🔑 AI 기반 IAM 정책 취약점 분석 ---")
iam_policies = get_iam_policies()
for policy in iam_policies:
    policy_doc = json.loads(policy['document'])
    for statement in policy_doc['Statement']:
        if statement['Effect'] == 'Allow':
            # 모든 액션 허용 탐지
            if '*' in statement['Action']:
                print(f"  - 🚨 경고: 정책 '{policy['name']}'에서 모든 액션(`*`)을 허용합니다. (권한 과부여 가능성)")
            # 모든 리소스 허용 탐지
            if '*' in statement['Resource']:
                print(f"  - 🚨 경고: 정책 '{policy['name']}'에서 모든 리소스(`*`)에 접근을 허용합니다. (권한 과부여 가능성)")
            
            # 특정 위험한 액션 탐지 (예: iam:DeleteUser, s3:DeleteBucket 등)
            dangerous_actions = ['iam:DeleteUser', 's3:DeleteBucket', 'ec2:TerminateInstances']
            if any(action in statement['Action'] for action in dangerous_actions):
                print(f"  - ⚠️ 주의: 정책 '{policy['name']}'에 위험한 액션({', '.join(dangerous_actions)})이 포함되어 있습니다.")

# --- 3. AI 기반 비정상 접근 탐지 ---
print("\n--- 🔑 AI 기반 비정상 IAM 활동 탐지 ---")
cloudtrail_events_df = get_cloudtrail_events()

# 비정상 활동 탐지를 위한 특징 추출 및 인코딩
cloudtrail_events_df['user_identity_encoded'] = pd.factorize(cloudtrail_events_df['user_identity'])[0]
cloudtrail_events_df['source_ip_encoded'] = pd.factorize(cloudtrail_events_df['source_ip_address'])[0]
cloudtrail_events_df['region_encoded'] = pd.factorize(cloudtrail_events_df['aws_region'])[0]
cloudtrail_events_df['event_name_encoded'] = pd.factorize(cloudtrail_events_df['event_name'])[0]

features = cloudtrail_events_df[['user_identity_encoded', 'source_ip_encoded', 'region_encoded', 'event_name_encoded']]

# IsolationForest 모델 훈련 및 예측
iso_forest = IsolationForest(random_state=42, contamination=0.1) 
cloudtrail_events_df['anomaly'] = iso_forest.fit_predict(features)

anomalous_activities = cloudtrail_events_df[cloudtrail_events_df['anomaly'] == -1]

if not anomalous_activities.empty:
    for index, row in anomalous_activities.iterrows():
        print(f"  - 🚨 경고: 비정상 IAM 활동 탐지! 사용자 '{row['user_identity']}' (IP: {row['source_ip_address']}, 지역: {row['aws_region']}, 이벤트: {row['event_name']})")
        # 💡 자동 대응: boto3를 사용하여 해당 IAM 계정의 세션을 취소하거나, MFA 강제
        # iam_client.create_virtual_mfa_device(...)
else:
    print("  - 비정상 IAM 활동이 탐지되지 않았습니다.")

# --- 4. 자동 대응 및 알림 (개념) ---
# send_slack_alert("IAM 보안 시스템에서 새로운 위협이 탐지되었습니다.")

print("\n--- 🚀 AI 기반 IAM 보안 시스템 (개념) ---")
```

## 📌 요약 정리 (Executive Summary)

1.  **IAM의 중요성**: 클라우드 환경에서 '누가', '무엇에', '어떻게' 접근할 수 있는지를 중앙에서 관리하고 통제하는 핵심 보안 서비스이다. 최소 권한 원칙 구현, 계정 탈취 방지, 컴플라이언스 준수에 필수적이다.
2.  **AWS IAM 핵심 구성 요소**:
    *   **사용자 (Users)**: AWS 리소스와 상호 작용하는 개별 엔티티.
    *   **그룹 (Groups)**: 사용자들의 집합으로, 그룹에 정책을 부여하여 일괄 관리.
    *   **역할 (Roles)**: 임시 자격 증명을 통해 권한을 위임하는 메커니즘. 최소 권한 원칙 구현 및 자격 증명 유출 위험 감소에 중요.
    *   **정책 (Policies)**: JSON 형식으로 작성되며, 리소스에 대한 권한(Action, Resource, Effect)을 정의.
3.  **IAM 보안 모범 사례**:
    *   **MFA (Multi-Factor Authentication)**: 강력한 인증으로 계정 탈취 방지.
    *   **최소 권한 원칙 (Principle of Least Privilege)**: 필요한 최소한의 권한만 부여.
    *   **임시 자격 증명 사용**: 장기 액세스 키 대신 역할 사용.
4.  **AI 기반 IAM 보안**:
    *   **정책 분석**: AI가 IAM 정책을 분석하여 권한 과부여, 와일드카드 사용, 위험한 액션 포함 여부 등 취약점을 탐지하고 개선 제안.
    *   **비정상 접근 탐지**: AI가 CloudTrail 로그를 분석하여 비정상 로그인(지역/IP 변경, 실패 횟수), 비정상 권한 변경/위임 시도 등을 탐지.
    *   **자동 대응**: 탐지된 위협에 대해 계정 잠금, MFA 강제, 알림 전송 등의 자동화된 조치 수행.
5.  **보안 관점**: IAM은 클라우드 보안의 첫 번째 방어선이자 가장 중요한 통제 지점이다. AI를 활용하여 IAM 정책의 취약점을 지속적으로 분석하고, 비정상적인 접근을 실시간으로 탐지하며, 자동화된 대응을 통해 클라우드 자원에 대한 무단 접근을 효과적으로 방지해야 한다.
