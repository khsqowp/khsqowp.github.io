## 🤔 네트워크 기초를 왜 배워야 할까요? (The "Why")

현대의 모든 IT 시스템은 네트워크 없이는 존재할 수 없습니다. 웹 서버, 데이터베이스, 클라우드 인프라, 심지어 IoT 기기까지 모든 것이 네트워크를 통해 연결되고 통신합니다. 사이버 공격의 대부분은 이 네트워크를 통해 이루어지며, 공격자는 네트워크의 취약점을 파고들거나 네트워크 트래픽을 조작하여 목표를 달성합니다.

**[네트워크 지식 없이는 불가능한 보안 활동]**
*   **침입 탐지 및 방어:** 네트워크 트래픽을 분석하여 비정상적인 통신 패턴(예: C2 서버 통신, 포트 스캔)을 탐지하고, 방화벽, IDS/IPS 등으로 차단 정책을 수립하려면 네트워크 프로토콜에 대한 이해가 필수적입니다.
*   **모의 해킹:** 시스템의 취약점을 찾고 공격을 시뮬레이션하려면 네트워크 구성, 포트, 서비스 등을 정확히 파악해야 합니다.
*   **포렌식 분석:** 침해 사고 발생 시 네트워크 패킷을 분석하여 공격자의 침투 경로, 사용 도구, 데이터 유출 여부 등을 파악해야 합니다.
*   **클라우드 보안:** 클라우드 환경의 가상 네트워크(VPC, Security Group)를 안전하게 설계하고 관리하려면 네트워크 기본 지식이 필수적입니다.

> **보안 전문가에게 네트워크 기초란?**
>
> 사이버 공간이라는 **'전장(Battlefield)'의 지형과 규칙**을 이해하는 것입니다. 네트워크의 작동 원리를 모른다면, 공격자가 어디로 침투하고, 어떤 무기를 사용하며, 어떻게 흔적을 남기는지 파악할 수 없습니다. AI 모델이 탐지해야 할 '위협'이 무엇인지, 그리고 그 위협이 네트워크 상에서 어떻게 발현되는지를 이해하는 **가장 근본적인 지식**입니다.

## 1. 🌐 네트워크의 기본: OSI 7계층 모델

OSI(Open Systems Interconnection) 7계층 모델은 네트워크 통신 과정을 7개의 추상적인 계층으로 나눈 표준 모델입니다. 각 계층은 독립적인 기능을 수행하며, 상위 계층은 하위 계층의 서비스에 의존합니다. 이 모델을 이해하면 복잡한 네트워크 통신 과정을 체계적으로 분석하고, 특정 계층에서 발생하는 보안 문제를 식별하는 데 큰 도움이 됩니다.

```mermaid
graph TD
    A[7. 응용 계층 (Application Layer)] --> B[6. 표현 계층 (Presentation Layer)];
    B --> C[5. 세션 계층 (Session Layer)];
    C --> D[4. 전송 계층 (Transport Layer)];
    D --> E[3. 네트워크 계층 (Network Layer)];
    E --> F[2. 데이터 링크 계층 (Data Link Layer)];
    F --> G[1. 물리 계층 (Physical Layer)];
```

| 계층 | 이름 (영문) | 주요 기능 | 보안 관점에서의 의미 | 주요 프로토콜 (예시) |
| :--- | :--- | :--- | :--- | :--- |
| **7계층** | **응용 (Application)** | 사용자에게 네트워크 서비스 제공 | 웹 취약점(XSS, SQLi), 서비스 거부(DoS), 악성코드 유포 | HTTP, FTP, SMTP, DNS |
| **6계층** | **표현 (Presentation)** | 데이터 형식 변환, 암호화/복호화 | 암호화 취약점, 데이터 변조 | JPEG, MPEG, SSL/TLS |
| **5계층** | **세션 (Session)** | 세션 연결/유지/종료 | 세션 하이재킹, 인증 우회 | NetBIOS, RPC |
| **4계층** | **전송 (Transport)** | 신뢰성 있는 데이터 전송 (End-to-End) | 포트 스캔, DoS (SYN Flooding), 방화벽 우회 | TCP, UDP |
| **3계층** | **네트워크 (Network)** | IP 주소를 이용한 경로 설정 (라우팅) | IP 스푸핑, 라우팅 프로토콜 공격, ICMP 공격 | IP, ICMP, ARP |
| **2계층** | **데이터 링크 (Data Link)** | 물리적 주소(MAC)를 이용한 데이터 전송 | ARP 스푸핑, MAC Flooding, 스위치 보안 | Ethernet, PPP |
| **1계층** | **물리 (Physical)** | 비트 스트림 전송 (전기 신호) | 물리적 침입, 도청, 케이블 절단 | 케이블, 허브, 리피터 |

> **용어 해설: 프로토콜 (Protocol)**
>
> 네트워크 통신에서 데이터를 주고받는 규칙과 절차의 집합입니다. 각 계층마다 고유한 프로토콜이 존재하며, 이 프로토콜의 취약점을 이용한 공격이 빈번하게 발생합니다.

### 1.2. 🌐 TCP/IP 4계층 모델

실제 인터넷에서 주로 사용되는 모델은 TCP/IP 4계층 모델입니다. OSI 7계층 모델을 간소화한 형태로, 각 계층의 역할은 OSI 모델과 유사합니다.

```mermaid
graph TD
    A[4. 응용 계층 (Application Layer)] --> B[3. 전송 계층 (Transport Layer)];
    B --> C[2. 인터넷 계층 (Internet Layer)];
    C --> D[1. 네트워크 접근 계층 (Network Access Layer)];
```

## 2. 🌐 IP 주소와 서브넷 마스크: 네트워크의 주소 체계

네트워크에 연결된 모든 장치는 고유한 주소를 가집니다. 이 주소 체계를 이해하는 것은 네트워크 보안의 기본입니다.

### 2.1. 🔢 IP 주소 (Internet Protocol Address)

네트워크에 연결된 장치(호스트)를 식별하고, 데이터를 주고받기 위한 논리적인 주소입니다.

*   **IPv4:** 32비트 주소 체계. `192.168.1.1`과 같이 옥텟(8비트) 4개로 구성됩니다. (약 43억 개 주소)
*   **IPv6:** 128비트 주소 체계. `2001:0db8:85a3:0000:0000:8a2e:0370:7334`와 같이 16진수 8개 블록으로 구성됩니다. (거의 무한대에 가까운 주소)

> **보안 관점:**
>
> *   **IP 스푸핑 (IP Spoofing):** 공격자가 자신의 IP 주소를 위조하여 다른 시스템인 것처럼 속이는 공격입니다.
> *   **IP 기반 접근 제어:** 특정 IP 주소 대역만 서비스에 접근하도록 허용하는 방화벽 규칙 등은 IP 스푸핑에 취약할 수 있습니다.

### 2.2. 🎭 서브넷 마스크 (Subnet Mask)

IP 주소에서 네트워크 주소와 호스트 주소를 구분하는 데 사용됩니다. 서브넷 마스크를 통해 하나의 네트워크를 여러 개의 작은 서브넷(Subnet)으로 나눌 수 있습니다.

*   **예시:**
    *   IP 주소: `192.168.1.100`
    *   서브넷 마스크: `255.255.255.0` (`/24`)
    *   네트워크 주소: `192.168.1.0`
    *   호스트 주소: `0.0.0.100`

> **보안 관점:**
>
> *   **네트워크 분리:** 서브넷을 통해 네트워크를 분리하면, 한 서브넷에서 발생한 침해가 다른 서브넷으로 확산되는 것을 방지할 수 있습니다. (보안 구역 설정)
> *   **네트워크 스캐닝:** 공격자는 서브넷 마스크를 이용하여 특정 네트워크 대역에 존재하는 모든 호스트를 스캔하여 공격 대상을 찾습니다.

## 3. 🌐 주요 네트워크 장비: 통신의 흐름과 보안

네트워크 장비들은 데이터의 흐름을 제어하고, 보안 정책을 적용하는 데 핵심적인 역할을 합니다.

| 장비 | OSI 계층 | 주요 기능 | 보안 관점에서의 의미 |
| :--- | :--- | :--- | :--- |
| **허브 (Hub)** | 1계층 | 모든 포트로 데이터 전송 (브로드캐스트) | 보안 취약: 모든 트래픽이 노출되어 스니핑(Sniffing) 공격에 취약 |
| **스위치 (Switch)** | 2계층 | MAC 주소를 이용한 특정 포트로 데이터 전송 | 허브보다 보안 강화. 하지만 MAC Flooding, ARP Spoofing에 취약 |
| **라우터 (Router)** | 3계층 | IP 주소를 이용한 네트워크 간 경로 설정 | 네트워크 경계 보안의 핵심. 방화벽 기능 내장 가능. 라우팅 프로토콜 공격 대상 |
| **방화벽 (Firewall)** | 3~7계층 | 미리 정의된 규칙에 따라 트래픽 허용/차단 | 네트워크 보안의 최전선. 정책 설정 오류 시 우회 가능 |
| **IDS/IPS** | 3~7계층 | 침입 탐지/방지 시스템 | 비정상 트래픽 탐지 및 차단. 오탐/미탐 발생 가능 |

> **용어 해설: MAC 주소 (Media Access Control Address)**
>
> 네트워크 인터페이스 카드(NIC)에 고유하게 할당된 48비트 물리적 주소입니다. 2계층에서 사용됩니다.

## 👨‍💻 현직자 통합 시나리오: 침해사고 분석가의 네트워크 트래픽 분석

**[상황]**
침해사고 분석가 '쉴더'는 내부 서버에서 비정상적인 외부 통신이 감지되었다는 경보를 받았습니다. '쉴더'는 해당 서버의 네트워크 트래픽 덤프 파일(`traffic.pcap`)을 분석하여 공격자의 통신 패턴과 침투 경로를 파악해야 합니다.

**[분석 목표]**
1.  OSI 7계층 모델을 기반으로 트래픽의 흐름을 이해합니다.
2.  IP 주소와 포트 정보를 통해 통신 대상과 서비스를 식별합니다.
3.  의심스러운 프로토콜이나 통신 패턴을 찾아냅니다.

**[사용 도구]**
*   `Wireshark`: 네트워크 패킷 분석 도구 (GUI)
*   `Scapy` (Python 라이브러리): 패킷 생성 및 분석 (스크립트 기반)

#### **PoC: `Scapy`를 이용한 네트워크 패킷 분석 (개념)**

```python
# ⚠️ Scapy는 설치가 필요합니다: pip install scapy
# ⚠️ 이 코드는 실제 네트워크 패킷을 생성하거나 분석하는 것이 아니라,
#    개념 이해를 위한 가상의 패킷 데이터를 다룹니다.
#    실제 환경에서는 Wireshark로 캡처한 .pcap 파일을 읽어 분석합니다.

from scapy.all import IP, TCP, UDP, Raw, Ether # Scapy 모듈 임포트
from scapy.layers.http import HTTPRequest # HTTP 레이어 임포트 (설치 필요: pip install scapy_http)

# --- 1. 가상 패킷 데이터 생성 ---
# 💡 Scapy는 파이썬 객체로 네트워크 패킷을 추상화합니다.
#    각 계층의 프로토콜을 객체로 쌓아 올리듯이 패킷을 구성합니다.

# 1-1. 정상적인 웹 요청 패킷 (HTTP GET)
normal_packet = Ether()/IP(dst="192.168.1.1", src="192.168.1.100")/TCP(dport=80)/Raw(load="GET /index.html HTTP/1.1\r\nHost: example.com\r\n\r\n")

# 1-2. 의심스러운 SSH 접속 시도 패킷 (외부 IP -> 내부 SSH 포트)
suspicious_packet = Ether()/IP(dst="192.168.1.50", src="203.0.113.5")/TCP(dport=22, flags="S") # SYN 플래그

# 1-3. DNS 쿼리 패킷 (UDP)
dns_query_packet = Ether()/IP(dst="8.8.8.8", src="192.168.1.10")/UDP(dport=53)/Raw(load="DNS Query for malware.com")

# --- 2. 패킷 분석 ---
packets = [normal_packet, suspicious_packet, dns_query_packet]

print("--- 🔬 네트워크 패킷 분석 시작 ---")
for i, pkt in enumerate(packets):
    print(f"\n--- 패킷 {i+1} ---")
    # pkt.show() # 패킷의 모든 계층 정보를 상세히 보여줌 (출력량이 많음)

    # ✅ 1. OSI 계층별 정보 추출
    if pkt.haslayer(Ether):
        print(f"  - 2계층 (데이터 링크): Source MAC={pkt[Ether].src}, Dest MAC={pkt[Ether].dst}")
    if pkt.haslayer(IP):
        print(f"  - 3계층 (네트워크): Source IP={pkt[IP].src}, Dest IP={pkt[IP].dst}, Protocol={pkt[IP].proto}")
    if pkt.haslayer(TCP):
        print(f"  - 4계층 (전송): Source Port={pkt[TCP].sport}, Dest Port={pkt[TCP].dport}, Flags={pkt[TCP].flags}")
    elif pkt.haslayer(UDP):
        print(f"  - 4계층 (전송): Source Port={pkt[UDP].sport}, Dest Port={pkt[UDP].dport}")
    
    # ✅ 2. 7계층 (응용) 프로토콜 분석 (HTTP 예시)
    if pkt.haslayer(HTTPRequest):
        http_req = pkt[HTTPRequest]
        print(f"  - 7계층 (응용): HTTP 요청 - Method={http_req.Method.decode()}, Host={http_req.Host.decode()}, Path={http_req.Path.decode()}")
    elif pkt.haslayer(Raw) and b"DNS Query" in pkt[Raw].load:
        print(f"  - 7계층 (응용): DNS 쿼리 - {pkt[Raw].load.decode()}")

    # ✅ 3. 의심스러운 패턴 탐지
    if pkt.haslayer(IP) and pkt[IP].src == "203.0.113.5" and pkt.haslayer(TCP) and pkt[TCP].dport == 22:
        print("  - 🚨 경고: 외부 IP로부터 내부 SSH 포트(22)로의 의심스러운 접속 시도 탐지!")
    if pkt.haslayer(Raw) and b"malware.com" in pkt[Raw].load:
        print("  - 🚨 경고: 악성 도메인 'malware.com'에 대한 DNS 쿼리 탐지!")

print("\n--- 🔬 네트워크 패킷 분석 완료 ---")
```
**[시나리오 분석]**
'쉴더' 분석가는 `Scapy`를 활용하여 네트워크 패킷을 계층별로 분석하고, IP 주소, 포트, 프로토콜, 그리고 응용 계층의 데이터까지 추출하는 과정을 시연했습니다. 이를 통해 외부 IP(`203.0.113.5`)에서 내부 SSH 포트(`22`)로의 접속 시도와 악성 도메인(`malware.com`)에 대한 DNS 쿼리 등 잠재적인 침해 징후를 식별할 수 있었습니다. 이러한 분석은 AI 모델이 학습할 '특징'을 추출하는 데 중요한 기반이 됩니다.

## 📌 요약 정리 (Executive Summary)

1.  **네트워크 기초의 중요성**: 모든 사이버 공격은 네트워크를 통해 이루어지므로, 네트워크의 작동 원리를 이해하는 것은 침입 탐지, 모의 해킹, 포렌식 분석 등 모든 보안 활동의 가장 근본적인 시작점이다.
2.  **OSI 7계층 모델**: 네트워크 통신 과정을 7개의 계층으로 추상화한 표준 모델로, 각 계층의 기능과 보안적 의미를 이해하면 체계적인 분석이 가능하다.
3.  **TCP/IP 4계층 모델**: 실제 인터넷에서 주로 사용되는 모델로, OSI 7계층을 간소화한 형태이다.
4.  **IP 주소와 서브넷 마스크**:
    *   **IP 주소**: 네트워크 장치를 식별하는 논리적 주소. IP 스푸핑 공격의 대상이 될 수 있다.
    *   **서브넷 마스크**: IP 주소에서 네트워크와 호스트를 구분하며, 네트워크 분리를 통한 보안 강화에 활용된다.
5.  **주요 네트워크 장비**:
    *   **허브**: 1계층, 모든 트래픽 노출로 스니핑에 취약.
    *   **스위치**: 2계층, MAC 주소 기반 통신으로 허브보다 보안 강화. MAC Flooding, ARP Spoofing에 취약.
    *   **라우터**: 3계층, 네트워크 간 경로 설정. 네트워크 경계 보안의 핵심.
    *   **방화벽, IDS/IPS**: 네트워크 보안의 최전선에서 트래픽을 제어하고 침입을 탐지/방지한다.
6.  **`Scapy` 활용**: 파이썬 기반의 강력한 패킷 조작 및 분석 라이브러리로, 네트워크 패킷을 계층별로 분석하고 의심스러운 패턴을 식별하는 데 활용될 수 있다. 이는 AI 모델이 학습할 특징을 추출하는 데 중요한 기반이 된다.
7.  **보안 관점**: 네트워크의 모든 요소는 잠재적인 공격 대상이 될 수 있으며, 각 계층의 프로토콜과 장비의 작동 원리를 깊이 이해해야만 효과적인 방어 전략을 수립하고 AI 모델이 탐지해야 할 '진짜 위협'을 정의할 수 있다.
