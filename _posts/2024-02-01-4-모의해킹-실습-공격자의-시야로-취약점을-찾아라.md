# 💥 4. 모의해킹 실습: 공격자의 시야로 취약점을 찾아라

## 🔄 이전 단원 복습: 보안 분석 (SAST, DAST)

이전 단원 `3. 보안 분석 (SAST, DAST).md`에서 우리는 애플리케이션 취약점 진단의 핵심 방법론인 정적 분석(SAST)과 동적 분석(DAST)의 원리, 장단점, 그리고 AI 기반으로 이러한 분석을 고도화하는 방법을 학습했습니다. 또한 AI 기반 CI/CD 파이프라인 보안 게이트 시나리오를 통해 개발 프로세스에 보안을 통합하는 방법을 살펴보았습니다.

> **핵심 복습:**
>
> *   **SAST:** 코드를 실행하지 않고 소스 코드 분석. 개발 초기 단계 적용, 취약점 위치 정확. AI는 오탐/미탐 감소, 코드 문맥 이해, 자동 수정 제안.
> *   **DAST:** 애플리케이션을 실행한 상태에서 실제 공격 시뮬레이션. 오탐률 낮음, 런타임 환경 취약점 탐지. AI는 공격 시나리오 최적화, 응답 분석 고도화.
> *   **통합 분석:** SAST와 DAST는 보완적으로 활용되며, AI는 두 도구의 결과를 통합 분석하여 취약점 진단의 정확도와 효율성을 극대화.
>
> SAST와 DAST는 자동화된 취약점 진단에 매우 효과적이지만, 도구의 한계로 인해 모든 취약점을 찾아내지는 못합니다. 특히 복잡한 비즈니스 로직의 결함이나, 여러 취약점이 복합적으로 작용하여 발생하는 시나리오 기반의 공격은 자동화된 도구만으로는 탐지하기 어렵습니다. 오늘 배울 '모의해킹 실습' 단원에서는 **공격자의 관점**에서 시스템의 취약점을 찾아내고, 이를 악용하여 침투하는 과정을 직접 경험하며, 방어자의 입장에서 AI 기반으로 모의해킹을 자동화하고 방어하는 방법을 학습합니다.

---

## 🤔 모의해킹 실습을 왜 배워야 할까요? (The "Why")

모의해킹(Penetration Testing)은 실제 해킹과 동일한 방법과 절차를 사용하여 시스템의 보안 취약점을 찾아내고, 이를 악용하여 침투하는 과정을 시뮬레이션하는 보안 활동입니다. 이는 단순히 취약점을 나열하는 것을 넘어, 해당 취약점이 실제 공격으로 이어질 경우 어떤 피해를 입힐 수 있는지, 그리고 어떤 경로로 침투가 가능한지 등을 종합적으로 평가합니다.

**[모의해킹 지식의 중요성]**
*   **공격자의 시야 확보:** 공격자의 사고방식과 전술을 이해함으로써, 방어자는 더욱 효과적인 방어 전략을 수립할 수 있습니다.
*   **자동화 도구의 한계 보완:** SAST/DAST와 같은 자동화된 도구가 놓칠 수 있는 복합적인 취약점, 논리적 결함, 제로데이 취약점 등을 발견할 수 있습니다.
*   **실제 위협 평가:** 발견된 취약점이 실제 공격으로 이어질 경우의 비즈니스 영향도를 정확히 평가하고, 우선순위에 따른 개선 계획을 수립할 수 있습니다.
*   **보안 시스템 검증:** 방화벽, IDS/IPS, WAF 등 기존 보안 시스템이 실제 공격에 얼마나 효과적으로 대응하는지 검증할 수 있습니다.
*   **AI 기반 모의해킹:** AI는 모의해킹 과정에서 정보 수집, 취약점 분석, 공격 페이로드 생성 등을 자동화하여 모의해킹의 효율성을 극대화할 수 있습니다.

> **보안 전문가에게 모의해킹 실습이란?**
>
> 애플리케이션이라는 **'건물의 보안 취약점을 공격자의 입장에서 직접 시험해보는 실전 훈련'**입니다. AI는 이 훈련 과정에서 **'공격 시나리오 생성 도우미'**이자 **'취약점 분석 엔진'** 역할을 수행하여, 인간 모의해킹 전문가의 역량을 증폭시키고, 방어 시스템의 약점을 더욱 빠르고 정확하게 찾아낼 수 있도록 돕습니다.

## 1. 💥 모의해킹 방법론: PTES (Penetration Testing Execution Standard)

PTES는 모의해킹을 체계적으로 수행하기 위한 표준 방법론입니다. 각 단계는 유기적으로 연결되어 있으며, AI는 각 단계에서 모의해킹 전문가를 지원할 수 있습니다.

```mermaid
graph TD
    A[1. 사전 협의 (Pre-engagement Interactions)] --> B[2. 정보 수집 (Intelligence Gathering)];
    B --> C[3. 위협 모델링 (Threat Modeling)];
    C --> D[4. 취약점 분석 (Vulnerability Analysis)];
    D --> E[5. 침투 (Exploitation)];
    E --> F[6. 후속 조치 (Post Exploitation)];
    F --> G[7. 보고서 작성 (Reporting)];
```

### 1.1. 🕵️ 1. 정보 수집 (Intelligence Gathering)

*   **목표:** 대상 시스템에 대한 공개된 정보(IP 주소, 도메인, 기술 스택, 직원 정보 등)를 최대한 수집합니다.
*   **AI 연동 전략:**
    *   **OSINT (Open Source Intelligence) 자동화:** AI가 웹 크롤링, 검색 엔진, 소셜 미디어 등을 통해 대상 시스템에 대한 공개 정보를 자동으로 수집하고 분석.
    *   **서브도메인/IP 대역 탐색:** AI가 도메인 정보를 기반으로 서브도메인, 관련 IP 대역 등을 자동으로 탐색.

### 1.2. 🗺️ 2. 위협 모델링 (Threat Modeling)

*   **목표:** 수집된 정보를 바탕으로 대상 시스템의 잠재적인 위협과 공격 경로를 식별하고 우선순위를 부여합니다.
*   **AI 연동 전략:**
    *   **공격 경로 예측:** AI가 시스템 아키텍처, 기술 스택, 알려진 취약점 정보를 기반으로 가능한 공격 경로를 예측하고 시나리오 생성.
    *   **위협 평가:** AI가 각 위협의 심각도와 발생 가능성을 평가하여 우선순위 지정.

### 1.3. 🔬 3. 취약점 분석 (Vulnerability Analysis)

*   **목표:** 대상 시스템의 취약점을 식별하고, 해당 취약점이 실제 공격으로 이어질 수 있는지 분석합니다. (SAST/DAST 도구 활용)
*   **AI 연동 전략:**
    *   **자동화된 취약점 스캐닝:** AI가 Nmap, Nessus, OpenVAS 등 취약점 스캐너를 자동화하여 실행하고 결과를 분석.
    *   **제로데이 취약점 탐지 지원:** AI가 코드 패턴, 시스템 동작 등을 분석하여 알려지지 않은 제로데이 취약점의 징후를 탐지.

### 1.4. 💥 4. 침투 (Exploitation)

*   **목표:** 발견된 취약점을 실제로 악용하여 시스템에 침투하고, 권한을 획득합니다.
*   **AI 연동 전략:**
    *   **익스플로잇 코드 생성:** AI가 발견된 취약점 정보(CVE ID, 버전 등)를 기반으로 해당 취약점을 악용할 수 있는 익스플로잇 코드(PoC)를 자동으로 생성하거나 수정.
    *   **페이로드 최적화:** AI가 대상 시스템의 환경에 맞는 최적의 공격 페이로드(Payload)를 생성.
    *   **우회 기법 제안:** AI가 방어 시스템(WAF, IPS)을 우회할 수 있는 공격 기법을 제안.

### 1.5. 🕵️ 5. 후속 조치 (Post Exploitation)

*   **목표:** 시스템 침투 후 추가적인 정보 수집, 권한 상승, 내부망 탐색, 백도어 설치 등 공격자가 수행할 수 있는 모든 행위를 시뮬레이션합니다.
*   **AI 연동 전략:**
    *   **권한 상승 경로 분석:** AI가 시스템 설정, 사용자 권한 등을 분석하여 권한 상승이 가능한 경로를 식별.
    *   **내부망 탐색 자동화:** AI가 침투된 시스템을 거점으로 내부망의 다른 시스템을 자동으로 탐색하고 취약점 분석.

### 1.6. 📄 6. 보고서 작성 (Reporting)

*   **목표:** 모의해킹 과정에서 발견된 모든 취약점, 침투 경로, 비즈니스 영향도, 개선 방안 등을 상세하게 보고서로 작성합니다.
*   **AI 연동 전략:**
    *   **보고서 자동 생성:** AI가 모의해킹 과정에서 수집된 모든 정보(탐지된 취약점, PoC 코드, 스크린샷 등)를 기반으로 보고서 초안을 자동 생성.
    *   **개선 방안 제안:** AI가 발견된 취약점에 대한 시큐어 코딩 가이드라인, 설정 변경 방법 등 구체적인 개선 방안을 제시.

## 2. 🛡️ AI 기반 모의해킹 자동화 및 방어

AI는 모의해킹의 각 단계를 자동화하여 효율성을 높일 뿐만 아니라, 방어자의 입장에서 AI 기반으로 모의해킹 공격을 탐지하고 방어하는 데도 활용될 수 있습니다.

### 2.1. 🤖 AI 기반 공격 표면 관리 (Attack Surface Management)

AI는 기업의 모든 자산(웹 애플리케이션, 클라우드 인프라, 네트워크 장비 등)을 지속적으로 모니터링하고, 새로운 공격 표면을 자동으로 식별하여 모의해킹 대상에 포함합니다.

### 2.2. 🤖 AI 기반 취약점 예측 및 우선순위 지정

AI는 과거의 취약점 데이터, 시스템 설정, 코드 변경 이력 등을 분석하여 잠재적인 취약점을 예측하고, 비즈니스 영향도를 고려하여 모의해킹의 우선순위를 지정합니다.

### 2.3. 🤖 AI 기반 익스플로잇 생성 및 검증

AI는 특정 취약점(CVE)에 대한 정보를 바탕으로 익스플로잇 코드를 자동으로 생성하거나, 기존 익스플로잇 코드를 변형하여 방어 시스템을 우회하는 방법을 찾습니다.

### 2.4. 🛡️ AI 기반 모의해킹 탐지 및 방어

방어자의 입장에서 AI는 모의해킹 공격을 실제 공격과 유사하게 탐지하고 방어하는 데 활용될 수 있습니다.

*   **비정상 행위 탐지:** AI가 모의해킹 도구(Nmap, Burp Suite, Metasploit)에서 발생하는 비정상적인 네트워크 트래픽, 시스템 로그 패턴을 탐지.
*   **공격 패턴 분석:** AI가 모의해킹 공격의 페이로드, 시퀀스 등을 분석하여 알려진 공격 패턴과 비교하거나 새로운 공격 패턴을 학습.
*   **자동 차단:** AI가 모의해킹 공격을 탐지하면, 방화벽, IPS, WAF 등 보안 솔루션에 연동하여 해당 공격을 자동으로 차단.

## 👨‍💻 현직자 통합 시나리오: AI 기반 자동화된 웹 모의해킹 시스템

**[상황]**
AI 보안 엔지니어 '제미니'는 회사의 웹 애플리케이션에 대해 주기적으로 자동화된 모의해킹을 수행하고, 발견된 취약점을 보고하며, WAF에 자동 방어 규칙을 추가하는 시스템을 구축하고자 합니다.

**[워크플로우]**
1.  **정보 수집 및 크롤링:** AI가 대상 웹 애플리케이션을 크롤링하여 모든 URL, 입력 필드, API 엔드포인트를 식별합니다. (Scrapy, Selenium 등 활용)
2.  **AI 기반 취약점 분석 및 침투:**
    *   **SQL Injection:** AI가 식별된 입력 필드에 SQL Injection 페이로드를 주입하고, 응답을 분석하여 취약점을 탐지하고 침투를 시도합니다. (이전 단원의 AI 웹 스캐너 확장)
    *   **XSS:** AI가 XSS 페이로드를 주입하고, 웹 페이지의 HTML 소스 코드에서 스크립트 실행 흔적을 탐지합니다.
    *   **Broken Access Control:** AI가 다른 사용자 역할로 로그인하여 관리자 페이지나 권한 없는 데이터에 접근을 시도합니다.
    *   **SSRF:** AI가 URL 파라미터에 내부 IP 주소나 클라우드 메타데이터 서비스 URL을 주입하여 내부망 접근을 시도합니다.
3.  **AI 기반 후속 조치 (개념):**
    *   침투에 성공하면, AI가 추가적인 정보 수집(예: 파일 시스템 탐색, 데이터베이스 조회)을 시도합니다.
    *   AI가 권한 상승이 가능한 경로를 분석합니다.
4.  **WAF 연동 및 자동 방어:**
    *   AI가 탐지한 취약점에 대해 WAF에 해당 공격 패턴을 차단하는 규칙을 자동으로 추가합니다.
    *   AI가 모의해킹 공격을 수행하는 동안 WAF/IPS 로그를 모니터링하여 방어 시스템의 탐지/차단 능력을 평가합니다.
5.  **보고서 자동 생성 및 알림:**
    *   AI가 모의해킹 과정에서 발견된 모든 취약점, 침투 경로, PoC 코드, 비즈니스 영향도, 개선 방안 등을 포함한 상세 보고서를 자동으로 생성합니다.

# --- 1. AI 기반 자동화된 웹 모의해킹 시스템 (개념) ---
class AIAutomatedPenTester:
    def __init__(self, target_url):
        self.target_url = target_url
        self.vulnerabilities = []
        self.crawled_urls = set()
        self.sql_injection_payloads = ["'", "''", "' OR 1=1 --", "' UNION SELECT null,null,null --"]
        self.xss_payloads = ["<script>alert(1)</script>", "<img src=x onerror=alert(1)>"]
        self.ssrf_payloads = ["http://169.254.169.254/latest/meta-data/", "http://127.0.0.1/admin"]
        
        # AI 모델 (가상): 응답 분석 및 취약점 예측
        self.ai_response_analyzer = RandomForestClassifier(random_state=42)
        X_train_data = [
            [0, 0, 0, 0, 0], # 정상 응답
            [1, 0, 0, 0, 0], # SQLi 오류 메시지
            [0, 1, 0, 0, 0], # XSS 스크립트 포함
            [0, 0, 1, 0, 0], # SSRF 내부 정보 노출
        ]
        y_train_labels = [0, 1, 1, 1]
        self.ai_response_analyzer.fit(X_train_data, y_train_labels)

    def _analyze_response_with_ai(self, response_text, payload_type):
        """AI 모델을 사용하여 응답 텍스트 분석 및 취약점 예측"""
        features = [0, 0, 0, 0, 0]
        if payload_type == 'SQLi' and ("SQL syntax" in response_text or "mysql_fetch_array" in response_text):
            features[0] = 1
        if payload_type == 'XSS' and ("alert(1)" in response_text or "onerror=alert(1)" in response_text):
            features[1] = 1
        if payload_type == 'SSRF' and ("instance-id" in response_text or "admin" in response_text):
            features[2] = 1
        
        prediction = self.ai_response_analyzer.predict([features])[0]
        return prediction == 1

    def crawl_and_find_inputs(self, url):
        """웹 페이지를 크롤링하고 입력 필드를 찾는 가상 함수"""
        if url in self.crawled_urls:
            return []
        self.crawled_urls.add(url)
        
        print(f"🌐 크롤링 중: {url}")
        response = send_web_request(url)
        if not response:
            return []
        
        soup = BeautifulSoup(response.text, 'html.parser')
        input_fields = []
        for form in soup.find_all('form'):
            for input_tag in form.find_all(['input', 'textarea']):
                if input_tag.get('name'):
                    input_fields.append({'url': url, 'param': input_tag.get('name'), 'method': form.get('method', 'GET').upper()})
        
        # URL 파라미터도 입력으로 간주
        parsed_url = requests.utils.urlparse(url)
        query_params = requests.utils.parse_qs(parsed_url.query)
        for param_name in query_params:
            input_fields.append({'url': url, 'param': param_name, 'method': 'GET'})

        # 링크 탐색 (재귀 크롤링)
        for link in soup.find_all('a', href=True):
            full_link = requests.utils.urljoin(url, link['href'])
            if full_link.startswith(self.target_url) and full_link not in self.crawled_urls:
                # 너무 깊게 크롤링하지 않도록 제한
                if len(self.crawled_urls) < 10: 
                    input_fields.extend(self.crawl_and_find_inputs(full_link))
        
        return input_fields

    def run_pentest(self):
        print(f"--- 🚀 AI 기반 자동화된 웹 모의해킹 시작: {self.target_url} ---")
        
        # 1. 정보 수집 및 크롤링
        all_inputs = self.crawl_and_find_inputs(self.target_url)
        print(f"\n🌐 총 {len(all_inputs)}개의 입력 필드/파라미터 탐지.")

        # 2. AI 기반 취약점 분석 및 침투
        for input_info in all_inputs:
            url = input_info['url']
            param = input_info['param']
            method = input_info['method']

            # SQL Injection 스캔
            for payload in self.sql_injection_payloads:
                data = {param: payload}
                response = send_web_request(url, method=method, data=data)
                if response and self._analyze_response_with_ai(response.text, 'SQLi'):
                    self.vulnerabilities.append(f"💥 SQL Injection 탐지: {url} (Param: {param}, Payload: {payload})")
                    add_waf_rule('SQLi', payload)
                    break # 다음 페이로드로 넘어가지 않고 다음 입력 필드로

            # XSS 스캔
            for payload in self.xss_payloads:
                data = {param: payload}
                response = send_web_request(url, method=method, data=data)
                if response and self._analyze_response_with_ai(response.text, 'XSS'):
                    self.vulnerabilities.append(f"💥 XSS 탐지: {url} (Param: {param}, Payload: {payload})")
                    add_waf_rule('XSS', payload)
                    break

            # SSRF 스캔 (URL 파라미터에만 적용)
            if param == 'url' or 'redirect' in param: # URL을 받는 파라미터 가정
                for payload in self.ssrf_payloads:
                    data = {param: payload}
                    response = send_web_request(url, method=method, data=data)
                    if response and response.status_code == 200 and self._analyze_response_with_ai(response.text, 'SSRF'):
                        self.vulnerabilities.append(f"💥 SSRF 탐지: {url} (Param: {param}, Payload: {payload})")
                        add_waf_rule('SSRF', payload)
                        break
            time.sleep(0.1) # 요청 간 지연

        print("\n--- 📄 모의해킹 결과 보고서 ---")
        if self.vulnerabilities:
            for i, vuln in enumerate(self.vulnerabilities):
                print(f"  - {i+1}. {vuln}")
        else:
            print("  - 웹 취약점이 탐지되지 않았습니다.")
        
        print("\n--- 🛡️ WAF 연동 결과 ---")
        print("탐지된 취약점에 대한 WAF 규칙이 자동으로 추가되었습니다.")
