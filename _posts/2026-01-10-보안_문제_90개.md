# 보안 문제 90개 - 정답 및 해설

## 1. 정보보호 정책에 반드시 포함되어야 할 내용으로 가장 적절한 것은 무엇인가요?

1. 보안 솔루션의 상세 기술 사양
2. 개별 시스템의 소스 코드
3. 정보보호의 목적, 범위, 책임 및 역할
4. 침해사고 발생 시 언론 대응 시나리오

**정답: 3번**

**해설:** 정보보호 정책은 최고경영자의 의지가 반영되어야 하며, 목적·적용 범위·책임과 역할을 명확히 정의해야 한다.


## 2. 다음 중 정보보호관리체계의 PDCA(Plan-Do-Check-Act) 사이클에서 "Check" 단계에 해당하는 활동은 무엇인가요?

1. 내부 감사 및 관리체계 검토
2. 정보보호 정책 수립
3. 위험 처리 대책 실행
4. 개선 대책 이행

**정답: 1번**

**해설:** Check 단계는 수립·운영된 관리체계를 점검하고 평가하는 단계로, 내부 감사와 성과 검토가 포함된다.


## 3. 인력 변동 발생 시 조치해야 할 사항으로 옳은 것은 무엇인가요?

1. 퇴직자가 발생하더라도 업무 인수인계를 위해 기존 계정과 비밀번호를 일정 기간 유지한다.
2. 부서 이동이나 퇴직 시 지체 없이 정보자산 반납, 계정 및 접근권한 회수·조정 절차를 이행한다.
3. 퇴직 예정자의 정보자산 반납 확인은 본인의 구두 보고로 대체하며 별도의 기록은 남기지 않는다.
4. 공유 계정은 계정 관리 대상이 아니므로 별도로 조치할 사항은 없다.

**정답: 2번**

**해설:** 인력 변동 시 지체 없이 자산 반납 및 접근권한 회수·조정 절차를 수립·이행해야 한다.


## 4. 다음 중 물리적 보안에 해당하는 것은 무엇인가요?

1. 방화벽 정책 설정
2. 서버실 출입 통제 및 출입 기록 관리
3. 데이터베이스 접근 권한 관리
4. 웹 애플리케이션 취약점 점검

**정답: 2번**

**해설:** 물리적 보안은 서버실, 전산실 등의 출입 통제·감시·보호를 통해 물리적 위협을 방지하는 영역이다.


## 5. 다음 중 ISMS의 접근통제 요구사항에 해당하지 않는 것은 무엇인가요?

1. 사용자 식별 및 인증
2. 접근 권한의 부여·변경·말소 관리
3. 접근통제 정책 및 절차 수립
4. 개인정보 파기 절차 수립

**정답: 4번**

**해설:** 개인정보 파기는 개인정보 보호 영역(3영역)에 해당하며, 접근통제는 사용자 인증과 권한 관리가 핵심이다.


## 6. 다음 중 암호화(Encryption) 적용이 가장 직접적으로 요구되는 대상은 무엇인가요?

1. 회사 연혁 자료
2. 백업 완료 로그
3. 주민등록번호, 계좌번호
4. 시스템 성능 모니터링 지표

**정답: 3번**

**해설:** 중요 정보 및 민감 정보는 저장·전송 시 암호화가 요구되며, 특히 개인정보나 인증정보는 필수 보호 대상이다.


## 7. 개발 환경의 보안 설정에 대한 설명으로 옳은 것은 무엇인가요?

1. 개발서버와 테스트서버는 운영 네트워크와 분리하여 운영한다.
2. 개발 서버에는 별도의 접근통제 정책을 적용할 필요가 없다.
3. 개발 업무의 효율성을 위해 운영망과 동일한 네트워크를 사용한다.
4. 외부 개발자가 사용하는 PC는 업무망에 직접 연결한다.

**정답: 1번**

**해설:** 개발 중인 소스코드나 취약점이 운영 환경에 영향을 미치지 않도록 분리하는 것이 원칙입니다.


## 8. 악성코드 방지 대책으로 가장 적절하지 않은 것은 무엇인가요?

1. 백신 프로그램 설치 및 최신 상태 유지
2. 외부 저장매체 사용 통제
3. 사용자 계정의 패스워드 복잡성 강화
4. 의심 파일 실행 제한

**정답: 3번**

**해설:** 패스워드 복잡성은 인증·접근통제 영역에 해당하며, 악성코드 방지는 실행 통제·백신·매체 관리가 핵심이다.


## 9. 기업 규모가 작아 서버팜(Server Farm)과 DB팜을 물리적으로 구분하기 어려운 경우, 권고하는 보안 대책은 무엇인가요?

1. 네트워크 분리 대신 암호 복잡도만 강화한다.
2. 위험평가 결과를 기반으로 호스트 기반 접근통제 등을 적용한다.
3. 추가 장비를 구매할 때까지 보안 대책을 유예한다.
4. 별도의 조치 없이 모든 네트워크를 통합하여 운영한다.

**정답: 2번**

**해설:** 물리적 분리가 어려울 때 서버 자체에서 접근을 제어하는 방식(Host-based ACL 등)이 대안이 될 수 있다.


## 10. 정보보호 관리체계 수립 시 수행하는 위험 관리(Risk Management) 활동에 해당하지 않는 것은 무엇인가요?

1. 정보자산 식별
2. 위협 및 취약점 분석
3. 위험 평가 및 처리 계획 수립
4. 개인정보 암호화 알고리즘 적용

**정답: 4번**

**해설:** 위험 관리는 자산·위협·취약점을 기반으로 위험을 평가하고 관리하는 활동이며, 암호화 알고리즘 적용은 기술적 통제 구현 단계에 해당한다.


## 11. ISMS 2영역(보호대책 요구사항)의 주요 목적으로 가장 적절한 것은 무엇인가요?

1. 정보보호 관련 문서의 구비
2. 정보시스템에 대한 기술적 보안 솔루션 설치
3. 식별된 위험을 허용 가능한 수준으로 낮추기 위한 보호대책 구현
4. 침해사고 발생 후 법적 분쟁 대응

**정답: 3번**

**해설:** 2영역은 1영역에서 도출된 위험을 기반으로 관리적·기술적·물리적 보호대책을 구현하여 위험을 감소시키는 것이 목적이다.


## 12. 다음 중 가장 적절한 계정 관리 방식은 무엇인가요?

1. 모든 관리자는 공용 admin 계정을 사용한다
2. 퇴직자 계정은 6개월 후 일괄 삭제한다
3. 개인별 계정을 사용하고 권한 변경 이력을 기록한다
4. 계정 생성·삭제는 시스템 관리자의 구두 승인으로 처리한다

**정답: 3번**

**해설:** 개인별 계정 사용, 권한 최소화, 변경 이력 관리를 통해 책임 추적성을 확보하여야 한다.


## 13. 개인정보 처리 시스템의 접근권한 관리에 대한 설명으로 가장 부적절한 것은 무엇인가요?

1. 개인정보 처리자별 접근 권한을 차등 부여한다
2. 퇴직자의 접근 권한은 즉시 말소한다
3. 업무상 필요하므로 모든 직원에게 전체 접근 권한을 부여한다
4. 접근 권한 부여·변경·말소 내역을 기록한다

**정답: 3번**

**해설:** 개인정보 접근권한은 최소 권한 원칙에 따라 부여되어야 하며, 전사적 전체 권한 부여는 중대한 개인정보보호법 위반사항이다.


## 14. 방화벽 정책 설정 시 '기본 정책(Default Policy)'으로 가장 권장되는 방식은 무엇인가요?

1. 모든 통신을 허용하고 위험한 포트만 차단한다 (Permit All).
2. 사용자가 요청하는 모든 포트를 실시간으로 개방한다.
3. 내부망 간의 통신은 별도의 정책 없이 모두 허용한다.
4. 모든 통신을 차단하고 허용된 서비스만 승인한다 (Deny All).

**정답: 4번**

**해설:** 최소 권한 원칙에 따라 명시적으로 허용된 것 외에는 모두 차단하는 것이 보안의 기본 아이디어이다.


## 15. 개인정보 등 중요 정보가 저장된 데이터베이스가 위치한 'DB Zone'의 네트워크 분리 원칙으로 가장 올바른 것은 무엇인가요?

1. 일반 사용자 네트워크와 대역만 나누어 관리한다.
2. 다른 네트워크 영역과 논리적 또는 물리적으로 분리한다.
3. 인터넷 구간과만 분리되어 있으면 내부망과는 자유롭게 통신한다.
4. 운영자 환경과 동일한 네트워크 대역에 배치한다.

**정답: 2번**

**해설:** DB존은 중요 정보 보호를 위해 다른 네트워크 영역과의 분리하여 독립성을 보장하여야 한다.


## 16. ISMS 심사 시 네트워크 접근통제가 미흡하다고 판단될 수 있는 상황은 무엇인가요?

1. 개발망과 운영망 사이에 방화벽이 설치되어 특정 포트만 허용되어 있다.
2. 외부 인력이 사용하는 PC에서 내부 개발 서버에 직접 접근이 가능하다.
3. DB 관리자가 전용 관리망을 통해서만 DB에 접속한다.
4. 네트워크 영역별로 별도의 VLAN을 할당하여 관리한다.

**정답: 2번**

**해설:** 외부자 영역과 내부망이 제대로 분리되지 않은 상태이므로 지침 위반에 해당한다.


## 17. 다음 중 비대칭키 암호 사용의 잘못된 사례는?

1. 서버 인증서 검증
2. 대칭키 암호화
3. 대용량 파일 직접 암호화
4. 키 교환

**정답: 3번**

**해설:** 대칭키에 비해 느린 비대칭키로 대용량 데이터를 직접 암호화하는 것은 성능 저하를 발생 시킨다. 비대칭키의 목적은 암호화가 아니다.


## 18. 다음 중 전자서명의 목적으로 가장 적절한 것은?

1. 무결성 및 부인 방지
2. 성능 향상
3. 데이터 압축
4. 기밀성 확보

**정답: 1번**

**해설:** 전자서명은 비대칭키를 활용한 기술로 무결성·인증·부인 방지가 목적이다.


## 19. 다음 중 로그 및 접속 기록 관리에 대한 설명으로 가장 적절한 것은 무엇인가요?

1. 모든 로그는 1일 이내 삭제해야 한다
2. 로그는 침해사고 분석 및 추적을 위해 보호·보관되어야 한다
3. 로그는 관리자 편의를 위해 자유롭게 수정 가능하다
4. 로그 관리는 선택 사항이다

**정답: 2번**

**해설:** 접속 기록 및 보안 로그는 위·변조 방지, 접근 통제, 일정 기간 보관이 요구되며 침해사고 대응의 핵심 근거가 된다.


## 20. 공개서버(웹서버 등)를 안전하게 운영하기 위한 네트워크 구성 원칙으로 가장 올바른 것은 무엇인가요?

1. 관리가 용이하도록 내부 업무망의 데이터베이스(DB)와 동일한 영역에 설치한다.
2. 외부 네트워크에 공개되는 서버는 내부 네트워크와 분리된 DMZ 영역에 설치한다.
3. 공개서버를 내부 네트워크에 직접(Direct) 연결하여 속도를 최적화한다.
4. 모든 서버를 하나의 통합 네트워크 대역(VLAN)으로 묶어 운영한다.

**정답: 2번**

**해설:** 공개서버는 내부 네트워크와 분리된 DMZ 영역에 설치하고 침입차단시스템 등으로 보호해야 한다.


## 21. 다음 중 보안 패치 관리의 올바른 절차는 무엇인가요?

1. 패치는 시스템 중단이 발생할 수 있으므로 가급적 적용하지 않는다
2. 모든 패치는 무조건 적용한다
3. 중요도 평가 후 시험 적용을 거쳐 운영 시스템에 반영한다
4. 주기적(주 1회)으로 백신 업데이트를 수행한다.

**정답: 3번**

**해설:** 보안 패치는 중요도·영향도 평가 → 테스트 → 운영 반영의 절차를 통해 안정성과 보안을 동시에 확보해야 한다.


## 22. 클라우드 환경에서 기술적 보호대책으로 가장 적절한 것은 무엇인가요?

1. 모든 리소스를 Public으로 설정
2. IAM 역할 기반 접근 통제 적용
3. 접근 로그 수집 후 3개월 보관
4. 사용자별 멀티 계정 허용

**정답: 2번**

**해설:** 클라우드 환경에서 IAM 기반 최소 권한 통제 및 로그 관리는 보안 필수사항이다.


## 23. 다음 중 침해사고 예방 및 대응 방법으로 가장 적절한 것은 무엇인가요?

1. 침해사고 발생 시 유출 사실이 알려지지 않도록 비밀리에 처리한다.
2. 침해사고 대응 절차 및 비상연락체계를 사전에 수립한다
3. 침해사고는 외부 감사 이후에 분석한다
4. 침해사고 대응은 IT 부서의 재량 사항이다

**정답: 2번**

**해설:** 침해사고에 대비하여 사전 대응 절차, 역할, 보고 체계를 문서화하고 주기적으로 점검하여야 한다.


## 24. 다음 중 신규 개발자(dev7)에게 MySQL 계정을 생성할 때 가장 적절한 권한 부여 방식은 무엇인가요?

1. GRANT ALL PRIVILEGES ON *.* TO 'dev7'@'%';

## 74. XSS와 CSRF의 차이로 옳지 않은 것은?

2. XSS는 클라이언트의 정보 탈취, 악성 페이지 리다이렉션 등이 목적이지만, CSRF는 공격 대상이 공격자가 원하는 행위를 수행하도록 유도하는 것이 목적이다.
3. XSS와 CSRF 둘다 스크립트를 실행할 수 없으면 공격이 이루어지지 않는다.
4. XSS의 보안 대책 관점은 스크립트 실행 방지 및 데이터 입출력 시 스크립트 포함 여부의 확인에 있으며, CSRF의 보안 대책 관점은 사용자의 요청이 실제 사용자가 보낸 것이 맞는 지, 공격자에 의해 유도된 요청인지 검증하는 데 있다.

**정답: 3번**

**해설:** CSRF 취약점의 경우는 스크립트 동작 여부와 무관함

**해설:** order by  절 뒤에 Prepared Statement 객체를 쓰면 에러가 발생한다. Prepared Statement 객체에서 바인딩하는 부분은 칼럼이 아니기 때문에, order by 절 뒤에 컬럼이나 asc/desc 예야어를 쓰면 에러가 발생하게 된다.


## 77. XSS의 보안 대책이 아닌 것을 선택하세요.

1. <, >, ', ", ;, (, ) 등 특수문자 입력 시, 서버측에서 특수문자를 제거한 뒤 필요한 문자만 허용 해 주는 화이트리스트 필터링을 적용한다.
2. 태그 사용이 필요하여 화이트리스트 필터링에 어려움이 있는 경우, 고객사와 협의하여 위험성이 있는 특수문자, 문자열, 태그, 이벤트 핸들러 등에 대하여 필터링 하는 블랙리스트 필터링을 최대한으로 적용한다.
3. 사용자가 입력 한 정보를 대칭키를 이용하여 암호화 한 뒤 저장한다.
4. <, >, ', " 등 주요 특수문자에 대한 화이트리스트 필터링을 적용한 뒤, 자주 사용하는 스크립트 구문을 필터링 하는 블랙리스트 필터링을 적용한다.

**정답: 3번**

**해설:** 3번은 XSS에 대한 보안 대책이 아니다.

## 79. 다음중 SQL Injection시 참고할 내용으로 올바르지 않은 것은?
