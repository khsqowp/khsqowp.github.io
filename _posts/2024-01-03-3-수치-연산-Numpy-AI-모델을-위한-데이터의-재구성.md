# 🔢 3. 수치 연산 (Numpy): AI 모델을 위한 데이터의 재구성

## 🔄 이전 단원 복습: 보안 관점에서의 Pandas 데이터 핸들링

이전 단원 `2. 데이터 핸들링 (Pandas).md`에서 우리는 Pandas를 사용하여 흩어져 있는 텍스트 로그를 구조화된 `DataFrame`으로 변환하고, 의미 있는 보안 인사이트를 추출하는 방법을 배웠습니다.

> **핵심 복습:**
>
> *   **DataFrame은 작업대:** Pandas의 DataFrame은 비정형 데이터를 분석 가능한 형태로 올려놓는 핵심 작업 공간입니다.
> *   **데이터 위생의 중요성:** `fillna()`, `astype()`, `to_datetime()` 등을 통해 지저분한 데이터를 정제해야만 분석의 신뢰성이 보장됩니다.
> *   **분석과 보강:** 필터링, 그룹화(`groupby`)를 통해 이상 징후를 탐지하고, `merge`를 통해 외부 위협 인텔리전스를 결합하여 데이터의 가치를 높였습니다.
>
> 이제 우리는 깨끗하게 정제되고 의미가 부여된 데이터를 확보했습니다. 하지만 AI 모델(특히 머신러닝/딥러닝 모델)은 '의미'를 직접 이해하지 못합니다. 모델이 이해할 수 있는 유일한 언어는 **'숫자'** 입니다. 오늘 배울 Numpy는 바로 이 '의미'의 세계와 '숫자'의 세계를 잇는 다리 역할을 합니다.

## 🤔 Numpy를 왜 배워야 할까요? (The "Why")

Pandas로 아무리 데이터를 잘 분석했더라도, 그 결과는 여전히 테이블 형태의 '표'일 뿐입니다. 머신러닝 모델에게 "이 IP는 10번 공격했으니 악성이야"라고 말해도 알아듣지 못합니다. 모델은 오직 `[10, 1]` 과 같은 숫자들의 배열(Array) 또는 행렬(Matrix)만을 입력으로 받기 때문입니다.

**[Numpy 이전의 문제점]**
*   **느린 연산 속도:** 파이썬의 기본 `list`는 유연하지만, 대량의 숫자 데이터를 연산(더하기, 곱하기 등)할 때 매우 느립니다. `for` 루프를 사용해야 하므로, 데이터가 커질수록 연산 시간은 기하급수적으로 늘어납니다.
*   **복잡한 수학 연산:** 행렬 곱, 역행렬, 푸리에 변환 등 복잡한 선형대수 연산을 파이썬 `list`로 직접 구현하는 것은 매우 어렵고 비효율적입니다.
*   **메모리 비효율:** 파이썬 `list`는 다양한 데이터 타입을 저장할 수 있는 만큼, 동일한 타입의 숫자만 저장할 때에도 부가적인 정보(메타데이터)를 많이 사용하여 메모리를 비효율적으로 사용합니다.

**[Numpy의 해결책]**
Numpy(Numerical Python)는 파이썬에서 과학 계산을 위한 핵심 라이브러리로, `ndarray`(N-dimensional array)라는 강력한 다차원 배열 객체를 제공합니다.

> **보안 전문가에게 Numpy란?**
>
> Pandas로 정제한 공격 증거(IP, 포트, 행위 패턴 등)들을 머신러닝 모델이라는 **'AI 분석관'**에게 넘겨주기 위한 **'사건 파일(숫자 행렬)'**로 변환하는 필수 도구입니다. Numpy를 통해 우리는 텍스트 기반의 로그를 AI가 학습하고 예측할 수 있는 고차원의 숫자 벡터(Vector)로 재구성할 수 있습니다.

## 1. 🎯 Numpy의 핵심: `ndarray`

`ndarray`는 동일한 타입의 데이터를 가지는 다차원 배열입니다. 이 '동일한 타입'이라는 제약 덕분에 C언어로 구현된 내부 코드가 매우 빠른 속도로 데이터를 처리할 수 있습니다.

### 1.1. 🔢 `ndarray` 생성과 속성

```python
import numpy as np

# 파이썬 리스트로부터 ndarray 생성
list_data = [1, 2, 3, 4, 5]
np_array_1d = np.array(list_data)

print("--- 1차원 배열 ---")
print(np_array_1d)
print(f"  - 형태(Shape): {np_array_1d.shape}") # (5,) -> 5개의 원소를 가진 1차원 배열
print(f"  - 차원(Dimension): {np_array_1d.ndim}") # 1
print(f"  - 데이터 타입(Dtype): {np_array_1d.dtype}") # int64

# 2차원 리스트로부터 ndarray(행렬) 생성
matrix_data = [[1, 2, 3], [4, 5, 6]]
np_matrix = np.array(matrix_data)

print("\n--- 2차원 배열 (행렬) ---")
print(np_matrix)
print(f"  - 형태(Shape): {np_matrix.shape}") # (2, 3) -> 2행 3열
print(f"  - 차원(Dimension): {np_matrix.ndim}") # 2
print(f"  - 데이터 타입(Dtype): {np_matrix.dtype}") # int64
```

### 1.2. ⚡️ 벡터화 연산 (Vectorized Operation)

Numpy의 가장 강력한 기능 중 하나는 `for` 루프 없이 배열의 모든 요소에 대해 한 번에 연산을 수행하는 **벡터화**입니다. 이는 연산 속도를 극적으로 향상시킵니다.

#### **PoC: 파이썬 `list` vs Numpy `ndarray` 속도 비교**

```python
import numpy as np
import time

# 100만개의 숫자를 가진 데이터 생성
data_size = 1_000_000
python_list = list(range(data_size))
numpy_array = np.arange(data_size)

# --- 파이썬 list를 사용한 연산 (for 루프) ---
start_time = time.time()
result_list = [x * 2 for x in python_list]
end_time = time.time()
print(f"🐍 파이썬 list 연산 시간: {end_time - start_time:.6f} 초")

# --- Numpy를 사용한 벡터화 연산 ---
start_time = time.time()
result_array = numpy_array * 2
end_time = time.time()
print(f"🔢 Numpy 벡터화 연산 시간: {end_time - start_time:.6f} 초")
```
**[실행 결과 예시]**
```
🐍 파이썬 list 연산 시간: 0.045123 초
🔢 Numpy 벡터화 연산 시간: 0.001521 초
```
**[분석]**
Numpy가 파이썬 `list`에 비해 수십 배 이상 빠른 것을 확인할 수 있습니다. 수백만 개의 보안 이벤트를 처리하거나 복잡한 AI 모델을 학습시킬 때, 이 속도 차이는 분석의 성패를 좌우합니다.

## 2. 🎯 데이터 재구성: 보안 로그를 숫자 행렬로

보안 분석의 궁극적인 목표 중 하나는 '정상'과 '악성' 행위를 구분하는 AI 모델을 만드는 것입니다. 이를 위해 로그 데이터를 모델이 학습할 수 있는 숫자 행렬(Feature Matrix)로 변환해야 합니다.

### 2.1. 🔬 Feature Engineering: 특징 추출

> **용어 해설: Feature Engineering**
>
> 원본 데이터(Raw Data)로부터 모델의 성능을 향상시킬 수 있는 새로운 특징(Feature)을 만들어내는 과정입니다. 예를 들어, 로그의 '시간' 정보 자체보다는 '특정 시간대(e.g., 새벽)에 발생한 이벤트 횟수'가 더 중요한 특징이 될 수 있습니다.

#### **PoC: Pandas와 Numpy를 연계한 특징 행렬 생성**

이전 단원에서 사용한 인증 로그를 AI 모델의 입력값으로 변환해 보겠습니다.

```python
import pandas as pd
import numpy as np
import io

# 가상 인증 로그
log_content = """timestamp,source_ip,user,status
2023-11-12 11:00:01,1.2.3.4,admin,SUCCESS
2023-11-12 11:00:05,5.6.7.8,rookies,FAILURE
2023-11-12 11:00:06,5.6.7.8,rookies,FAILURE
2023-11-12 11:00:08,5.6.7.8,admin,FAILURE
2023-11-12 11:01:20,1.2.3.4,guest,SUCCESS
"""
auth_df = pd.read_csv(io.StringIO(log_content))
auth_df['timestamp'] = pd.to_datetime(auth_df['timestamp'])

# --- 1. Feature Engineering (Pandas) ---
# 시간 관련 특징 추출
auth_df['hour'] = auth_df['timestamp'].dt.hour
auth_df['minute'] = auth_df['timestamp'].dt.minute

# 문자열 데이터를 숫자 데이터로 변환 (Label Encoding)
# ⚠️ 보안 관점: 이 방식은 'admin'과 'guest' 사이에 수치적 관계(e.g., admin < guest)가 없는데도
# 모델이 그렇게 오해할 소지가 있습니다. 더 정교한 방법(e.g., One-Hot Encoding)이 필요할 수 있습니다.
auth_df['user_code'] = pd.factorize(auth_df['user'])[0]

# 'status' 열을 0(FAILURE)과 1(SUCCESS)로 변환 (이것이 우리가 예측할 목표값(Target)이 됨)
auth_df['target'] = (auth_df['status'] == 'SUCCESS').astype(int)

print("--- 📊 특징이 추가된 DataFrame ---")
print(auth_df[['hour', 'minute', 'user_code', 'target']])

# --- 2. Numpy 배열로 변환 ---
# 모델의 입력으로 사용할 특징(Feature)들만 선택
features = ['hour', 'minute', 'user_code']
X = auth_df[features].values # .values 속성은 DataFrame을 Numpy 배열로 변환

# 모델이 예측해야 할 목표값(Target)
y = auth_df['target'].values

print("\n--- 🔢 AI 모델 입력용 특징 행렬 (X) ---")
print(type(X))
print(X)

print("\n--- 🔢 AI 모델 목표값 벡터 (y) ---")
print(type(y))
print(y)
```
**[분석]**
`auth_df`라는 Pandas DataFrame이 `X`라는 **특징 행렬(Feature Matrix)**과 `y`라는 **목표 벡터(Target Vector)**로 완벽하게 변환되었습니다. 이제 이 `X`와 `y`를 머신러닝 모델(예: `Scikit-learn`)에 전달하여 "어떤 특징(시간, 사용자)을 가진 로그가 성공(1)이고 실패(0)인지"를 학습시킬 수 있습니다.

## 3. 🎯 브로드캐스팅(Broadcasting): 차원이 다른 배열 간의 연산

Numpy의 브로드캐스팅은 서로 다른 형태(shape)를 가진 배열 간의 연산을 가능하게 하는 매우 강력한 기능입니다.

> **보안 시나리오:**
>
> 수많은 IP 주소들의 트래픽 양(bytes_sent)을 담은 배열이 있다고 가정해 봅시다. 이때 특정 임계값(Threshold)을 초과하는 IP를 모두 탐지하고 싶을 때, `for` 루프 없이 브로드캐스팅을 사용하여 한 번에 처리할 수 있습니다.

#### **PoC: 임계값 기반 이상 트래픽 탐지**

```python
import numpy as np

# 각기 다른 10개 IP의 트래픽 양(bytes)을 담은 배열
# (실제로는 Pandas로 집계한 결과를 .values로 변환하여 얻음)
traffic_bytes = np.array([1024, 512, 81920, 2048, 128, 99840, 4096, 51200, 3072, 64])

# 임계값 설정 (50KB)
threshold = 50 * 1024 # 51200

# 브로드캐스팅을 이용한 임계값 초과 여부 검사
# traffic_bytes (shape: (10,)) 와 threshold (shape: 스칼라) 간의 연산
is_over_threshold = traffic_bytes > threshold

print("--- 📡 IP별 트래픽 양 (bytes) ---")
print(traffic_bytes)
print(f"\n--- 🚨 임계값 ({threshold} bytes) ---")

print("\n--- 🔬 임계값 초과 여부 (True/False) ---")
print(is_over_threshold)

# Boolean Indexing: True인 위치의 원본 데이터만 추출
suspicious_traffic = traffic_bytes[is_over_threshold]
print("\n--- 💥 이상 트래픽으로 의심되는 트래픽 양 ---")
print(suspicious_traffic)
```
**[분석]**
`traffic_bytes > threshold` 연산 시, 스칼라 값인 `threshold`가 `traffic_bytes` 배열의 모든 요소에 **'전파(broadcast)'**되어 각각 비교 연산을 수행했습니다. 그 결과, `for` 루프 없이도 단 한 줄의 코드로 임계값을 초과하는 모든 트래픽(`[81920, 99840, 51200]`)을 효율적으로 찾아냈습니다.

## 👨‍💻 현직자 통합 시나리오: AI 보안 엔지니어의 악성코드 특징 벡터화

**[상황]**
AI 보안 엔지니어 '제미니'는 악성코드와 정상 파일을 분류하는 머신러닝 모델을 개발 중입니다. 모델을 학습시키려면, 각 파일로부터 추출한 특징들을 숫자 벡터로 만들어야 합니다.

**[추출된 특징]**
*   파일 크기 (bytes)
*   PE(Portable Executable) 헤더에 포함된 `.text` 섹션의 엔트로피(Entropy)
    > **용어 해설: 엔트로피(Entropy)**
    >
    > 데이터의 무작위성을 측정하는 지표. 일반적으로 암호화되거나 압축된 데이터는 엔트로피가 높게 나타납니다. 악성코드는 탐지를 피하기 위해 코드를 압축(Packing)하는 경우가 많아 정상 파일보다 엔트로피가 높은 경향이 있습니다.
*   파일 내에 'CreateRemoteThread'와 같은 의심스러운 API 함수 이름이 포함되어 있는지 여부 (1 또는 0)

```python
import pandas as pd
import numpy as np

# Pandas를 사용하여 분석된 파일 특징 데이터 생성
data = {
    'filename': ['virus.exe', 'calc.exe', 'malware.dll', 'kernel32.dll'],
    'file_size': [307200, 22528, 153600, 98765],
    'text_entropy': [7.8, 6.1, 7.5, 6.3],
    'has_suspicious_api': [1, 0, 1, 0],
    'label': ['malicious', 'benign', 'malicious', 'benign'] # 정답 레이블
}
file_features_df = pd.DataFrame(data)

print("--- 🐼 Pandas로 정리된 파일 특징 ---")
print(file_features_df)

# --- 데이터 정규화 (Normalization) ---
# file_size와 같이 값의 범위가 매우 다른 특징이 있으면 모델 학습에 악영향을 줄 수 있음.
# 모든 특징의 값 범위를 0과 1 사이로 조정하는 'Min-Max Scaling' 적용

# Numpy를 사용한 수동 스케일링
size_col = file_features_df['file_size'].values
scaled_size = (size_col - np.min(size_col)) / (np.max(size_col) - np.min(size_col))

entropy_col = file_features_df['text_entropy'].values
scaled_entropy = (entropy_col - np.min(entropy_col)) / (np.max(entropy_col) - np.min(entropy_col))

# --- 최종 특징 행렬 생성 ---
# 스케일링된 데이터와 기존 데이터를 Numpy로 결합
X_features = np.array([
    scaled_size,
    scaled_entropy,
    file_features_df['has_suspicious_api'].values
]).T # .T는 행과 열을 바꾸는 전치(Transpose) 연산

# 레이블도 숫자로 변환
y_labels = np.array([1 if label == 'malicious' else 0 for label in file_features_df['label']])

print("\n" + "="*50)
print("🔢 최종적으로 AI 모델에 입력될 특징 행렬 (X_features)")
print("="*50)
print(X_features)
print(f"\n형태(Shape): {X_features.shape}") # 4개의 파일, 3개의 특징

print("\n" + "="*50)
print("🎯 최종적으로 AI 모델이 예측할 목표 벡터 (y_labels)")
print("="*50)
print(y_labels)
print(f"\n형태(Shape): {y_labels.shape}")
```
**[시나리오 분석]**
'제미니' 엔지니어는 Pandas로 정리된 데이터를 가져와, Numpy의 강력한 수치 연산 능력을 활용하여 **정규화(Normalization)**를 수행하고, 최종적으로 `(4, 3)` 형태의 **특징 행렬**을 성공적으로 만들어냈습니다. 이 숫자 행렬은 이제 악성코드를 탐지하는 AI 모델을 훈련시키는 데 사용될 준비가 되었습니다.

## 📌 요약 정리 (Executive Summary)

1.  **Numpy의 역할**: AI 모델이 이해할 수 있는 유일한 언어인 **'숫자 배열'**을 다루기 위한 핵심 라이브러리이다. Pandas가 '의미'를 다룬다면, Numpy는 '수치'를 다룬다.
2.  **`ndarray`와 벡터화**: Numpy의 핵심 객체인 `ndarray`는 동일 타입의 데이터로 구성되어 C 기반의 빠른 연산을 가능하게 한다. `for` 루프 없이 연산하는 **벡터화(Vectorization)**는 파이썬 `list` 대비 수십 배 이상의 속도 향상을 제공한다.
3.  **데이터의 숫자 변환**:
    *   **Feature Engineering**: 원본 데이터로부터 모델 학습에 도움이 될 새로운 숫자 특징을 만들어내는 과정이다.
    *   **`.values`**: Pandas DataFrame을 Numpy `ndarray`로 변환하는 가장 간단한 방법이다.
4.  **핵심 연산**:
    *   **브로드캐스팅(Broadcasting)**: 형태가 다른 배열 간의 연산을 가능하게 하여, 임계값 기반의 이상 탐지 등을 효율적으로 수행할 수 있다.
    *   **정규화/스케일링(Normalization/Scaling)**: 값의 범위가 다른 여러 특징들을 0과 1 사이의 동일한 범위로 조정하여 모델의 학습 안정성을 높인다.
5.  **보안 AI 워크플로우**: **[로그(Text) -> Pandas(DataFrame) -> Numpy(ndarray)]** 의 흐름은 텍스트 기반의 보안 데이터를 AI가 학습할 수 있는 숫자 행렬로 변환하는 표준적인 파이프라인이다.
