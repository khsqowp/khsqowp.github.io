# 🦠 1. 악성코드 분석 방법론: AI 기반 위협 인텔리전스의 시작

## 🔄 이전 단원 복습: LLM 기반 로그 질의 응답

이전 단원 `4. 자연어 질의 응답.md`에서 우리는 AI 기반 로그 분석의 중요한 응용 분야인 자연어 처리(NLP) 기반 로그 질의 응답 시스템에 대해 학습했습니다. LLM(Large Language Models)을 활용하여 비정형 로그 데이터를 자연어 질문으로 분석하고, 사용자에게 직관적인 답변을 제공하며, 보안 분석가의 업무 효율성을 극대화하는 방법을 배웠습니다.

> **핵심 복습:**
>
> *   **자연어 질의 응답의 가치:** 복잡한 쿼리 언어 없이도 로그 데이터에 쉽게 접근하여 필요한 정보를 얻고, 보안 분석가의 업무 효율성을 혁신적으로 높임.
> *   **LLM의 역할:** 자연어 질문 이해, 로그 검색 및 분석, 답변 생성, 쿼리/스크립트 생성 등.
> *   **보안 고려사항:** 민감 정보 유출 위험(마스킹/익명화), 환각(Hallucination) 현상(RAG, 답변 검증), 프롬프트 인젝션(필터링, 시스템 프롬프트 강화) 등.
>
> AI 기반 보안 관제 및 로그 분석은 시스템 전반의 위협을 탐지하고 대응하는 데 매우 효과적입니다. 하지만 특정 위협, 즉 **악성코드(Malware)**는 그 자체로 고도의 분석과 전문적인 대응을 요구합니다. 악성코드는 끊임없이 진화하며, 그 종류와 감염 방식, 행위 패턴이 매우 다양합니다. 오늘부터 시작하는 `3-2. 악성코드 분석 및 대응` 파트에서는 AI 기반으로 악성코드의 특징을 추출하고, 분류하며, 행위를 분석하고, 자동화된 대응을 수행하는 방법을 학습합니다. 그 첫걸음은 바로 **악성코드 분석 방법론**입니다.

---

## 🤔 악성코드 분석 방법론을 왜 배워야 할까요? (The "Why")

악성코드는 사이버 공격의 가장 흔하고 치명적인 도구 중 하나입니다. 랜섬웨어, 스파이웨어, 트로이목마, 웜, 바이러스 등 다양한 형태의 악성코드는 기업과 개인에게 막대한 피해를 입힙니다. 악성코드 분석은 이러한 악성코드가 어떤 기능을 수행하고, 어떻게 시스템을 감염시키며, 어떤 정보를 탈취하는지 등을 파악하여 효과적인 방어 전략을 수립하는 데 필수적인 과정입니다.

**[악성코드 분석 지식의 중요성]**
*   **위협 인텔리전스 확보:** 새로운 악성코드의 작동 원리를 이해하고, 공격자의 전술, 기술, 절차(TTPs)를 파악하여 위협 인텔리전스(Threat Intelligence)를 구축합니다.
*   **탐지 및 차단 규칙 생성:** 분석 결과를 바탕으로 백신, IDS/IPS, EDR 등 보안 솔루션의 탐지 시그니처나 차단 규칙을 생성하고 업데이트합니다.
*   **침해 사고 대응:** 감염된 시스템에서 악성코드의 흔적을 찾아내고, 확산 경로를 파악하며, 제거 및 복구 계획을 수립합니다.
*   **AI 기반 악성코드 분석:** AI는 방대한 악성코드 샘플을 분석하여 특징을 자동으로 추출하고, 분류하며, 행위를 예측하여 악성코드 분석가의 업무 효율성을 극대화하고, 알려지지 않은 악성코드까지 탐지할 수 있도록 돕습니다.

> **보안 전문가에게 악성코드 분석 방법론이란?**
>
> 악성코드라는 **'미지의 바이러스'**를 해부하여 **'그 정체와 작동 원리'**를 밝혀내고, 이에 대한 **'치료제(방어 전략)'**를 개발하는 **'디지털 의학'**입니다. AI는 이 해부 과정에서 **'고성능 현미경'**이자 **'자동 진단 시스템'** 역할을 수행하여, 인간 분석가가 놓칠 수 있는 미세한 특징이나 복잡한 행위 패턴을 찾아내고, 새로운 악성코드에 대한 대응 시간을 단축시킵니다.

## 1. 🔬 악성코드 분석의 종류: 정적 분석 vs 동적 분석

악성코드 분석은 크게 두 가지 방법론으로 나뉩니다.

### 1.1. 🔬 정적 분석 (Static Analysis)

*   **설명:** 악성코드를 **실행하지 않고** 파일 자체를 분석하여 정보를 얻는 방법입니다.
*   **주요 활동:**
    *   **파일 정보 확인:** 파일 형식(PE, ELF), 크기, 컴파일 시간, 섹션 정보 등.
    *   **문자열 추출:** 실행 파일 내에 포함된 문자열(API 함수 이름, URL, IP 주소, 에러 메시지 등) 추출.
    *   **패킹 여부 확인:** UPX, ASpack 등 패커(Packer) 사용 여부 확인.
    *   **디스어셈블리:** 실행 파일을 어셈블리 코드로 변환하여 코드 흐름 분석.
    *   **역공학 (Reverse Engineering):** 디컴파일러를 사용하여 고수준 언어(C, C++)로 변환하여 로직 분석.
*   **장점:** 안전하고, 분석 시간이 짧으며, 패킹 여부나 특정 문자열 등 초기 정보 획득에 용이.
*   **단점:** 난독화, 패킹, 안티-분석 기법 등으로 인해 실제 행위를 파악하기 어려움.

#### **AI 연동 전략**

| 구분 | AI 연동 방안 |
| :--- | :--- |
| **파일 특징 추출 자동화** | AI가 파일의 헤더 정보, 섹션 정보, 임포트/익스포트 함수 목록, 문자열 등을 자동으로 추출하고 특징 벡터화. |
| **패킹 여부/종류 분류** | AI가 파일의 엔트로피, 섹션 특징 등을 분석하여 패킹 여부 및 패커 종류 분류. |
| **유사 악성코드 분류** | AI가 파일의 정적 특징을 기반으로 유사한 악성코드 패밀리 분류. |
| **난독화 해제 지원** | AI가 난독화된 코드 패턴을 분석하여 난독화 해제에 필요한 힌트 제공. |

### 1.2. 🔬 동적 분석 (Dynamic Analysis)

*   **설명:** 악성코드를 **격리된 환경(샌드박스)**에서 직접 실행하여 실제 행위를 관찰하고 분석하는 방법입니다.
*   **주요 활동:**
    *   **프로세스 모니터링:** 생성/종료되는 프로세스, 자식 프로세스 생성 여부.
    *   **파일 시스템 모니터링:** 파일 생성/수정/삭제, 레지스트리 변경.
    *   **네트워크 모니터링:** C2(Command & Control) 서버 통신, DNS 쿼리, HTTP 요청.
    *   **API 호출 모니터링:** 악성코드가 호출하는 시스템 API 함수 목록 및 인자.
    *   **메모리 덤프 분석:** 실행 중인 악성코드의 메모리 덤프를 분석하여 숨겨진 코드나 데이터 추출.
*   **장점:** 실제 행위를 파악할 수 있어 난독화된 악성코드 분석에 효과적.
*   **단점:** 분석 환경 구축의 어려움, 분석 시간 소요, 샌드박스 회피 기법에 취약.

#### **AI 연동 전략**

| 구분 | AI 연동 방안 |
| :--- | :--- |
| **행위 특징 추출 자동화** | AI가 샌드박스에서 생성된 방대한 행위 로그(API 호출 시퀀스, 파일/레지스트리 변경, 네트워크 통신)를 자동으로 파싱하고 특징 벡터화. |
| **악성 행위 패턴 탐지** | AI가 정상 행위 패턴을 학습하고, 악성코드의 비정상적인 행위 패턴(예: 특정 API 호출 시퀀스, C2 서버 통신)을 탐지. |
| **샌드박스 회피 탐지** | AI가 샌드박스 환경에서 악성코드가 분석을 회피하려는 시도(예: 특정 API 호출 지연, 가상 환경 탐지)를 탐지. |
| **행위 기반 악성코드 분류** | AI가 악성코드의 행위 특징을 기반으로 악성코드 패밀리 분류. |

## 2. 🛡️ 악성코드 분석 환경 구축: 안전한 분석을 위한 준비

악성코드를 분석할 때는 반드시 격리된 환경에서 수행해야 합니다. 실제 시스템에 감염될 경우 심각한 피해를 입을 수 있기 때문입니다.

### 2.1. 🌐 가상 환경 (Virtual Environment)

*   **설명:** `VMware`, `VirtualBox`, `Hyper-V` 등 가상화 소프트웨어를 사용하여 실제 시스템과 분리된 가상 머신(VM)을 구축합니다.
*   **장점:** 저렴하고 쉽게 구축 가능, 스냅샷 기능을 통해 분석 환경을 초기 상태로 쉽게 되돌릴 수 있음.
*   **단점:** 악성코드가 가상 환경을 탐지하고 분석을 회피할 수 있음.

### 2.2. 📦 샌드박스 (Sandbox)

*   **설명:** 악성코드를 안전하게 실행하고 행위를 관찰하기 위해 특별히 설계된 격리된 환경입니다. `Cuckoo Sandbox`, `Any.Run` 등이 대표적입니다.
*   **장점:** 악성코드의 모든 행위를 자동으로 기록하고 분석 보고서를 생성.
*   **단점:** 샌드박스 회피 기법에 취약, 분석 시간이 오래 걸릴 수 있음.

## 3. 🛡️ AI 기반 악성코드 분석 및 대응 전략

AI는 악성코드 분석의 모든 단계에서 인간 분석가를 지원하고, 분석의 효율성과 정확도를 극대화하며, 자동화된 대응을 가능하게 합니다.

### 3.1. 🤖 AI 기반 악성코드 분류 (Malware Classification)

*   **기술:** 머신러닝(SVM, RandomForest), 딥러닝(CNN, RNN).
*   **AI 연동 전략:**
    *   **정적 특징 기반 분류:** AI가 파일의 정적 특징(헤더, 섹션, 문자열, API 임포트)을 기반으로 악성코드 여부 및 패밀리 분류.
    *   **동적 특징 기반 분류:** AI가 샌드박스에서 추출된 행위 특징(API 호출 시퀀스, 파일/레지스트리 변경)을 기반으로 악성코드 여부 및 패밀리 분류.
    *   **하이브리드 분류:** 정적/동적 특징을 모두 활용하여 분류 정확도 향상.

### 3.2. 🤖 AI 기반 악성코드 행위 예측 (Malware Behavior Prediction)

*   **기술:** 딥러닝(RNN, LSTM), 강화 학습(Reinforcement Learning).
*   **AI 연동 전략:**
    *   **행위 시퀀스 예측:** AI가 악성코드의 초기 행위를 기반으로 다음 발생할 행위를 예측하여 선제적 대응.
    *   **C2 서버 통신 예측:** AI가 악성코드의 네트워크 통신 패턴을 분석하여 C2 서버의 위치나 통신 방식을 예측.

### 3.3. 🤖 AI 기반 악성코드 탐지 및 대응 자동화

*   **기술:** 지도 학습, 비지도 학습, SOAR 연동.
*   **AI 연동 전략:**
    *   **자동 시그니처 생성:** AI가 분석된 악성코드의 특징을 기반으로 백신, IDS/IPS, EDR의 탐지 시그니처를 자동으로 생성.
    *   **자동 차단/격리:** AI가 악성코드를 탐지하면, EDR, 방화벽 등 보안 솔루션에 연동하여 감염된 시스템을 자동으로 격리하거나 악성코드 실행을 차단.
    *   **위협 인텔리전스 업데이트:** AI가 분석된 악성코드 정보를 위협 인텔리전스 플랫폼에 자동으로 업데이트.

## 👨‍💻 현직자 통합 시나리오: AI 기반 악성코드 자동 분석 및 대응 시스템

**[상황]**
AI 보안 엔지니어 '제미니'는 매일 수집되는 수많은 의심 파일에 대해 AI 기반의 자동 악성코드 분석 및 대응 시스템을 구축하고자 합니다. 이 시스템은 의심 파일을 자동으로 분석하고, 악성코드 여부 및 종류를 분류하며, 탐지된 악성코드에 대해 자동화된 대응 조치를 취해야 합니다.

**[워크플로우]**
1.  **의심 파일 수집:** 이메일 첨부 파일, 웹 다운로드 파일, 네트워크 트래픽에서 추출된 파일 등 의심 파일을 자동으로 수집합니다.
2.  **AI 기반 정적 분석:**
    *   AI가 파일의 헤더 정보, 섹션 정보, 임포트/익스포트 함수 목록, 문자열 등을 자동으로 추출하고 특징 벡터화합니다.
    *   AI가 파일의 엔트로피를 분석하여 패킹 여부를 판단합니다.
    *   AI 기반 분류 모델(RandomForest)이 정적 특징을 기반으로 악성코드 여부를 1차 분류합니다.
3.  **AI 기반 동적 분석 (샌드박스 연동):**
    *   1차 분류에서 악성코드로 의심되는 파일을 샌드박스(Cuckoo Sandbox 등)에 자동으로 제출합니다.
    *   AI가 샌드박스에서 생성된 행위 로그(API 호출 시퀀스, 파일/레지스트리 변경, 네트워크 통신)를 자동으로 파싱하고 특징 벡터화합니다.
    *   AI 기반 분류 모델(LSTM)이 행위 특징을 기반으로 악성코드 여부 및 패밀리를 2차 분류합니다.
4.  **AI 기반 위협 스코어링 및 대응:**
    *   정적/동적 분석 결과를 종합하고, 외부 위협 인텔리전스(VirusTotal) 정보를 연동하여 위협 스코어를 부여합니다.

def get_static_features(file_path):
    """파일의 정적 특징 추출 (개념)"""
    features = {}
    try:
        pe = pefile.PE(file_path)
        features['file_size'] = os.path.getsize(file_path)
        features['num_sections'] = pe.FILE_HEADER.NumberOfSections
        
        # 임포트 함수 목록에서 악성 의심 API 포함 여부
        malicious_apis = ['CreateRemoteThread', 'WriteProcessMemory', 'NtQueueApcThread']
        features['has_malicious_api'] = 0
        if hasattr(pe, 'DIRECTORY_ENTRY_IMPORT'):

                for imp in entry.imports:
                    if imp.name and imp.name.decode(errors='ignore') in malicious_apis:
                        features['has_malicious_api'] = 1
                        break
        
        # 엔트로피 (패킹 여부 판단에 사용)
        entropy = 0
        for section in pe.sections:
            entropy += section.get_entropy()
        features['avg_entropy'] = entropy / pe.FILE_HEADER.NumberOfSections if pe.FILE_HEADER.NumberOfSections > 0 else 0

    except pefile.PEFormatError:
        print(f"  - ❌ {file_path}: PE 파일 형식이 아닙니다.")
        return None
    except Exception as e:
        print(f"  - ❌ {file_path}: 정적 분석 중 오류 발생 - {e}")
        return None
    return features

## 📌 요약 정리 (Executive Summary)

1.  **악성코드 분석의 중요성**: 악성코드는 사이버 공격의 핵심 도구이므로, 그 작동 원리를 파악하여 위협 인텔리전스를 확보하고, 탐지/차단 규칙을 생성하며, 침해 사고에 대응하는 데 필수적이다.
2.  **악성코드 분석의 종류**:
    *   **정적 분석 (Static Analysis)**: 악성코드를 실행하지 않고 파일 자체를 분석 (파일 정보, 문자열, 패킹, 디스어셈블리). AI는 특징 추출 자동화, 패킹/종류 분류, 유사 악성코드 분류 지원.
    *   **동적 분석 (Dynamic Analysis)**: 격리된 환경(샌드박스)에서 실행하여 실제 행위 관찰 (프로세스, 파일 시스템, 네트워크, API 호출 모니터링). AI는 행위 특징 추출 자동화, 악성 행위 패턴 탐지, 샌드박스 회피 탐지 지원.
3.  **악성코드 분석 환경**: 안전한 분석을 위해 가상 환경(VM)이나 샌드박스(Cuckoo Sandbox) 구축이 필수적.
4.  **AI 기반 악성코드 분석 및 대응 전략**:
    *   **악성코드 분류**: AI가 정적/동적 특징을 기반으로 악성코드 여부 및 패밀리 분류 (머신러닝, 딥러닝).
    *   **악성코드 행위 예측**: AI가 악성코드의 초기 행위를 기반으로 다음 발생할 행위 예측 (딥러닝).
    *   **탐지 및 대응 자동화**: AI가 분석된 악성코드 특징을 기반으로 자동 시그니처 생성, 자동 차단/격리, 위협 인텔리전스 업데이트.
5.  **현직자 통합 시나리오**: AI 기반 악성코드 자동 분석 및 대응 시스템을 통해 의심 파일을 자동으로 수집하고, AI가 정적/동적 분석을 수행하여 악성코드 여부 및 종류를 분류하며, 위협 스코어링을 통해 자동화된 대응 조치(격리, 시그니처 업데이트, 알림)를 취하는 워크플로우를 구축한다.
6.  **보안 관점**: AI는 악성코드 분석의 모든 단계에서 인간 분석가를 지원하고, 분석의 효율성과 정확도를 극대화하며, 알려지지 않은 악성코드까지 탐지하고 자동화된 대응을 가능하게 하는 핵심 도구이다. 정적/동적 분석의 장점을 결합하고 AI를 활용하여 신속하고 효과적인 악성코드 대응 체계를 구축하는 것이 중요하다.
