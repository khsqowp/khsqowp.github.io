# 🎯 4. 모의해킹_실습_기초: 윤리적 해킹 시작하기

## 🎯 이 문서에서 배울 내용

이 문서를 끝까지 읽으면 다음을 할 수 있게 됩니다:

- ✅ 모의해킹이 무엇인지 이해하기
- ✅ 윤리적 해킹과 불법 해킹의 차이 알기
- ✅ 모의해킹 기본 절차 이해하기
- ✅ 간단한 취약점 찾기 실습하기

> **💡 미리보기:** 모의해킹은 "착한 해커"가 되어 시스템의 약점을 먼저 찾아주는 활동입니다. 진짜 해커가 오기 전에 문제를 발견하고 고치는 것이죠!

## ⚖️ 윤리적 해킹의 규칙

### 📜 절대 지켜야 할 규칙

```
1. 허가받기:
   ✅ "이 웹사이트 테스트해도 되나요?"
   ✅ 회사와 계약서 작성
   ✅ 테스트 범위 명확히 정하기
   ❌ 허가 없이 테스트 절대 금지!

2. 범위 지키기:
   ✅ 계약한 범위만 테스트
   ❌ "잠깐 다른 것도..." → 안 돼요!

3. 데이터 보호:
   ✅ 발견한 정보는 비밀 유지
   ❌ SNS에 자랑 금지!
   ❌ 데이터 복사 금지!

4. 피해 최소화:
   ✅ 서비스 중단 최소화
   ✅ 데이터 변조 금지
   ✅ "테스트용" 계정 사용

5. 즉시 보고:
   ✅ 심각한 취약점 발견 시 즉시 연락
   ✅ 상세한 보고서 작성
   ✅ 해결 방법 제시
```

### ⚠️ 법적 주의사항

```
불법이에요:
❌ 허가 없이 남의 웹사이트 해킹
❌ 발견한 취약점 악용
❌ 데이터 훔치기, 판매
❌ 서비스 방해 (DoS 공격)

합법이에요:
✅ 회사와 계약 후 모의해킹
✅ 버그바운티 프로그램 참여
✅ 자신의 시스템 테스트
✅ 학습용 실습 사이트 이용
```

## 🗺️ 모의해킹 6단계

### 1️⃣ 1단계: 정보 수집

**무엇을 알아낼까요?**

```
대상 웹사이트에 대해 조사:
┌────────────────────────────────┐
│ 📋 기본 정보:                  │
│ - 도메인: example.com          │
│ - IP 주소: 192.168.1.100       │
│ - 웹 서버: Apache 2.4          │
│ - 프로그래밍 언어: PHP 7.4     │
│                                │
│ 👥 회사 정보:                  │
│ - 회사 이름, 위치               │
│ - 직원 정보 (LinkedIn)         │
│ - 사용하는 기술                 │
└────────────────────────────────┘
```

**도구:**

```bash
# 1. 도메인 정보 조회 (whois)
whois example.com

결과:
Domain Name: EXAMPLE.COM
Registrar: GoDaddy
Creation Date: 2020-01-01
Expiration Date: 2026-01-01

# 2. IP 주소 찾기 (nslookup)
nslookup example.com

결과:
Server: 8.8.8.8
Address: 8.8.8.8#53
Name: example.com
Address: 93.184.216.34

# 3. 웹 서버 정보 (curl)
curl -I https://example.com

결과:
HTTP/1.1 200 OK
Server: Apache/2.4.41 (Ubuntu)
X-Powered-By: PHP/7.4.3
```

## 🧪 실습: 안전한 환경에서 시도하기

### 실습 1: DVWA로 SQL Injection 체험

**DVWA = Damn Vulnerable Web Application**
(의도적으로 취약하게 만든 학습용 웹사이트)

```
Step 1: DVWA 설치
https://github.com/digininja/DVWA

또는 온라인 버전:
http://demo.testfire.net/ (Altoro Mutual)

Step 2: 로그인
- ID: admin
- PW: password

Step 3: 보안 수준 설정
- Security Level: Low로 설정

Step 4: SQL Injection 메뉴 선택

Step 5: 테스트!
```

**시도해보기:**

```
User ID 입력창에 입력:

시도 1: 1
결과: 정상적인 사용자 정보 표시

시도 2: 1'
결과: SQL 에러 메시지
→ SQL Injection 취약점 확인!

시도 3: 1' OR '1'='1
결과: 모든 사용자 정보 표시
→ 취약점 악용 성공!

시도 4: 1' UNION SELECT null, version()#
결과: 데이터베이스 버전 정보 표시
→ 추가 정보 수집 성공!
```

### 실습 3: 약한 비밀번호 찾기

- 로그인 페이지에서 시도
- Burp Suite에서 요청 확인

Step 4: Intruder 기능 사용
- 비밀번호 부분 선택
- "Sniper" 공격 타입 선택
- 페이로드: admin, password, 123456 등

Step 5: 공격 시작!
- "Start attack" 클릭
- 응답 길이 확인
- 다른 응답 = 성공!
```

## 📝 모의해킹 보고서 작성하기

### 보고서 템플릿

```markdown
# 모의해킹 보고서

## 1. 요약
- 테스트 대상: example.com
- 테스트 기간: 2025-01-15 ~ 2025-01-20
- 발견 취약점: 5개 (Critical: 1, High: 2, Medium: 2)

## 2. 발견한 취약점

### 취약점 #1: SQL Injection
**심각도:** Critical 🔴

**위치:**
- URL: https://example.com/login.php
- 파라미터: username

**설명:**
로그인 페이지에서 사용자 입력값을 검증 없이 SQL 쿼리에 직접 사용합니다.

**재현 방법:**
1. https://example.com/login.php 접속
2. Username: `admin' OR '1'='1--`
3. Password: 아무거나
4. 로그인 버튼 클릭

**증거:**
[스크린샷 첨부]

**영향:**
- 모든 사용자 계정으로 로그인 가능
- 데이터베이스 내 모든 정보 유출 가능
- 데이터 변조/삭제 가능

**해결 방법:**
```php
// ❌ 현재 코드
$query = "SELECT * FROM users WHERE username = '$username'";

// ✅ 수정 코드
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ?");
$stmt->execute([$username]);
```

**참고 자료:**
- OWASP SQL Injection: https://owasp.org/...
- CWE-89: https://cwe.mitre.org/...

---

## 3. 권장사항

### 즉시 조치 필요 (Critical/High):
1. SQL Injection 수정 (Critical)
2. XSS 필터링 추가 (High)
3. HTTPS 적용 (High)

### 계획적 조치 (Medium/Low):
4. 비밀번호 정책 강화 (Medium)
5. 세션 타임아웃 설정 (Medium)

---

## 4. 결론

총 5개의 취약점을 발견했으며, 그 중 1개는 즉시 수정이 필요한
Critical 등급입니다. 빠른 시일 내에 조치를 권장합니다.
```

## ❓ 자주 묻는 질문 (FAQ)

### Q1: 모의해킹을 배우면 해커가 되나요?
**A:** 아니요! 모의해킹은 **방어**를 위한 기술입니다.
```
모의해킹 전문가:
✅ 회사의 보안을 지킴
✅ 합법적으로 높은 연봉
✅ 사회에 기여

악의적 해커:
❌ 불법 행위
❌ 감옥행
❌ 평생 기록 남음
```

### Q2: 허가 없이 친구 웹사이트 테스트해도 되나요?
**A:** **절대 안 됩니다!**
```
설령 친구라도:
1. 서면 허가 받기
2. 테스트 범위 명시
3. 결과 공유 합의

이것도 없으면 불법!
```

### Q3: 버그바운티가 뭔가요?
**A:** 취약점 발견하면 돈을 주는 프로그램입니다!
